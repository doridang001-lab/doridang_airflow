# Excel Rename Automation — PRD

## Overview

### 한 줄 정의
날짜 기반 규칙으로 .xlsx 파일 이름을 일괄 변경하는 Python 자동화 스크립트

### 제품 목표
- 지정 폴더 내 .xlsx 파일을 `YYYYMMDD_원본파일명` 형식으로 일괄 자동 변환
- 변경 전 미리보기(Preview) 확인 단계를 통해 의도치 않은 이름 변경 방지
- 단순 터미널 명령 실행으로 반복 작업 시간 절감

### 배경
업무 중 다수의 엑셀 파일을 날짜 기반 명명 규칙에 맞게 수동으로 변경하는 작업이 반복되고 있다. 파일 수가 많을수록 실수가 생기고 시간이 낭비되므로, 규칙을 적용한 자동화 도구가 필요하다.

---

## Core Value

### 사용자 문제
- 수십~수백 개의 .xlsx 파일을 하나씩 날짜 접두어를 붙여 이름 변경해야 함
- 수작업 중 파일명 오타, 누락 등 실수 발생
- 변경 작업 후 원본 파일명을 기억하기 어려워 추적이 불가

### 해결 방식
- Python 스크립트가 지정 폴더를 스캔하여 .xlsx 파일 목록 수집
- `YYYYMMDD_원본파일명.xlsx` 형식으로 변경 계획(Preview) 테이블 출력
- 사용자가 확인 후 `y` 입력 시 일괄 `os.rename()` 처리
- 원본 파일을 직접 이름 변경(덮어쓰기) 방식으로 추가 복사본 없이 처리

### 차별점
- Preview 단계가 있어 실수 방지
- 별도 설치 없이 Python 표준 라이브러리(`os`, `pathlib`, `datetime`)만 사용
- 날짜 소스를 파일 수정일(mtime) 또는 오늘 날짜 중 선택 가능

## Targets & Scenarios

### 타겟 사용자
- 반복적으로 엑셀 파일 이름을 날짜 기반으로 정리해야 하는 사무직 직원
- 기초 Python 실행 환경을 갖춘 내부 운영팀
- 파일 관리 자동화에 관심 있는 비개발자

### 사용 시나리오
- 매일 다운로드되는 보고서 .xlsx 파일들을 `20240115_보고서명.xlsx` 형식으로 정리
- 월말 정산 시 폴더 내 모든 엑셀 파일에 날짜 접두어를 일괄 적용
- 스크립트 실행 → Preview 확인 → 확인 입력 → 자동 변경 완료

## Success Metrics

### 핵심 KPI
- 파일 100개 기준 실행 시간 3초 이내
- Preview 단계에서 변경 대상/결과를 100% 정확히 표시
- 사용자 확인 없이는 단 한 개의 파일도 변경되지 않음 (안전성 100%)
- 동일 이름 충돌 발생 시 자동 감지 및 경고 출력

### 리스크/이슈
- 이미 날짜 접두어가 붙은 파일을 중복 적용할 경우 파일명 오염 가능 → 중복 감지 로직 필요
- 날짜 소스를 파일 수정일로 할 경우 복사/이동 후 수정일이 달라질 수 있음
- Windows/macOS 경로 구분자 차이로 인한 호환성 이슈 → `pathlib` 사용으로 해소
