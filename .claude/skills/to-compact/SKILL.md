## 역할

현재 대화 전체를 구조화된 compact 요약으로 만들어 클립보드에 복사하고, 새 터미널을 열어 사용자가 Ctrl+V 후 추가 요청을 입력하면 새 세션에서 바로 이어갈 수 있게 한다.

## 실행 절차

### Step 1 — Compact 요약 생성

현재 대화 전체를 분석해서 아래 포맷으로 compact 텍스트를 작성한다.  
이 텍스트는 새 세션의 Claude가 맥락을 즉시 파악할 수 있도록 **사용자 관점의 자연스러운 브리핑** 형식으로 작성한다.

```
[이전 세션 컨텍스트]
프로젝트: C:\airflow (Airflow ETL 파이프라인 - 도리당 F&B 데이터)

## 진행했던 작업
- (진행된 작업 항목들을 구체적으로 요약)

## 결정된 사항
- (대화 중 확정된 설계/구현 결정들)

## 현재 파일 상태
- (생성/수정된 파일 목록과 간단한 설명)

## 이어서 할 작업
(다음으로 할 작업 또는 미완성 작업)

---
```

규칙:
- 기술적으로 정확하고 구체적으로 작성 (파일명, 함수명, 경로 등 포함)
- 새 Claude가 읽으면 즉시 작업 재개가 가능한 수준
- 마지막에 `---` 구분선 추가 (사용자가 그 아래에 추가 요청을 입력하도록)

### Step 2 — 임시 파일에 저장

compact 텍스트를 아래 경로에 저장한다.

- WSL 환경: `/mnt/c/temp/claude_compact.txt`
- Windows 환경: `C:\temp\claude_compact.txt`

디렉토리가 없으면 먼저 생성:
```bash
mkdir -p /mnt/c/temp
```

Write 도구로 파일 저장 후 Bash로 클립보드에 복사:
```bash
clip.exe < /mnt/c/temp/claude_compact.txt
```

### Step 3 — 새 터미널 열기

```bash
cmd.exe /c start wt
```

### Step 4 — 사용자 안내

다음 메시지를 출력한다:

```
클립보드에 compact 내용이 복사되었습니다.

새 터미널에서:
1. cc (WSL + claude 실행)
2. Ctrl+V
3. 이어서 할 작업 입력 후 Enter

예시:
[이전 세션 컨텍스트]
...
---
to-compact 스킬 계속 작업해줘
```

## 주의 사항

- compact 텍스트는 반드시 **새 Claude가 읽을 수 있는 완결된 맥락**을 담아야 한다
- 코드 스니펫보다는 **결정사항과 상태 요약**에 집중
- 파일 저장 실패 시 compact 텍스트를 화면에 그대로 출력해서 사용자가 수동으로 복사할 수 있게 한다
