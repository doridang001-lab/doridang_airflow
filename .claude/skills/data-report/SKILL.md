## 역할

사용자가 제공하는 데이터, 가설, 초안, 메모를 받아 도리당 TF 리포트 표준 포맷으로 구조화하고, 완성된 마크다운 파일을 OneDrive에 저장한다.

## 저장 경로

- WSL 경로: `/mnt/c/Users/민준/OneDrive - 주식회사 도리당/data/report/`
- 파일명 규칙: `YYMMDD_제목.md` (예: `260327_묵은지분석.md`)

## 리포트 표준 포맷

```
"YY.MM.DD(요일) 리포트 제목"

# 0. 선결론 (선택 — 핵심 결론을 한 문장으로)

# 1. 문제 및 배경 / 분석 목적
- 이슈 발생 배경
- 분석이 필요한 이유

# 2. 목표
- 이 분석으로 확인하려는 것

# 3. 분석 기준
## 3.1. 대상 매장
## 3.2. 비교 기간
## 3.3. 주요 지표

# 4. 가설
## 4.1. [가설 1 — "~라면 ~해야 함" 형식]
- 데이터 설명
- 표 (있는 경우)
>
> - 인사이트 1
> - 인사이트 2

## 4.2. [가설 2]
...

# 5. 분석 한계점 (선택)
- 데이터 제약
- 통제 불가 변수

# 6. 결론
- 가설별 판단 요약
- 최종 판단

> 최종 판단
> - 핵심 결론
> - 권고사항
```

## 포맷 규칙

- 데이터 비교는 반드시 **표(테이블)** 로 작성
- 비교 기간은 전/중/후 또는 변경전/변경후 구조로 구성
- 증감률은 `+X%` / `-X%` 형식
- 핵심 인사이트와 권고사항은 `>` 블록인용 안에 작성
- 채널 표시: 🛵 배민, 🚀 쿠팡, 🥕 당근
- 제목 줄에 날짜(요일) 포함

## instructions

1. 사용자 입력(데이터, 가설, 초안, 메모)을 분석
2. 어떤 섹션이 주어졌고 어떤 섹션이 빠졌는지 파악
3. 빠진 섹션은 입력 내용을 바탕으로 추론하여 보완
4. 표준 포맷에 맞게 전체 리포트 마크다운 생성
   - 데이터가 있으면 표로 변환
   - 결론/판단은 블록인용으로 강조
   - 수치 비교는 증감률 포함
5. 오늘 날짜 + 리포트 주제로 파일명 결정
6. `/mnt/c/Users/민준/OneDrive - 주식회사 도리당/data/report/` 경로에 파일 저장
7. 저장 완료 후 파일명과 주요 섹션 구성 요약 출력

## behavior

- 입력이 불완전해도 있는 내용으로 최대한 구조화
- 가설이 여러 개면 4.1, 4.2, 4.3... 순서로 분리
- 데이터 수치가 없는 섹션은 `[데이터 필요]` 태그로 표시
- 저장 전 리포트 전문을 화면에 출력하여 확인 가능하게 함
- 파일 저장 후: `저장 완료: /경로/파일명.md` 형식으로 알림
