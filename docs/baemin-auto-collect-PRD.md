# PRD: 배민 자동 수집 자동화

> 작성일: 2026-05-08  
> 작성자: 민준  
> 상태: 승인됨

---

## 1. 목적

Chrome 익스텐션(`doridang_collector_1.3`)으로 매장 담당자가 **수동 수집**하던
배달의민족 주문내역 / 우가클 / 우리가게NOW 지표를 **Airflow DAG으로 완전 자동화**한다.

---

## 2. 배경 / 문제

| 현황 | 문제점 |
|------|--------|
| 담당자가 각 매장별로 크롬 익스텐션 수동 실행 | 시간 소요, 누락 발생 |
| 수집 시점이 담당자마다 다름 | 분석 데이터 품질 저하 |
| 매장 수 증가 시 수동 운영 불가 | 향후 30+ 매장 확대 불가 |

---

## 3. 사용자 및 이해관계자

| 역할 | 기대 효과 |
|------|----------|
| 운영 담당자 | 수동 수집 업무 완전 제거 |
| 데이터 분석팀 | 일관된 수집 시간 + 포맷으로 데이터 품질 향상 |
| 점장 / 운영팀 | 매일 아침 전날 매출 자동 확인 가능 |

---

## 4. 기능 요구사항

| 기능 | 우선순위 | 설명 |
|------|----------|------|
| 자동 로그인 | P0 | 실제 사용자처럼 로그인 (탐지 우회 포함) |
| 주문내역 수집 | P0 | 전일 또는 지정 기간 주문 상세 + 정산정보 |
| 우가클(우리가게클릭) 수집 | P0 | 마케팅 성과 데이터 (월 단위) |
| 메인 지표 수집 | P1 | 우리가게NOW 6개 지표 (별점, 주문접수율 등) |
| 로그아웃 + 랜덤 대기 | P0 | 탐지 방지용 자연스러운 세션 종료 |
| CSV 저장 | P0 | OneDrive analytics 폴더 자동 저장 |
| 날짜 범위 수동 지정 | P1 | `dag_run.conf`로 start/end_date 지정 가능 |

---

## 5. 비기능 요구사항

- **탐지 우회**: 배민 봇 탐지 시스템 회피 (실제 사용자 행동 모방)
- **신뢰성**: 매장별 실패 시 나머지 매장은 계속 진행 (전체 중단 금지)
- **테스트 용이성**: `TARGET_STORES` 리스트로 대상 매장 제한 가능
- **확장성**: `TARGET_STORES = []` 설정으로 전 매장 자동 확대

---

## 6. 범위 (Scope)

### In Scope
- 배달의민족 플랫폼 수집 자동화
- 테스트 대상: 송파점, 역삼점

### Out of Scope (이번 버전)
- 쿠팡이츠 / 요기요 / 땡겨요 자동화
- 리뷰 수집
- 광고 변경 이력 수집
- 실시간 알림

---

## 7. 성공 기준

| 지표 | 기준 |
|------|------|
| 자동 실행 | 매일 08:50 KST 정상 트리거 |
| 주문 수집 정확도 | 수동 수집 대비 건수 오차 ≤ 1% |
| 로그인 성공률 | ≥ 95% |
| 수집 완료 시간 | 2개 매장 기준 30분 이내 |
