## Implementation Steps

### 1. `beamin_staging.py` — `resolve_macro_role()` 추가
`safe_run_id_part()` 정의 근처(파일 상단부)에 추가. **핵심 제약: `safe_run_id_part()`가 non-ASCII를 제거하므로 "상위"/"하위"는 둘 다 `_`로 뭉개진다 → 폴더 slug는 반드시 ASCII(`top`/`bottom`).**

```python
_MACRO_ROLE_ALIASES: dict[str, tuple[str, str]] = {
    "상위": ("상위", "top"), "top": ("상위", "top"), "upper": ("상위", "top"), "1": ("상위", "top"),
    "하위": ("하위", "bottom"), "bottom": ("하위", "bottom"), "lower": ("하위", "bottom"), "2": ("하위", "bottom"),
}


def resolve_macro_role(raw: str | None) -> dict:
    """환경변수 역할값을 {range, slug}로 정규화한다.

    미설정/미지원 값이면 range=None(전량 수집), slug=""(프리픽스 없음, 현행 동작).
    """
    key = str(raw or "").strip().lower()
    collect_range, slug = _MACRO_ROLE_ALIASES.get(key, (None, ""))
    return {"range": collect_range, "slug": slug}
```

### beamin_staging.py (현재 export_staging_to_inbox — 이 함수만 수정)
```python
from modules.transform.utility.paths import LOCAL_DB

def safe_run_id_part(value: str | None) -> str:
    return re.sub(r"[^A-Za-z0-9_.~-]+", "_", str(value or "manual")).strip("_")[:120]

def export_staging_to_inbox(
    local_baemin: Path,
    run_id: str | None,
    inbox_dir: Path,
    meta: dict | None = None,
    *,
    replace_existing: bool = False,
) -> Path:
    if not local_baemin.exists():
        raise RuntimeError(f"staging 경로 없음: {local_baemin}")
    run_part = safe_run_id_part(run_id)
    inbox_run = inbox_dir / f"manual__{run_part}"
    tmp_run = inbox_run.with_name(f"_tmp__{inbox_run.name}")
    if tmp_run.exists():
        shutil.rmtree(tmp_run)
    if inbox_run.exists():
        if not replace_existing:
            raise FileExistsError(f"inbox run 폴더가 이미 존재함: {inbox_run}")
        shutil.rmtree(inbox_run)
    dst_root = tmp_run / "baemin_macro"
    file_count = 0
    for src in local_baemin.rglob("*"):
        if not src.is_file():
            continue
        rel = src.relative_to(local_baemin)
        dst = dst_root / rel
        dst.parent.mkdir(parents=True, exist_ok=True)
        shutil.copy2(src, dst)
        file_count += 1
    tmp_run.mkdir(parents=True, exist_ok=True)
    if meta is not None:
        (tmp_run / "_meta.json").write_text(
            json.dumps(meta, ensure_ascii=False, indent=2, default=str), encoding="utf-8")
    tmp_run.rename(inbox_run)
    logger.info("배민 inbox export 완료: %s (%d files)", inbox_run, file_count)
    return inbox_run
```

```
  4. 전체 Test Cases 재실행

```

## Constraints
- 폴더 slug는 **반드시 ASCII**(`top`/`bottom`). "상위"/"하위"를 그대로 폴더명에 넣으면 `safe_run_id_part()`가 `_`로 뭉개 다시 충돌 → 절대 금지.
- `export_staging_to_inbox`의 `folder_prefix` 기본값은 `""` — Pc2 DAG 등 기존 호출부는 인자 추가 없이 현행 동작 유지(하위호환 필수).
- 중앙 `ingest_inbox`/`_merge_meta`/`_cleanup_processed_folder`는 **수정하지 않는다**. `manual__*` glob과 `startswith("manual__")`로 프리픽스 폴더가 자동 처리됨.
- `DB_Beamin_Macro_Dags_Retry.py`는 인박스를 안 거치고 실제 analytics에 직접 쓰며 중앙(상위) PC에서만 도므로 이번 변경 대상 아님.
- 레거시 `DB_Beamin_Macro_Pc2_Dags.py`는 이번 방식과 무관(그 인박스는 Upload DAG가 읽지 않음). 하위 PC에서는 unpause 하지 않는다(중복 수집 방지). 삭제/정리는 이번 범위 밖.
- conf/param의 `collect_range` 명시값이 있으면 env 역할보다 우선(수동 override 존중).
