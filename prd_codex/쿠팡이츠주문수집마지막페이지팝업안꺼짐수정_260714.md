# 쿠팡이츠 주문수집 마지막 페이지 팝업 안 꺼짐(무한루프 오인) 수정

## Task
쿠팡이츠 수동수집에서 **매장 예산 초과 → 이어수집 → 마지막 페이지 도달** 경로를 탄 뒤,
CSV는 저장(`💾 저장`)되지만 진행 모달(스피너 ⏳)이 닫히지 않고 멈춘다(사용자는 "무한루프"로 인식).
원인은 `_collectOrdersManualWithCheckpoints()`의 `storeDeadlineExceeded` 이어수집 분기가
종료 시 `showSuccessModal`을 호출하지 않고 `return true` 하기 때문. 이 한 곳만 고친다.

- CSV 저장 개수 표시는 `partialIndex` 변수 사용

## Implementation Steps

1. **문제 위치 확인**: `_collectOrdersManualWithCheckpoints()`의 검증 불일치 블록(`if (!salesMatch || !countMatch)`) 안,
   `if (storeDeadlineExceeded) { ... }` 분기. 현재 이어수집 루프 후 `_downloadOrdersPartial`만 하고 `showSuccessModal` 없이 `return true`.
   → 진행 모달이 안 닫혀서 스피너가 계속 도는 "hang"이 됨.

2. **(권장) 매출/주문수 재계산 헬퍼 추가** — 4000~4008줄과 중복 제거용. 함수 오브젝트 내 적당한 위치에 메서드 추가:
   ```js
   _computeOrdersTotals(rows) {
     const valid = (rows || []).filter(r => r.is_cancelled !== 'Y');
     const sales = valid.reduce((sum, r) => {
       const val = parseInt((r['매출액'] || '0').toString().replace(/,/g, ''), 10);
       return sum + (isNaN(val) ? 0 : val);
     }, 0);
     const count = new Set(valid.map(r => r.order_id).filter(Boolean)).size;
     return { sales, count };
   }
   ```
   (헬퍼가 부담되면 4025 분기에 인라인으로 동일 계산해도 됨. 단 로직은 4000~4008과 반드시 동일하게.)

3. **`storeDeadlineExceeded` 분기 끝에 종료 모달 추가**. 기존:
   ```js
   if (resumed?.allRows?.length) {
     await this._downloadOrdersPartial(resumed.allRows, shopInfo);
   }
   return true;
   ```
   →
   ```js
   let finalRows = allRows;
   if (resumed?.allRows?.length) {
     finalRows = resumed.allRows;
     partialIndex++;
     await this._downloadOrdersPartial(resumed.allRows, shopInfo);
   }
   const { sales: finalSales, count: finalCount } = this._computeOrdersTotals(finalRows);
   const finalSalesMatch = expectedSales === 0 || finalSales === expectedSales;
   const finalCountMatch = expectedCount === 0 || finalCount === expectedCount;
   if (finalSalesMatch && finalCountMatch) {
     Utils.showSuccessModal('수집 완료', `총 ${partialIndex}개 파일 저장 완료`, { status: 'ok' });
   } else {
     Utils.showSuccessModal(
       '수집 검증 불일치',
       `저장 완료: ${partialIndex}개 파일<br>주문: ${finalCount}/${expectedCount || '읽기 실패'}건<br>매출: ${finalSales.toLocaleString()}원/${expectedSales ? expectedSales.toLocaleString() + '원' : '읽기 실패'}`,
       { status: 'warning' }
     );
   }
   return true;
   ```

4. **회귀 방지 확인**: 이 함수 내 모든 `return true;` 경로가 직전에 `Utils.showSuccessModal(...)` 또는 `_showResumeRangeModal`(자체 모달)로 진행 모달을 종료하는지 대조.

## Constraints
- 수집 로직/while 루프 조건(`depth < 10`, `!reachedEnd` 등)은 **건드리지 말 것** — 실제 무한반복이 아니라 종료 모달 누락이 원인
- `_downloadOrdersPartial` 호출 순서/중복 저장 유발 금지 (final 저장은 resumed.allRows가 있을 때 1회만)
- 매출/주문수 계산은 4000~4008줄 원본 로직과 **완전히 동일**하게 (취소 제외 `is_cancelled !== 'Y'`, 콤마 제거 parseInt)
- status 문자열은 기존 사용값(`'ok'`, `'warning'`)만 사용
