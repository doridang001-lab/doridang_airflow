### 2. `_collectCurrentPageData` — 표적 재수집 + 무결성 반환
1. 안정화 대기 후 모든 `document.querySelectorAll('.order-search-result-content > li.col-12')`를 `_parseOrderItem`으로 1차 수집. 결과를 `order_id → { rows, captured }` 맵에 저장(빈 페이지 재시도 로직은 기존 유지).
2. `captured === false`인 주문 = `missing` 목록(해당 `li` 엘리먼트 보관).
3. `missing`이 있으면 표적 재수집 (최대 3라운드):
   ```js
   for (let round = 0; round < 3 && missing.length && !this._stopFlag; round++) {
     const stillMissing = [];
     for (const it of missing) {
       // 펼쳐져 있으면 접었다 다시: 펼치기 버튼 한 번 더 클릭으로 상태 토글
       await this._closeErrorPopups();
       const res = await this._parseOrderItem(it.item, shopInfo, iso);
       if (res.captured) { map.set(res.order_id, res); }
       else stillMissing.push(it);
     }
     missing = stillMissing;
     await this._randomDelay(600, 1000);
   }
   ```
4. 끝까지 남은 `missing`이 있으면 `pageComplete=false`, 로그 `⚠️ ${currentPage}페이지 매출액 미캡처 ${missing.length}건`.
5. 맵의 모든 `rows`를 평탄화해 반환. **반환 시그니처 변경**: `return { rows: pageRows, pageComplete, missingOrderIds };` (호출부도 함께 수정).

### 3. `_recoverToPage(targetPage)` 신규 — 재조회 후 next 순차 이동
쿠팡엔 직접 페이지 점프 버튼이 없으므로 next-btn만으로 수집 없이 이동:
```js
async _recoverToPage(targetPage) {
  Utils.updateProgressModal({ debug: `🔄 ${targetPage}페이지로 복구 이동 (재조회)` });
  if (!await this._clickOrdersSearchButton()) return false;
  await this._waitForOrdersDataReady(8000);
  await this._waitForStableState(5000);
  let cur = this._getCurrentPage();
  let guard = 0;
  while (cur < targetPage && !this._stopFlag) {
    if (guard++ > targetPage + 10) return false;
    const prevId = this._getFirstOrderId();
    if (!this._clickNextPage()) return false;
    let loaded = await this._waitForPageLoad(prevId, 15000);
    if (!loaded) {
      await this._randomDelay(1500, 2500);
      await this._closeErrorPopups();
      loaded = await this._waitForPageLoad(prevId, 10000);
      if (!loaded) return false;
    }
    cur = this._getCurrentPage();
    Utils.updateProgressModal({ message: `복구 이동 중... ${cur}/${targetPage}페이지` });
  }
  return cur >= targetPage;
}
```

- 불완전 페이지 재시도: `pageComplete === false && recoveryCount < MAX_RECOVERY` 이면 `recoveryCount++` 후 `await this._recoverToPage(currentPage)` 로 같은 페이지를 다시 잡아 재수집(성공 시 누락분이 dedup 통과해 채워짐). 실패하면 경고만 남기고 계속.

## Test Cases
확장은 자동 테스트가 없으므로 수동 검증. 각 항목 PASS 확인.
1. [문법] 파일 저장 후 `chrome://extensions` → `doridang_collector_개발용` 새로고침 → 콘솔에 문법 오류 없음. 기대: 빨간 에러 없음.
2. [소량 회귀] 주문 10건 이내 매장 수집 → 기대: 기존과 동일, 매출액 불일치 경고 없음.
3. [대량 핵심] 주문 60건+ 매장 수집 → 기대: 진행 모달에 `⚠️ N페이지 매출액 미캡처 K건` 발생 시 표적 재수집으로 채워지고, **전체 재조회(attempt) 1회로 완료**. 30분 무한 반복 사라짐.
4. [전환 복구] 페이지 전환 타임아웃 발생 시 `🔄 복구 이동` → `✅ 이어서 수집` 로그 후 처음이 아니라 실패 지점부터 이어짐.
5. [정합성] 완료 모달의 수집 매출액 = 페이지 상단 기대 매출액(불일치 경고 없음).
6. [CSV] 저장된 CSV에서 `order_id` 중복 없음 + 모든 비취소 주문에 `매출액` 채워짐.
