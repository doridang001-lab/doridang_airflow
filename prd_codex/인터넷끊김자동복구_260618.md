# 인터넷끊김자동복구

## Task
쿠팡 배치 수집 중 인터넷이 잠깐 끊기면 탭 로딩이 멈추고 타임아웃 → 실패 처리됨.
F5(reload)하면 복구되므로, 끊김 감지 시 복구 대기 후 F5 재시도 로직을 추가한다.

## Project Conventions
- 파일: Chrome Extension MV3 JavaScript (IIFE 내부, strict mode)
- `log(msg, cls)` 함수로 대시보드 로그 출력 (`cls`: `'ok'` / `'err'` / `'info'`)
- `sleep(ms)` = `new Promise(r => setTimeout(r, ms))`
- `stopRequested` 전역 변수 = 사용자 중단 플래그, 모든 루프에서 체크
- 함수 추가 위치: IIFE 내부 (`(function () { 'use strict'; ... })()`)
- 기존 함수 시그니처 변경 시 호출부도 전부 반영

### Step 1 — `isOnline` 함수 추가
`waitUrl` 함수(현재 381번째 줄) 바로 **아래**에 삽입:

```js
  // 연결 확인 (4초 타임아웃)
  async function isOnline() {
    try {
      const ctrl = new AbortController();
      setTimeout(() => ctrl.abort(), 4000);
      await fetch('https://store.coupangeats.com/merchant/login',
        { method: 'HEAD', cache: 'no-store', signal: ctrl.signal });
      return true;
    } catch (_) { return false; }
  }

  // 인터넷 복구될 때까지 대기 (최대 maxMs, 기본 10분)
  async function waitForOnline(maxMs = 600000) {
    const deadline = Date.now() + maxMs;
    let warned = false;
    while (Date.now() < deadline) {
      if (stopRequested) return false;
      if (await isOnline()) {
        if (warned) log('🌐 인터넷 연결 복구 — 수집 재개합니다', 'ok');
        return true;
      }
      if (!warned) { log('🔴 인터넷 끊김 — 복구될 때까지 대기...', 'err'); warned = true; }
      await sleep(5000);
    }
    log('인터넷 복구 대기 타임아웃 (10분)', 'err');
    return false;
  }
```

교체 후:
```js
  async function waitUrl(tabId, regexMap, ms, _netRetry = 0) {
    const deadline = Date.now() + ms;
    while (Date.now() < deadline) {
      if (stopRequested) return null;
      const url = await getUrl(tabId);
      for (const [key, re] of Object.entries(regexMap)) {
        const m = url.match(re);
        if (m) return {key, match:m, url};
      }
      await sleep(800);
    }
    // 타임아웃 → 오프라인 확인 → 복구 후 F5 재대기 (최대 3회)
    if (_netRetry < 3 && !stopRequested && !await isOnline()) {
      const ok = await waitForOnline(600000);
      if (ok && !stopRequested) {
        log(`[네트워크 복구] F5 후 재대기 (${_netRetry + 1}/3)...`);
        try { await chrome.tabs.reload(tabId); } catch(_) {}
        await sleep(3000);
        return waitUrl(tabId, regexMap, ms, _netRetry + 1);
      }
    }
    return null;
  }
```

### runner-DESKTOP-HG136JL-2.js (수정 전 waitUrl + waitForPageLoadWithRetry)
```js
  async function waitUrl(tabId, regexMap, ms) {
    const deadline = Date.now() + ms;
    while (Date.now() < deadline) {
      if (stopRequested) return null;
      const url = await getUrl(tabId);
      for (const [key, re] of Object.entries(regexMap)) {
        const m = url.match(re);
        if (m) return {key, match:m, url};
      }
      await sleep(800);
    }
    return null;
  }

  async function waitForPageLoadWithRetry(tabId, label, maxRetries = 10, perRetryMs = 10000) {
    for (let attempt = 0; attempt <= maxRetries; attempt++) {
      if (stopRequested) return false;
      if (attempt > 0) {
        log(`[F5 ${attempt}/${maxRetries}] ${label} 로딩 지연 — 새로고침...`);
        try { await chrome.tabs.reload(tabId); } catch(_) {}
        await sleep(2000);
      }
      const loaded = await waitForPageLoad(tabId, perRetryMs);
      if (loaded) return true;
      if (stopRequested) return false;
    }
    log(`${label} F5 ${maxRetries}회 시도 후 로딩 실패`, 'err');
    return false;
  }
```

## Test Cases
1. **isOnline 동작** — 크롬 콘솔에서 `isOnline()` 호출 → 온라인 시 `true`, 비행기모드 시 `false`
2. **waitUrl 네트워크 복구** — 배치 실행 중 와이파이 끊기 → 로그에 `🔴 인터넷 끊김` 표시 확인
3. **복구 재개** — 와이파이 재연결 → 로그에 `🌐 인터넷 연결 복구` → F5 재시도 → 수집 계속 진행
4. **10분 초과 미복구** — 10분 후 해당 계정 fail 처리 → 다음 계정으로 진행 (배치 중단 없음)
5. **정상 수집 무영향** — 네트워크 문제 없을 때 기존과 동일하게 수집 완료

## Constraints
- `isOnline` / `waitForOnline` 은 IIFE 내부 함수로 선언 (전역 노출 금지)
- `waitUrl` 재귀 깊이 `_netRetry` 최대 3회 — 무한 재귀 방지
- `waitForOnline` 내부에서 `stopRequested` 체크 필수 — 사용자 중단 즉시 반영
- `waitForPageLoadWithRetry` 호출부는 변경 없음 (시그니처 그대로 유지)
- Step 4 동기화는 반드시 원본(`runner-DESKTOP-HG136JL-2.js`) → 복사본 순서로 실행
