## Implementation Steps
1. **dag-debug** (스킬) — 수집실패/미수집/이중집계/타입오류 표준 디버깅 절차.
   - 트리거: "DAG 실패", "수집 안 됨", "미수집", "이중집계", "0원", "prd_codex 태스크".
   - 절차 LOOP: ①증상·범위(dag_id/store/sale_date) ②실측(로그·마커·parquet, `scripts/_diag_*`·`_check_*`·`_verify_*` 재사용, 베이스 `scripts/_base.py`) ③원인분류(수집단 vs 변환단 vs 스케줄) ④최소재현(`python -X utf8 -c "from dags.<domain>.<DAG> import dag"` + 함수 단위) ⑤수정(로직은 pipelines/, DAG는 오케스트레이션만) ⑥재검증 ⑦재발방지(NoData 마커·가드·lookback).
   - 참조: `harness/common.md`·`baemin_macro.md`·`coupang_macro.md`·`unified_sales_grp.md`·`fin_product.md`·`review_checklist.md`, `docs/codex/dag-change-checklist.md`·`crawling-gotchas.md`.
2. **unified-check** (스킬) — UnifiedSales/Review 교차검증.
   - 점검: 0원 탐지 / 배달행 이중집계(okpos+posfeed 우선, **toorder 제외**; okpos 매장 자체는 정상) / 소스별 교차합계 / qty 타입 / 미수집 lookback.
   - 참조: `harness/unified_sales_grp.md`·`review_checklist.md`, `scripts/_check_okpos_match.py`·`_verify_ingest_integrity.py`·`_verify_okpos_columns.py`, DAG `DB_UnifiedSales_Dags.py`·`DB_UnifiedReview_Dags.py`·`DB_UnifiedSales_Schedule_Guard_Dags.py`.
3. **ollama-classify** (스킬) — 로컬 Ollama(qwen) 분류. **신규 클라이언트 작성 금지, `modules/transform/utility/qwen_client.py` 재사용**(아래 Reference Code).
   - 절차: ①환경확인(client/model_list 출력) ②모델 형식·존재 검증(`name:tag`) ③샘플 3~5건 검증 ④JSON 스키마 고정+fallback ⑤배치 dry-run.
   - 참조 파이프라인: `DB_ToOrderMenu_LLM.py`, `DB_FinProduct.py`, `DB_FinProduct_Map.py`, `harness/fin_product.md`.
4. **harness-run** (스킬) — phase/step 실행.
   - `python scripts/execute.py`(기본 git 미변경), `--git`/`--git --push`는 명시 시에만. 결과는 `Strategy_HarnessMonitoring_01_Dashboard_Dags`로 점검. 컨테이너는 `./phases`·`./harness`·`./scripts`·`./AGENTS.md` 마운트 필요.
5. **baemin-collect** (스킬) — 배민 Selenium 수집 지식.
   - 대전제 API 불가→Selenium. 함정: NOW 매장선택 not interactable→**URL이동 전환**, 렌더러 타임아웃 누적→드라이버 재시작·fail-fast, 60계정 격리.
   - 참조: `modules/extract/croling_beamin.py`, `harness/baemin_macro.md`, `DB_Beamin_Macro_Dags*.py`.
6. **coupang-extension** (스킬) — 쿠팡이츠 크롬확장 배치 수집 지식.
   - 대전제 Selenium은 Akamai 차단→크롬확장 배치. 함정: 연타 금지, shell-ready 실측 확인, F5 reload-resume(**manual 한정·reload 2회 한도·날짜필터 재적용**), watchdog.
   - 참조: `coupang_extension_build/`(background.js·runner.js), `harness/coupang_macro.md`, `docs/coupang-boot-automation.md`.
7. **dag-fixer** (에이전트, model: sonnet, tools: Read,Grep,Glob,Edit,Write,Bash) — prd_codex/실패 DAG를 받아 dag-debug 절차를 실행 주체로 자율 수행(조사→재현→수정→검증). **승인 없이 바로 실행**하되 OneDrive·push만 승인. 완료 시 원인/수정파일/검증결과 한글 요약.
8. **crawl-runner** (에이전트, model: sonnet, tools: Read,Grep,Glob,Edit,Bash) — Selenium/크롬확장 수집 실행·복구. crawl 스킬이 설계, 이 에이전트가 실행. 배민→baemin-collect, 쿠팡→coupang-extension 지식 적용. 무작정 재시도 금지(원인 분류 먼저).
