## Implementation Steps

1. **import에 `_kst_now` 추가**
   `DB_OKPOS_Card_Test.py` 상단의 `from modules.transform.pipelines.db.DB_OKPOS_Sales import ( ... )`
   블록에 `_kst_now` 를 추가한다 (이미 같은 모듈에서 다수 심볼을 import 중).

2. **`_resolve_sale_date()` 날짜 우선순위 변경** (현재 48~69행)
   - 현재: `conf.sale_date` → `context.ds` → `datetime.now()-1`
   - 변경: `conf.sale_date` → **KST 어제(`_kst_now() - timedelta(days=1)`)**
   - 버그 원천인 `context.ds` 분기를 **제거**한다 (ds는 UTC 논리날짜라 이틀 stale).
   - `source` 라벨: conf면 `"dag_run.conf.sale_date"`, 아니면 `"kst_yesterday"`.
   - `context["ti"].xcom_push(key="sale_date", value=sale_date)` 와 `return sale_date` 는 **유지**
     (download/save/validate 태스크가 XCom으로 sale_date를 받는 체인 불변).

   변경 후 형태(명세):
   ```python
   def _resolve_sale_date(**context) -> str:
       dag_run = context.get("dag_run")
       conf = (getattr(dag_run, "conf", None) or {}) if dag_run else {}
       sale_date = str(conf.get("sale_date") or "").strip()
       if sale_date:
           try:
               datetime.strptime(sale_date, "%Y-%m-%d")
           except ValueError as exc:
               raise ValueError(f"sale_date must be YYYY-MM-DD: {sale_date}") from exc
           source = "dag_run.conf.sale_date"
       else:
           sale_date = (_kst_now() - timedelta(days=1)).strftime("%Y-%m-%d")
           source = "kst_yesterday"
       context["ti"].xcom_push(key="sale_date", value=sale_date)
       logger.info("OKPOS card test sale_date resolved: %s (%s)", sale_date, source)
       return sale_date
   ```

3. **`validate_okpos_card_lookback`는 수정하지 않는다.** 수집이 KST 어제로 정상화되면
   검증 윈도우(`수집일+1`부터 7일 역방향)가 6/15·6/14…를 올바르게 포함한다.
