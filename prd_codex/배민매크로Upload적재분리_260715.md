# 배민 매크로 Upload 적재 분리 (수집 → inbox → 중앙 스케줄 Upload)

## Task
배민 매크로 수집을 analytics에 **직접 적재**하던 구조를 분리한다. 각 PC의 수집 DAG은
`Collect_Data/영업관리부_수집/_baemin_upload_inbox/manual__{run_id}/`에 데이터 + `_meta.json`
만 내보내고, **중앙 PC 한 대**의 새 `DB_Beamin_Macro_Upload_Dags`가 스케줄로 inbox 전체를
`ANALYTICS_DB/baemin_macro/...` 파티션으로 적재하고 ToOrder 교차검증을 수행한다. 여러 PC가
동시에 OneDrive analytics에 쓰는 레이스를 없애기 위함이다. 이미 존재하는 PC2 흐름
(`DB_Beamin_Macro_Pc2_Dags` + `ingest_baemin_pc2_inbox`)의 패턴을 이식·재사용한다.

## Implementation Steps

### 1. `modules/transform/pipelines/db/beamin_staging.py` (신규)
PC2(`DB_Beamin_Macro_Pc2_Dags.py`)의 인라인 스테이징 헬퍼를 이 모듈로 **이동**(로직 동일).
공개 함수(언더스코어 없이):
- `BAEMIN_STAGE_ATTRS` — 아래 Reference의 `_BAEMIN_STAGE_ATTRS` 그대로 이동
- `local_stage_paths(subdir: str) -> tuple[Path, Path]`:
  `local_analytics = LOCAL_DB / subdir`, `return local_analytics, local_analytics / "baemin_macro"`
  (PC2는 `subdir="analytics_stage_pc2"`, 메인은 `"analytics_stage_main"`)
- `init_empty_staging(local_baemin: Path)` — PC2 `_init_empty_staging` 이동
- `patch_baemin_staging_paths(local_analytics: Path)` / `restore_baemin_staging_paths(analytics_original, originals)`
  — PC2 `_patch_/_restore_baemin_staging_paths` 이동
- `export_staging_to_inbox(local_baemin, run_id, inbox_dir: Path, meta: dict | None = None) -> Path`:
  PC2 `_export_staging_to_inbox`를 이동하되 **inbox_dir를 인자로** 받고(기존 하드코딩된
  `COLLECT_DB/"영업관리부_수집"/"_baemin_pc2_inbox"` 대신), `meta`가 있으면 원자적 rename
  이전 tmp 폴더 안에 `_meta.json`(`json.dumps(meta, ensure_ascii=False)`, utf-8)으로 저장.
  `manual__{safe_run_id}` 폴더명 규칙과 tmp→rename 원자성은 그대로 유지.
- `_safe_run_id_part` 헬퍼도 함께 이동.

### 2. `dags/db/DB_Beamin_Macro_Pc2_Dags.py` (수정, 동작 보존)
인라인 헬퍼(`_local_stage_paths`, `_init_empty_staging`, `_patch_/_restore_baemin_staging_paths`,
`_export_staging_to_inbox`, `_BAEMIN_STAGE_ATTRS`, `_safe_run_id_part`) 삭제하고
`beamin_staging`에서 import. 호출부는:
- `local_stage_paths("analytics_stage_pc2")`
- `export_staging_to_inbox(local_baemin, run_id, inbox_dir=COLLECT_DB/"영업관리부_수집"/"_baemin_pc2_inbox")`
(meta 없이 호출 → 기존과 동일 결과). **inbox 경로/파일 배치 불변**.

### 3. `modules/transform/pipelines/db/DB_Beamin_pc2_distribute.py` (수정, 일반화)
- 현재 `ingest_baemin_pc2_inbox` 본문을 `ingest_inbox(inbox_dir: Path, read_meta: bool = False) -> dict`로
  일반화. 반환 dict에 기존 요약 문자열(`summary`) + `read_meta=True`면 폴더별 `_meta.json`을
  읽어 집계한 `meta`(아래 키들의 union/concat) 포함:
  `account_list`(list concat), `validation`(list concat), `ad_stores`(list concat, dedup),
  `store_info_per_account`(list concat), `failed`(accounts/stores/orders/ads 리스트 merge),
  `target_date`(가장 흔한 값; 섞이면 로그 warning).
- `_cleanup_processed_folder(folder, inbox_dir)`로 inbox 루트를 인자화(부모 경로 검증에 사용).
- 기존 `ingest_baemin_pc2_inbox(**context)`는 `ingest_inbox(INBOX_DIR)["summary"]` 반환으로 유지
  (호출부 `DB_UnifiedSales_Dags.py`에서 문자열 반환 계약 불변).
- 신규 상수 `UPLOAD_INBOX_DIR = COLLECT_DB / "영업관리부_수집" / "_baemin_upload_inbox"`.
- `_meta.json`은 분배 대상 파일(`baemin_macro/...`)이 아니므로 `_target_files` glob에서 제외됨
  (suffix `.json` 미포함 + relpath 규칙). meta 읽기는 폴더 루트에서 직접 `folder/"_meta.json"`.

### 4. `modules/transform/pipelines/db/DB_Beamin_Macro_upload.py` (신규)
Upload DAG 태스크 콜러블. `ingest`가 집계 meta를 XCom push → 후속 검증이 소비.
기존 `DB_Beamin_Macro_Dags.py`의 검증/알림 헬퍼(`validate_orders`/`validate_ad_funnel`/
`validate_toorder`/`precheck_manual_baemin_orders`/`trigger_retry_if_needed`/`_send_alert`/
`_save_validate_log`/`_has_validation_issue`/`_safe_run_id_part`/`_collect_manual_baemin_orders`
및 관련 상수 `MANUAL_BAEMIN_ORDERS_DIR`/`_ALERT_EMAILS`/`KST`)를 이 모듈로 이동/복제하되,
XCom 소스를 `ingest`가 push한 집계 meta로 바꾼다.
- `ingest(**context) -> str`: `from ...DB_Beamin_pc2_distribute import ingest_baemin_upload_inbox`처럼
  `ingest_inbox(UPLOAD_INBOX_DIR, read_meta=True)` 호출 → `result["meta"]`를 XCom(key별) push,
  `result["summary"]` 반환.
- `precheck_manual(**context)`: 기존 `precheck_manual_baemin_orders` 로직 그대로.
- `validate_orders(**context)`: `context["ti"].xcom_pull(task_ids="ingest", key="validation")` 사용.
- `validate_ad_funnel(**context)`: `xcom_pull(..., key="ad_stores")` → `_validate_and_retry_ad_funnel(ad_stores, target_date)`.
- `validate_toorder(**context)`: `xcom_pull(..., key="account_list")`, `key="store_info_per_account")`
  → `validate_toorder_orders(account_list, store_info_per_account, target_date)`.
- `trigger_retry_if_needed(**context)`: `xcom_pull(..., key="failed")` → `count_failed_items`/`retry_needed`/
  `build_retry_conf` → `DB_Beamin_Macro_Dags_Retry` 트리거(기존 로직 이식).
- `notify_upload_result(**context)`: ingest+검증 요약 텔레그램/메일(`send_telegram`, `_send_alert`).
- `target_date` 결정: 집계 meta의 `target_date` 우선, 없으면 `pendulum.yesterday(KST).format("YYYY-MM-DD")`.

```python
def _init_empty_staging(local_baemin: Path) -> None:
    local_root = LOCAL_DB.resolve()
    target = local_baemin.resolve()
    if local_baemin.exists():
        if not target.is_relative_to(local_root):
            raise RuntimeError(f"staging 삭제 범위 오류: {local_baemin}")
        shutil.rmtree(local_baemin)
    local_baemin.mkdir(parents=True, exist_ok=True)

def _patch_baemin_staging_paths(local_analytics: Path):
    analytics_original = os.environ.get("ANALYTICS_DB")
    os.environ["ANALYTICS_DB"] = str(local_analytics)
    local_baemin = local_analytics / "baemin_macro"
    originals = []
    for module_name, attr_name, relative_parts in _BAEMIN_STAGE_ATTRS:
        module = importlib.import_module(module_name)
        originals.append((module, attr_name, getattr(module, attr_name, _MISSING)))
        setattr(module, attr_name, local_baemin.joinpath(*relative_parts))
    return analytics_original, originals
# _restore_baemin_staging_paths: 역순 setattr 복원 + ANALYTICS_DB env 복원

def _export_staging_to_inbox(local_baemin: Path, run_id, inbox_dir: Path) -> Path:  # ← inbox_dir 인자화
    run_part = _safe_run_id_part(run_id)
    inbox_run = inbox_dir / f"manual__{run_part}"
    tmp_run = inbox_run.with_name(f"_tmp__{inbox_run.name}")
    if tmp_run.exists(): shutil.rmtree(tmp_run)
    if inbox_run.exists(): shutil.rmtree(inbox_run)
    dst_root = tmp_run / "baemin_macro"
    for src in local_baemin.rglob("*"):
        if not src.is_file(): continue
        dst = dst_root / src.relative_to(local_baemin)
        dst.parent.mkdir(parents=True, exist_ok=True)
        shutil.copy2(src, dst)
    tmp_run.mkdir(parents=True, exist_ok=True)
    tmp_run.rename(inbox_run)  # ← meta 저장은 rename 이전 tmp_run 안에 _meta.json write
    return inbox_run
```

```python
SUPPORTED_SUFFIXES = {".parquet", ".csv"}

# 폴더별로: _target_files(folder) → _distribute_one_file(folder, src) → 성공 시 _cleanup_processed_folder(folder)
def _distribute_one_file(folder, src_file):
    rel = src_file.relative_to(folder)          # baemin_macro/<subtype>/.../file
    subtype = rel.parts[1]
    dst_stem = (ANALYTICS_DB / rel).with_suffix("")
    new_df = read_file(src_file)
    if subtype == "orders" and ORDER_KEY in new_df.columns:
        combined = _upsert_orders(dst_stem, new_df)   # 주문번호 기준 기존행 제거 후 concat
    else:
        combined = new_df
    write_table(combined, dst_stem)               # parquet(zstd) 적재 + 레거시 csv 제거
    return subtype, len(new_df)

def _cleanup_processed_folder(folder, inbox_dir):  # ← inbox_dir 인자화
    folder = folder.resolve()
    if folder.parent != inbox_dir.resolve(): raise ValueError(...)
    if not folder.name.startswith("manual__"): raise ValueError(...)
```
