# 쿠팡 Orders 파티션 분열 수정 및 order_id 기준 dedup 전환

## Task

쿠팡 매크로 orders parquet이 같은 물리 매장(`해운대 중동점`)의 데이터를 두 파티션(`store=해운대중동점`, `store=중동점`)으로 분산 적재하는 문제를 수정한다. 원인은 `store_normalize.py`에 `"도리당 중동점"` 매핑이 없어 CSV store_name 마지막 토큰 `"중동점"`이 그대로 파티션명이 된 것. 추가로 `_load_orders`의 dedup 전략을 9-컬럼 key 방식에서 **order_id 기준 최신 교체** 방식으로 변경한다.

### Step 2 — 파티션 병합 함수 추가 (일회성 마이그레이션)

`DB_CoupangMacro_load.py`에 아래 함수를 추가한다:

```python
def migrate_jungdong_partition(dry_run: bool = True) -> str:
    """store=중동점 파티션을 store=해운대중동점으로 병합하고 구파티션 삭제.
    
    중동점이 해운대중동점의 모든 order_id를 포함하므로
    중동점 데이터가 정본. 병합 후 해운대중동점으로 저장하고 중동점 삭제.
    """
    import shutil

    src_base = COUPANG_ORDERS_DB / "brand=도리당" / "store=중동점"
    dst_base = COUPANG_ORDERS_DB / "brand=도리당" / "store=해운대중동점"

    if not src_base.exists():
        return "중동점 파티션 없음, 스킵"

    src_files = sorted(src_base.rglob("orders_*.parquet"))
    results = []

    for src_path in src_files:
        ym = src_path.parent.name  # "ym=2026-06"
        dst_dir = dst_base / ym
        dst_path = dst_dir / src_path.name

        src_df = pd.read_parquet(src_path)

        if dst_path.exists():
            dst_df = pd.read_parquet(dst_path)
            # order_id 기준으로 중동점(최신)이 우선
            existing_ids = set(dst_df["order_id"].unique()) if "order_id" in dst_df.columns else set()
            new_ids = set(src_df["order_id"].unique()) if "order_id" in src_df.columns else set()
            # 중동점에 없는 order_id만 기존 해운대중동점에서 유지 (실질적으로 없음)
            keep_old = dst_df[~dst_df["order_id"].isin(new_ids)] if "order_id" in dst_df.columns else pd.DataFrame()
            merged = pd.concat([keep_old, src_df], ignore_index=True)
        else:
            merged = src_df

        results.append(f"{ym}: {len(merged)}행")
        if not dry_run:
            dst_dir.mkdir(parents=True, exist_ok=True)
            merged.astype(str).to_parquet(dst_path, index=False)
            logger.info("migrate: %s -> %s (%d행)", src_path, dst_path, len(merged))

    if not dry_run:
        shutil.rmtree(src_base)
        logger.info("deleted src partition: %s", src_base)

    mode = "DRY-RUN" if dry_run else "완료"
    return f"{mode} | {', '.join(results)}"
```

실행:
```python
from modules.transform.pipelines.db.DB_CoupangMacro_load import migrate_jungdong_partition
print(migrate_jungdong_partition(dry_run=True))   # 확인
print(migrate_jungdong_partition(dry_run=False))  # 실행
```

### Step 3 — `_load_orders` dedup 전략 변경 (order_id 기준 최신 교체)

현재 로직: `concat(existing, new_df)` → 9-컬럼 dedup
변경 로직: **새 CSV의 order_id에 해당하는 기존 행을 전부 제거** → new_df 추가

`_load_orders` 함수 내 파티션 저장 루프(line 306~329)를 아래로 교체:

```python
for (brand, store, ym), rows in grouped_rows.items():
    out_dir = _orders_partition_dir(brand, store, ym)
    out_dir.mkdir(parents=True, exist_ok=True)
    out_path = out_dir / f"orders_{ym}.parquet"

    new_df = pd.DataFrame(rows)

    if out_path.exists():
        try:
            existing = pd.read_parquet(out_path)
        except Exception as exc:
            logger.error("failed to read existing orders parquet %s: %s", out_path, exc)
            existing = pd.DataFrame()

        if not existing.empty and "order_id" in existing.columns and "order_id" in new_df.columns:
            # 새 CSV에 있는 order_id → 기존 행 전체 제거 (order_id 기준 latest 교체)
            new_ids = set(new_df["order_id"].unique())
            keep = existing[~existing["order_id"].isin(new_ids)]
            combined = pd.concat([keep, new_df], ignore_index=True)
        else:
            combined = pd.concat([existing, new_df], ignore_index=True)
    else:
        combined = new_df

    # 동일 CSV 내 중복 방어 (9-컬럼 key)
    combined, removed = _deduplicate_coupang_orders(combined)
    if removed:
        logger.warning("orders parquet 중복 제거: %s | %d행", out_path, removed)

    combined = combined.astype(str)
    combined.to_parquet(out_path, index=False)
    outputs.append(str(out_path))
    logger.info("saved orders parquet: %s (%d rows)", out_path, len(combined))
```

핵심 변경점:
- `order_id`가 있는 경우: 새 CSV의 order_id에 해당하는 기존 행 전부 제거 후 new_df 추가
- `order_id`가 없는 경우(fallback): 기존 concat + 9-컬럼 dedup 유지
- `_deduplicate_coupang_orders`는 동일 CSV 내부 중복 방어용으로 유지

## Constraints

- `store_normalize.py` STORE_NAME_MAP 수정 시 `"도리당 중동점"` 키 형식 준수 (브랜드 공백 지점명)
- `migrate_jungdong_partition`은 일회성 함수 — 실행 후 별도 DAG에 등록 불필요
- `_deduplicate_coupang_orders` 함수 자체는 수정하지 않음 (CSV 내부 중복 방어용으로 유지)
- 기존 `repair_coupang_orders_parquet` 함수 유지 (다른 스토어 repair 용도)
- parquet 저장 시 `.astype(str)` 유지 (기존 패턴)
- `migrate_jungdong_partition` dry_run=False 실행 전 반드시 dry_run=True 결과 확인
