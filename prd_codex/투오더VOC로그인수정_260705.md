```python

    deadline = time.time() + 8.0

        current_url = driver.current_url
        if "/dashboard" in current_url or "/auth" not in current_url:
            return
        time.sleep(0.5)

    current_url = driver.current_url
    if "/dashboard" not in current_url and "/auth" in current_url:
```

**3-3.** `_collect_account_session` 에서 `_do_login`을 감싼 부분: `CredentialError`면 stage 이름을 `"login_credential"`로 보존. 현재:

→ 교체:

## Verification Loop
구현 완료 후 아래 루프를 **모든 Test Cases PASS까지** 반복한다.
```
LOOP until all PASS:
  1. Test Cases 1~7 순서대로 실행
  2. FAIL 항목 → 원인 분석
  3. 코드 수정
  4. 전체 Test Cases 재실행
종료 조건: 전체 PASS + Constraints 위반 없음
```
> 실 로그인/DAG 트리거 검증(브라우저 접속, Airflow `t2_collect` 수동 실행)은 사람이 수행. Codex는 정적 Test Cases까지만 책임진다.

- `isRemember` 체크박스는 절대 건드리지 않는다 — 오직 `isCompany` 클릭만 제거.
