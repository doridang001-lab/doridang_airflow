# 쿠팡이츠 orders 10057 차단 완화 — 로그인 단계 새로고침 연타 제거 ("원큐")

## Task
쿠팡이츠 60매장 배치수집에서 orders 가 10057("과도한 요청 — 일시적 제한")으로 반복
차단된다. 원인은 orders 수집이 아니라 **로그인 단계의 F5 새로고침 연타**다. 로그인
에러 스팬 텍스트가 사실은 `ACCESS_DENIED`/"권한이 존재하지 않습니다"/"과도한 요청" =
쿠팡의 약한 쓰로틀 신호인데, 이를 일반 로그인 에러로 오인해 `chrome.tabs.reload`+
재로그인(MAX_LOGIN=2)을 하고, 로그인 실패는 쓰로틀 플래그 없이 반환되어 쿨다운 없이
6-12초마다 다음 계정 로그인을 연타 → 하드 10057로 악화. **로그인 단계를 "원큐"로
바꿔** 새로고침 연타를 없애고, 쓰로틀 감지 시 즉시 쿨다운+백필로 이월한다.

### 1) 로그인 에러 텍스트 분류 헬퍼 추가 (throttle 헬퍼 근처, ~227행 `isHardThrottlePayload` 위/아래)

목록을 runner 로컬에 둔다.
```js
// 로그인 에러 스팬 텍스트가 쿠팡 쓰로틀(약한 10057 신호)인지 판별
const LOGIN_THROTTLE_KEYWORDS = [
  '10056','10057','ACCESS_DENIED','ACCESS_DENY','ACCESS_DENIDE',
  '과도한 요청','일시적으로 제한','단시간 내 반복 요청','권한이 존재하지 않'
];
function isLoginThrottleText(txt) {
  const s = String(txt || '');
  return LOGIN_THROTTLE_KEYWORDS.some(k => s.includes(k));
}
// 자격증명 오류(재시도해도 안 풀림 — 새로고침 금지)
function isLoginCredError(txt) {
  return /아이디 혹은 비밀번호|비밀번호가 일치하지 않/.test(String(txt || ''));
}
```

### 2) 로그인 루프에서 새로고침 재시도 제거 (runner.js:846-899)
핵심: **로그인 에러 스팬이 뜨면(=서버가 거절 응답함) F5 재시도하지 않는다.** reload+
재제출은 부하만 늘려 "원큐" 위배.
- `result === 'error'` 분기(현재 874-878행의 `continue` → 루프 상단 F5)를 **즉시 `break`**
  로 바꾼다. `loginErrorDetected = true` 와 `lastLoginErrText` 는 그대로 보존.
- 순수 `'timeout'`(에러 스팬 없이 화면만 미렌더)일 때만 기존 1회 reload 재시도 유지 —
  이건 쓰로틀이 아니라 렌더 실패이므로 재로드가 정당.
- 결과적으로 로그인은 **에러 스팬 시 1회 시도**, timeout 시 최대 2회.

### 3) 로그인 실패 반환에 쓰로틀 플래그 부여 → 쿨다운 + 백필 (runner.js:883-898)
`!loginDone` 블록에서 `lastLoginErrText` 로 분기해 반환 객체를 만든다.
- **쓰로틀**(`isLoginThrottleText(lastLoginErrText)` true):
  ```js
  return {status:'warn', note:'로그인 단계 쓰로틀 감지 — 다음 계정/백필',
          ordersOk:false, cmgOk:false, hardThrottle:true};
  ```

  → `needsThrottleCooldown` true → `waitBeforeNextAccount` 가 2/4/8분 escalating 쿨다운
  적용. `markAccountStoresTried`(이미 호출됨)로 blocked=false·ordersOk=false →
  `incompleteStores`/`backfillStores` 포함 → 다음날 백필 큐 자동 이월.
- **자격증명 오류**(`isLoginCredError`): reload/재시도 없이 fail 1회. note 를
  "로그인 실패 — 아이디/비밀번호 확인 필요"로 명확히(쓰로틀 플래그 없음).
- **기타/timeout**: 기존 note 유지, 쓰로틀 플래그 없음.

### runner.js:770-795 (로그인 결과 감지 — lastLoginErrText 세팅)
```js
async function waitForLoginResult(tabId, ms) {
  const deadline = Date.now() + ms;
  while (Date.now() < deadline) {
    if (stopRequested) return 'stop';
    const url = await getUrl(tabId);
    if (RE_ORDERS.test(url)) return 'ok';
    if (RE_LOGIN.test(url)) {
      try {
        const res = await chrome.scripting.executeScript({
          target: { tabId },
          func: () => {
            const s = document.querySelector('span.login-error-text');
            return (s && s.offsetParent !== null && s.textContent.trim().length > 0)
              ? s.textContent.trim() : '';
          }
        });
        const txt = res?.[0]?.result;
        if (txt) { lastLoginErrText = txt; return 'error'; }
      } catch(_) {}
    }
    await sleep(800);
  }
  return 'timeout';
}
```

### runner.js:846-899 (수정 대상 — 로그인 루프 + 실패 반환)
```js
const MAX_LOGIN = 2;
let loginDone = false;
let loginErrorDetected = false;
for (let attempt = 1; attempt <= MAX_LOGIN; attempt++) {
  if (stopRequested) { markAccountStoresTried(acc, targetStores, '중단됨');
    return {status:'fail', note:'중단됨', ordersOk:false, cmgOk:false}; }
  if (attempt > 1) {
    if (RE_ORDERS.test(await getUrl(workTabId))) { loginDone = true; break; }
    log(`[${acc.id}] 로그인 재시도 (${attempt}/${MAX_LOGIN}) — F5...`);
    try { await chrome.tabs.reload(workTabId); } catch(_) {}
    await sleep(2000);
    await waitForLoginReady(workTabId);
  }
  await injectContent(workTabId);
  await sleep(300);
  try {
    await chrome.tabs.sendMessage(workTabId, { type:'AUTO_LOGIN', platform:'coupang',
      id:acc.id, pw:acc.pw, storeName:acc.stores[0]||'', autoClick:true });
  } catch(e) { log(`[${acc.id}] AUTO_LOGIN 오류 (${attempt}/${MAX_LOGIN}): ${e.message}`, 'err'); }

  const result = await waitForLoginResult(workTabId, 30000);
  if (result === 'ok')  { loginDone = true; break; }
  if (result === 'error') {                      // ← 여기: continue 를 break 로
    loginErrorDetected = true;
    log(`[${acc.id}] 로그인 에러 스팬 감지 — 재시도 (${attempt}/${MAX_LOGIN})`,'err');
    continue;  // → F5 at top of loop   ← 제거 대상
  }
  if (RE_ORDERS.test(await getUrl(workTabId))) { loginDone = true; break; }
}

if (!loginDone) {
  if (stopRequested) { markAccountStoresTried(acc, targetStores, '중단됨');
    return {status:'fail', note:'중단됨', ordersOk:false, cmgOk:false}; }
  const curUrl = await getUrl(workTabId);
  // 로그인 에러 스팬은 대부분 연속 로그인 과부하(throttle) — 자격증명은 유효.
  const loginErr = loginErrorDetected && RE_LOGIN.test(curUrl);
  const note = loginErr
    ? `로그인 에러 - 재시도 예정${lastLoginErrText ? ' ('+lastLoginErrText+')' : ''}`
    : (RE_LOGIN.test(curUrl) ? '로그인 실패(Akamai 또는 일시 오류)' : 'orders 페이지 미도달');
  log(`[${acc.id}] ${note}`, 'err');
  markAccountStoresTried(acc, targetStores, note);
  setRow(i, {orders:'warn', note, current:false, promote:true});
  return {status:'fail', note, ordersOk:false, cmgOk:false};   // ← 쓰로틀 플래그 분기 추가
}
```

### runner.js:266-282 (쿨다운 트리거 — hardThrottle 플래그가 여기로 전파됨)
```js
function needsThrottleCooldown(res = {}) {
  return !!(res.blocked || res.hardThrottle || res.ordersTimedOut || res.partialThrottle);
}
async function waitBeforeNextAccount(res) {
  let d;
  if (needsThrottleCooldown(res)) {
    d = THROTTLE_COOLDOWN_STEPS[Math.min(throttleStreak, THROTTLE_COOLDOWN_STEPS.length - 1)];
    throttleStreak++;
    log(`↩ 쿼다운 ${Math.round(d / 60000)}분 대기...`, 'info');
  } else {
    throttleStreak = 0;
    d = rand(DELAY_MIN, DELAY_MAX);
    log(`다음 계정까지 ${Math.round(d/1000)}초 대기...`);
  }
  await sleep(d);
}
```
