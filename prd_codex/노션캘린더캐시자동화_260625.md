# 노션캘린더캐시자동화

## Task
AI 브리핑에서 노션 일정이 누락되는 문제 해결.
근본 원인: `scripts/notion_calendar_cache.py`가 매일 아침 브리핑 전 자동 실행되지 않아 캐시 파일이 없음.
Windows 작업 스케줄러에 캐시 스크립트를 등록하고, 브리핑 모니터링 목록에 태스크를 추가해 매주 상태 확인.

## Project Conventions
- logging: `logger = logging.getLogger(__name__)` — print 절대 금지
- import 경로: `from modules.transform.utility.XXX import ...` (절대경로)
- 파이프라인 파일 위치: `modules/transform/pipelines/strategy/morning_briefing_pipeline.py`
- 스케줄 상수: `from modules.transform.utility.schedule import ...` (하드코딩 금지)
- 경로 상수: `from modules.transform.utility.paths import ...` (하드코딩 금지)

## Files to Create / Modify
- **수정**: `modules/transform/pipelines/strategy/morning_briefing_pipeline.py`
  - `_WINDOWS_SCHEDULED_TASKS` 리스트에 `NotionCalendarCache` 태스크 항목 추가
- **실행 (코드 없음)**: Windows 작업 스케줄러 등록 명령어

## Implementation Steps

### Step 1 — 테스트 (Chrome CDP debugger가 열려 있어야 함)
```powershell
cd C:\airflow
python scripts/notion_calendar_cache.py
# 기대: "wrote config/notion_calendar_cache.json (N events)"
```
Chrome CDP debugger: `127.0.0.1:9223` (notion_calendar_host_chrome.ps1 먼저 실행)

### Step 2 — Windows 작업 스케줄러 등록
브리핑 DAG 실행 시각: 평일 06:50 → 캐시 태스크는 **06:40** 등록

```powershell
schtasks /create `
  /tn "NotionCalendarCache" `
  /tr "python C:\airflow\scripts\notion_calendar_cache.py" `
  /sc WEEKLY `
  /d MON,TUE,WED,THU,FRI `
  /st 06:40 `
  /f
```

등록 확인:
```powershell
schtasks /query /tn "NotionCalendarCache" /fo LIST
```

### Step 3 — 브리핑 모니터링 목록에 추가
`modules/transform/pipelines/strategy/morning_briefing_pipeline.py`의
`_WINDOWS_SCHEDULED_TASKS` 리스트 끝에 아래 항목 추가:

```python
    {
        "label": "노션 캘린더 캐시",
        "task_name": "NotionCalendarCache",
        "duplicate": False,
    },
```

## Test Cases
1. **캐시 스크립트 실행** `python scripts/notion_calendar_cache.py`
   → 기대: `wrote config/notion_calendar_cache.json (N events)` (N≥0)

2. **캐시 파일 내용 확인**
   ```python
   python -c "
   import json; from pathlib import Path
   d = json.loads(Path('config/notion_calendar_cache.json').read_text(encoding='utf-8'))
   print('date:', d['date'], '/ events:', len(d['events']))
   "
   ```
   → 기대: `date: 2026-06-25 / events: N`

3. **작업 스케줄러 등록 확인**
   ```powershell
   schtasks /query /tn "NotionCalendarCache" /fo LIST
   ```
   → 기대: `Status: Ready` 또는 `다음 실행 시간: 06:40:00`

4. **파이프라인 import 확인**
   ```python
   python -c "
   from modules.transform.pipelines.strategy.morning_briefing_pipeline import collect_briefing_data
   print('import ok')
   "
   ```
   → 기대: `import ok`

5. **모니터링 목록 반영 확인** (Step 3 완료 후)
   ```python
   python -c "
   from modules.transform.pipelines.strategy.morning_briefing_pipeline import _WINDOWS_SCHEDULED_TASKS
   names = [t['task_name'] for t in _WINDOWS_SCHEDULED_TASKS]
   assert 'NotionCalendarCache' in names, '등록 안됨'
   print('ok:', names)
   "
   ```
   → 기대: `ok: [..., 'NotionCalendarCache']`

## Verification Loop
```
LOOP until all PASS:
  1. Test Cases 1~5 순서대로 실행
  2. FAIL 항목 → 원인 분석
  3. 코드/명령 수정
  4. 전체 Test Cases 재실행
종료 조건: 전체 PASS
```

## Constraints
- `_WINDOWS_SCHEDULED_TASKS` 기존 항목 순서/내용 변경 금지 (끝에만 추가)
- schtasks 태스크명 `NotionCalendarCache` 그대로 사용 (대소문자 일치)
- `notion_calendar_host_chrome.ps1`로 Chrome CDP debugger가 열려 있어야 캐시 스크립트 성공
- 캐시 파일 경로는 `config/notion_calendar_cache.json` (Windows 호스트 경로), Docker에서는 `/opt/airflow/config/notion_calendar_cache.json`으로 마운트됨
