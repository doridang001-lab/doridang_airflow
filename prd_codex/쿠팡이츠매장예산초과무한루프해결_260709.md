# 쿠팡이츠 매장 예산 초과 무한 루프 해결

## Task

쿠팡이츠 주문 수집 크롬 확장에서, 37페이지짜리 대형 매장(예: 행신점 754007)이 매장당 예산 8분(`ORDERS_STORE_BUDGET_MS = 480000`)을 넘겨 페이지 35 근처에서 항상 중단되고 백필로 이월된다. 백필은 1페이지부터 재시작하므로 같은 지점에서 또 터지고 영원히 완주하지 못한다.

**진전(새 주문 수집)이 있고 throttle 신호가 없을 때만** 예산을 480초씩 최대 2회 연장(매장당 상한 24분)해, 대형 매장이 한 번의 실행에서 완주하도록 만든다. 봇 탐지 리스크를 키우지 않기 위해 페이싱 상수는 **변경하지 않는다**.

## Project Conventions

- 대상은 Python이 아니라 **크롬 확장 JS**다. `logger`/`print` 규칙은 적용되지 않는다.
- 로깅은 전부 `Utils.updateProgressModal({ debug: '...' })` 로 진행 모달에 남긴다. `console.log` 는 기존 코드가 쓰는 곳에만 유지하고 새로 추가하지 않는다.
- 모든 수집 로직은 `Sites['coupangeats']` 객체의 메서드로 존재한다. 새 헬퍼도 이 객체의 메서드로 추가한다 (`,` 로 구분되는 객체 리터럴 프로퍼티).
- 상수는 객체 최상단(파일 8번째 줄 부근, `ORDERS_STORE_BUDGET_MS` 옆)에 모아 둔다.
- 들여쓰기 2칸, 세미콜론 사용, 함수는 `async _name() {}` 스타일. 기존 파일과 동일하게.

### 1. 예산 연장 상수 추가

  ORDERS_STORE_BUDGET_MAX_EXTENSIONS: 2,      // 매장당 최대 24분

### 2. `_tryExtendStoreBudget()` 헬퍼 신설

`_markOrdersTransientError()` (`03_coupangeats.js:101`) 바로 위 또는 아래에 추가한다.

```js
  _tryExtendStoreBudget({ extensions, idCount, lastExtensionIdCount }) {
    if (this._stopFlag) return false;
    if (extensions >= this.ORDERS_STORE_BUDGET_MAX_EXTENSIONS) return false;
    if (idCount <= lastExtensionIdCount) return false;          // 진전 없음 → 연장 거부
    if (this._isOrdersThrottleVisible()) return false;
    if (this._hadRecentOrdersTransientError()) return false;
    return true;
  },
```

**중요**: 이 함수는 `_hadRecentOrdersTransientError()` 를 본다. 기존 예산 초과 분기들은 `this._markOrdersTransientError()` 를 **먼저** 호출하므로, 호출 순서를 반드시 `_tryExtendStoreBudget()` → (거부 시) `_markOrdersTransientError()` 로 유지해야 한다. 순서가 뒤집히면 연장이 절대 승인되지 않는다.

### 5. 페이지 중간 예산 초과도 연장 대상으로

`_collectCurrentPageData` 는 페이지 진입 시점의 `storeDeadlineMs` 를 **값으로** 받아(`03_coupangeats.js:3129`) 주문 항목마다 검사한다(`03_coupangeats.js:2081-2087`). 따라서 페이지 중간에 예산이 터지면 루프 상단의 연장 검사에 도달하지 못하고 `pageDeadlineExceeded` 로 빠져나온다.

### 7. manual 이어수집 경로 버그 수정

`03_coupangeats.js:3811-3823` 에 두 가지 버그가 있다.

- `_resumeOrdersCurrentPage(shopInfo, _resumeDepth, ...)` 가 depth 를 증가시키지 않아 `MAX_MANUAL_RESUME`(10) 가드가 동작하지 않는다 → `_resumeDepth + 1` 로 전달.

## Constraints

- **페이싱 상수를 절대 건드리지 마라.** `_manualOrdersPace`, `_humanizeManualOrdersAfterItem`, `_humanizeManualOrdersAfterPage`, `_randomDelay` 의 숫자를 줄이면 쿠팡 봇 탐지가 재발한다. 이 프로젝트는 과거 Akamai 차단으로 Selenium 을 버리고 확장으로 전환한 이력이 있다.
- `_tryExtendStoreBudget()` 는 반드시 `_markOrdersTransientError()` **이전에** 호출한다. 순서가 뒤집히면 `_hadRecentOrdersTransientError()` 가 항상 true 라 연장이 승인되지 않는다.
- 연장 조건에서 "진전 있음"(`idCount > lastExtensionIdCount`) 체크를 빼지 마라. 이게 유일한 무한 연장 방지 장치다.
- 페이지 중간 연장 시 `lastExtendedPage !== currentPage` 가드를 빼지 마라. 같은 페이지 무한 재수집을 막는다.
- batch 경로에 in-place 이어수집을 넣지 마라. 이번 범위 밖이다 (사용자가 명시적으로 "진전 기반 예산 연장"만 선택).
- `_collectOrdersPages` 의 `pageCount` 는 `continue` 경로에서 중복 증가하지 않아야 한다. 루프 상단 연장은 `pageCount++` 앞이라 안전하고, 페이지 중간 연장은 `pageCount--` 로 되돌린다.
- 폴더 구조·파일명 변경 금지. 두 사본을 항상 동기 상태로 유지한다.

## Do Not Ask — Decide Yourself

- 파일이 이미 존재하면: 덮어쓰기
- import 가 모호하면: Reference Code 의 패턴 따라가기
- 타입 힌트 여부: 기존 파일과 동일하게 (이 파일은 순수 JS, 타입 없음)
- 주석 여부: 최소화 (WHY 만, WHAT 설명 금지)
- 변수명·함수명 스타일: camelCase, private 메서드는 `_` 접두, 기존 파일과 동일하게
- 줄 번호가 안 맞으면: 코드 문자열로 위치를 찾아 적용
