# 쿠팡확장 로그인 재시도 + 타이핑 방식 개선 + 탭 응답없음 방지

## Task
쿠팡이츠 크롬확장(`coupang_extension_build/`) 배치 수집에서 로그인 에러가 반복 발생하고(7/14 로그), 러너/작업 탭이 장시간 배치 중 완전히 응답없어져(로그도 안 찍힘) 수동 새로고침이 필요한 문제가 있다. 세 가지를 고친다: (1) 원인불명 로그인 에러만 F5 1회+재로그인(최대 2회)로 복구, 쓰로틀/봇차단 에러는 기존처럼 즉시 스킵 유지, (2) 쿠팡 ID/PW 입력을 글자별 타이핑처럼 바꿔 봇탐지 리스크를 낮추고, (3) Chrome의 자동 탭 discard가 유력한 "탭 응답없음" 원인이므로 `autoDiscardable:false`로 방지한다.

## Project Conventions
- 이 저장소는 Python/Airflow 프로젝트지만 이번 작업은 순수 크롬 확장(MV3, plain JS) 코드다. Python 컨벤션(logging, paths.py 등) 적용 대상 아님.
- `console.log`/`console.error`는 기존 코드 스타일대로 유지(이 파일들은 이미 print 대신 console 사용 중, 다른 검증 규칙 아님).
- 확장은 **사용자 복사 배포 방식**이다: `C:\airflow\coupang_extension_build\` 가 소스이고, 실제 크롬 확장 폴더(`C:\Users\민준\OneDrive - 주식회사 도리당\Extention\doridang_collector_개발용`)로 사용자가 수동 복사 후 리로드한다. Claude/Codex가 OneDrive 경로를 직접 수정하지 않는다.
- 배민/요기요/땡겨요 로그인 분기(`content/04_auto_login.js`)는 이번 작업 범위 밖 — 건드리지 않는다.
- 기존 로직/변수명/로그 문자열 톤을 최대한 유지하고, 최소 diff로 수정한다.

## Implementation Steps

### 1. `runner.js` — 로그인 에러 시 F5 재시도 조건부 허용
`processAccount` 함수 내 로그인 재시도 for 루프(`for (let attempt = 1; attempt <= MAX_LOGIN; attempt++) { ... }`)를 찾는다. 그 안의 `waitForLoginResult` 결과 처리부:

```js
const result = await waitForLoginResult(workTabId, 30000);
if (result === 'ok')  { loginDone = true; break; }
if (result === 'error') {
  loginErrorDetected = true;
  log(`[${acc.id}] 로그인 에러 스팬 감지 — F5 재시도 없이 중단 (${attempt}/${MAX_LOGIN})`,'err');
  break;
}
```

를 아래처럼 바꾼다:

```js
const result = await waitForLoginResult(workTabId, 30000);
if (result === 'ok')  { loginDone = true; break; }
if (result === 'error') {
  loginErrorDetected = true;
  const throttleErr = isLoginThrottleText(lastLoginErrText);
  if (throttleErr || attempt >= MAX_LOGIN) {
    log(`[${acc.id}] 로그인 에러 스팬 감지 — ${throttleErr ? 'F5 재시도 없이 중단' : '재시도 한도 도달, 중단'} (${attempt}/${MAX_LOGIN})`,'err');
    break;
  }
  log(`[${acc.id}] 로그인 에러 스팬 감지 — F5 재시도 예정 (${attempt}/${MAX_LOGIN})`,'err');
  continue;  // → 루프 상단의 F5 reload 로직(attempt>1 분기)이 다음 attempt에서 실행됨
}
```

- `isLoginThrottleText`는 이미 `runner.js`에 정의되어 있다(`10057`, `ACCESS_DENIED`, `과도한 요청`, `일시적으로 제한`, `권한이 존재하지 않` 등 키워드 매칭). 이 함수는 건드리지 않는다.
- `MAX_LOGIN`은 그대로 `2`로 둔다 (변경 불필요).
- 루프 상단에 이미 있는 `if (attempt > 1) { ... chrome.tabs.reload(workTabId) ... }` 블록이 `continue` 이후 다음 attempt에서 자연스럽게 F5+재로그인을 수행하므로 별도 F5 호출 코드를 추가할 필요 없다.
- 루프 이후의 실패 사유 계산부(`loginThrottle`/`credError`/`loginErr` 판정, `isLoginThrottleText(lastLoginErrText)` 사용하는 부분)는 이미 올바르게 동작하므로 수정하지 않는다.

### 2. `content/04_auto_login.js` — 쿠팡 ID/PW 글자별 타이핑
파일 최상단(함수 `handleAutoLogin` 정의 이전)에 헬퍼 함수를 추가한다:

```js
async function simulateTyping(input, text) {
  input.focus();
  await Utils.sleep(150 + Math.random() * 150);
  input.value = '';
  for (const ch of String(text)) {
    input.value += ch;
    input.dispatchEvent(new KeyboardEvent('keydown', { key: ch, bubbles: true }));
    input.dispatchEvent(new Event('input', { bubbles: true }));
    input.dispatchEvent(new KeyboardEvent('keyup', { key: ch, bubbles: true }));
    await Utils.sleep(60 + Math.random() * 90);
  }
  input.dispatchEvent(new Event('change', { bubbles: true }));
}
```

`handleAutoLogin`의 `platform === 'coupang'` 분기 안, 기존 ID 입력 블록:

```js
idInput.focus();
await Utils.sleep(300);

idInput.value = id;
idInput.dispatchEvent(new Event('input', { bubbles: true }));
idInput.dispatchEvent(new Event('change', { bubbles: true }));
idInput.blur();

console.log('[Auto Login] ✅ ID 입력 완료:', idInput.value);
```

를:

```js
await simulateTyping(idInput, id);
idInput.blur();

console.log('[Auto Login] ✅ ID 입력 완료:', idInput.value);
```

로 교체. PW 입력 블록도 동일한 패턴으로 `await simulateTyping(pwInput, pw); pwInput.blur();`로 교체한다. try/catch 래핑과 로그 문구는 그대로 유지한다.

- `baemin`/`yogiyo`/`ddangyo` 분기는 손대지 않는다.
- `Utils.sleep`은 `content/01_utils.js`에 이미 정의되어 있으므로 재사용한다(새로 만들지 않음).

### 3. `background.js` — 수동 수집 탭 discard 방지 + `runner.js` — 배치 탭 discard 방지
`background.js`의 `chrome.action.onClicked` 리스너:

```js
chrome.action.onClicked.addListener(async (tab) => {
  await showClickFeedback(tab.id);

  try {
    await chrome.tabs.sendMessage(tab.id, { type: 'COLLECT' });
  } catch (e) {
    await chrome.scripting.executeScript({ target: { tabId: tab.id }, files: ['content.js'] });
    await chrome.tabs.sendMessage(tab.id, { type: 'COLLECT' });
  }
});
```

맨 앞에 discard 방지 코드 추가(실패해도 무시):

```js
chrome.action.onClicked.addListener(async (tab) => {
  try { await chrome.tabs.update(tab.id, { autoDiscardable: false }); } catch (_) {}
  await showClickFeedback(tab.id);
  ...
```

`runner.js`에서 작업 탭을 생성하는 3곳(`chrome.tabs.create({url:'about:blank', active:true})` — 대략 1193, 1234, 1349행 부근, 변수명 `bfWorkTab`/`workTab`)에서, 탭 생성 직후 동일하게 discard 방지를 건다. 예:

```js
workTab = await chrome.tabs.create({url:'about:blank', active:true});
try { await chrome.tabs.update(workTab.id, { autoDiscardable: false }); } catch (_) {}
```

추가로, 배치 시작 진입점(사용자가 "▶ 전체 수집" 등을 눌러 배치가 시작되는 함수, `runner.js` 최상단 로직 근처)에서 러너 탭 자기 자신에 대해서도 1회 설정한다:

```js
try {
  const selfTab = await chrome.tabs.getCurrent();
  if (selfTab) await chrome.tabs.update(selfTab.id, { autoDiscardable: false });
} catch (_) {}
```

- `chrome.tabs.getCurrent()`는 확장 페이지(runner.html) 컨텍스트에서만 자기 탭 정보를 반환한다 — 이 특성 그대로 사용.
- 이 변경은 기존 로직을 바꾸지 않는 순수 추가이므로 부작용 리스크가 낮다. `try/catch`로 감싸 실패해도 배치 진행에 영향 없게 한다.
- `manifest.json`은 이미 `"tabs"` 권한을 갖고 있으므로 `chrome.tabs.update`의 `autoDiscardable` 사용에 추가 권한이 필요 없다.

const MAX_LOGIN = 2;
let loginDone = false;
let loginErrorDetected = false;
for (let attempt = 1; attempt <= MAX_LOGIN; attempt++) {
  if (stopRequested) {
    markAccountStoresTried(acc, targetStores, '중단됨');
    return {status:'fail', note:'중단됨', ordersOk:false, cmgOk:false};
  }
  if (attempt > 1) {
    if (RE_ORDERS.test(await getUrl(workTabId))) { loginDone = true; break; }
    log(`[${acc.id}] 로그인 재시도 (${attempt}/${MAX_LOGIN}) — F5...`);
    try { await chrome.tabs.reload(workTabId); } catch(_) {}
    await sleep(2000);
    await waitForLoginReady(workTabId);
  }
  await injectContent(workTabId);
  await sleep(300);
  try {
    await chrome.tabs.sendMessage(workTabId, {
      type:'AUTO_LOGIN', platform:'coupang',
      id:acc.id, pw:acc.pw, storeName:acc.stores[0]||'', autoClick:true
    });
  } catch(e) { log(`[${acc.id}] AUTO_LOGIN 오류 (${attempt}/${MAX_LOGIN}): ${e.message}`, 'err'); }

  const result = await waitForLoginResult(workTabId, 30000);
  if (result === 'ok')  { loginDone = true; break; }
  if (result === 'error') {
    loginErrorDetected = true;
    log(`[${acc.id}] 로그인 에러 스팬 감지 — F5 재시도 없이 중단 (${attempt}/${MAX_LOGIN})`,'err');
    break;
  }
  if (RE_ORDERS.test(await getUrl(workTabId))) { loginDone = true; break; }
}

## Test Cases
1. [정적 검증] 세 파일 모두 문법 오류 없는지: 각 파일을 `node --check <file>` 로 확인 → 기대: 에러 없음 (단, `chrome.*` 전역 미정의 경고는 무시 — `node --check`는 파싱만 검사하므로 참조 오류는 안 뜸).
2. [로그인 분기 로직] `runner.js`에서 `result === 'error'` 블록에 `isLoginThrottleText`, `continue`, `attempt >= MAX_LOGIN` 세 요소가 모두 포함됐는지 grep으로 확인 → 기대: 3개 모두 매치.
3. [타이핑 함수 존재] `content/04_auto_login.js`에 `simulateTyping` 함수 정의와, coupang 분기 내 `simulateTyping(idInput` / `simulateTyping(pwInput` 호출이 각각 1회씩 있는지 grep 확인 → 기대: 함수정의 1개 + 호출 2개.
4. [discard 방지] `background.js`와 `runner.js`에 `autoDiscardable: false`가 각각 최소 1곳(`background.js`), 4곳(`runner.js`: 작업탭 3곳 + 러너 자기탭 1곳) 있는지 grep 확인 → 기대: 총 5곳.
5. [수동 E2E — 코드만으로 불가능한 부분] 실제 크롬에 복사 적용 후 배치 1회 돌려서: (a) 원인불명 로그인 에러가 나면 "F5 재시도 예정" 로그가 찍히고 재로그인 시도가 실제로 일어나는지, (b) 10057류 에러는 여전히 즉시 스킵되는지, (c) 장시간 배치 후 러너 탭이 살아있는지 — 이건 Codex가 자동 실행할 수 없으므로 사용자에게 수동 확인 요청 문구를 결과 보고에 포함할 것.

## Verification Loop
```
LOOP until all PASS:
  1. Test Cases 1~4 순서대로 실행 (grep/node --check)
  2. FAIL 항목 → 원인 분석 (문법 오류 vs 패턴 누락)
  3. 코드 수정
  4. 전체 Test Cases 1~4 재실행
종료 조건: 1~4 전체 PASS + Constraints 위반 없음
  (Test Case 5는 사용자 수동 확인 대상이므로 루프 종료 조건에서 제외, 결과 보고서에만 안내)
```
