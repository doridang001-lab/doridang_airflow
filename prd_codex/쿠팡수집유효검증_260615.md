## Implementation Steps

### 1. CLI 인자 (argparse)
- `--date YYYY-MM-DD` 기본값: `pendulum.now("Asia/Seoul").subtract(days=1).format("YYYY-MM-DD")` (어제, KST)
- `--price-col` 선택 `{total_price, 매출액}`, 기본 `total_price` (ToOrder `price`는 총주문금액 성격 → 기존 검증기와 동일 기준)
- `--tol` 허용 오차 int, 기본 `0`
- `--show-latest` flag: 기준 parquet 최신 가용일의 쿠팡이츠+쿠팡포장 합계 표 출력
- `--csv` flag: 결과를 `ANALYTICS_DB/"unified_sales_validation"/f"coupang_toorder_platform_{date}.csv"` 저장

### 2. 기준 측 `_toorder_coupang_by_store(target_date) -> dict`
- `pq = ANALYTICS_DB / "toorder_daily_store_platform" / "toorder_store_platform_daily.parquet"`
- `df = pd.read_parquet(pq)`
- `sub = df[(df["date"] == target_date) & (df["platform"].isin(["쿠팡이츠", "쿠팡 포장"]))]`
- `sub = sub.assign(_key=normalize_for_join(sub["store"]))`
- `return sub.groupby("_key")["price"].sum().astype(int).to_dict()` (비면 `{}` + 경고 로그)
- parquet의 `date`는 문자열(`'2025-07-31'` 형식). 최신 가용일 = `df["date"].max()` 도 함께 구해 빈 데이터 안내에 사용.

### 3. 수집 측 `_coupang_collected_by_store(target_date, price_col) -> dict`
`DB_Coupang_02_validate._coupang_orders_by_store` 로직 이식:
```python
ym = target_date[:7]
dt = datetime.strptime(target_date, "%Y-%m-%d")
date_patterns = [target_date.replace("-", "."), f"{dt.year}.{dt.month}.{dt.day}", f"{dt.year}.{dt.month:02d}.{dt.day:02d}"]
csv_paths = list(COUPANG_ORDERS_DB.glob(f"brand=*/store=*/ym={ym}/orders_{ym}.csv"))
result = {}
for csv_path in csv_paths:
    store = csv_path.parts[-3][len("store="):]
    df = pd.read_csv(csv_path, dtype=str, encoding="utf-8-sig")
    if df.empty: continue
    mask = df["order_date"].apply(lambda x: any(p in str(x) for p in date_patterns))
    daily = df[mask]
    if daily.empty: continue
    price = pd.to_numeric(daily[price_col].str.replace(",", "", regex=False), errors="coerce").fillna(0)
    cancel = daily["is_cancelled"].astype(str).str.strip().str.upper() == "Y"
    실매출액 = int(price[~cancel].sum()) - int(price[cancel].sum())
    key = normalize_for_join(pd.Series([store])).iloc[0]   # 기준 측과 동일 join 키
    result[key] = result.get(key, 0) + 실매출액
return result
```
- 단일 store 문자열은 `pd.Series([store])`로 감싸 `normalize_for_join` 적용 후 `.iloc[0]`.

### 4. 비교/리포트 `compare(target_date, price_col, tol) -> dict`
- `toorder = _toorder_coupang_by_store(target_date)`, `collected = _coupang_collected_by_store(...)`
- 전체 매장 = `sorted(set(toorder) | set(collected))`
- 매장별 `{store: {"toorder": t, "collected": c, "diff": t-c, "matched": abs(t-c) <= tol, "gap": (t==0) ^ (c==0)}}`
- 총액: ToOrder 합, 수집 합, 차액, 일치 매장수/전체. 불일치·gap 매장만 따로 강조 로그.
- `--csv` 시 매장별 표를 DataFrame으로 저장(utf-8-sig).

### 5. 빈 데이터 / show-latest 처리
- target_date에 기준 또는 수집이 0건이면: `logger.warning("⚠️ %s 기준 데이터 없음 — 최신 가용일: %s", target_date, latest)` 후 비교 스킵.
- `--show-latest`: `latest = df["date"].max()` 로 2번 로직 재실행해 매장별/총액 표 출력.

### 6. `main()` + `if __name__ == "__main__":`
- argparse 파싱 → show-latest 분기 또는 compare 실행 → 결과 콘솔 표 출력.

### modules/transform/utility/store_normalize.py
```python
import pandas as pd, re
_PREFIX_RE = re.compile(r"^도리당\s*"); _SPACE_RE = re.compile(r"\s+")
STORE_NAME_MAP = { "도리당 일산백석점": "도리당 백석점", "도리당 구로디지털단지점": "도리당 구로디지털점", ... }  # Single Source of Truth
def normalize(series): return series.replace(_MAP)
def normalize_for_join(series):
    # 정규화 → '도리당' 접두어·공백 제거. 양쪽에 적용 시 이름 차이 흡수.
    # "도리당 미사점" / "미사점" → 둘 다 "미사점"
    return normalize(series).map(lambda s: _SPACE_RE.sub("", _PREFIX_RE.sub("", str(s)).strip()))
```

## Constraints
- 운영 파이프라인/DAG 수정 금지 — `scripts/` 신규 파일만 추가.
- `logging` 사용(print 금지), 경로는 `paths.py` 상수만 사용(하드코딩 금지).
- 매장명 매칭은 반드시 `normalize_for_join` 사용(STORE_NAME_MAP 준수, 자체 문자열 처리 금지).
- 취소 주문 차감: `실매출액 = 비취소 합 - 취소 합` (기존 검증기와 동일 기준).
- `쿠팡이츠`와 `쿠팡 포장`을 **둘 다** 합산 (포장 누락이 기존 허점 — 반드시 포함).
- 현재 양쪽 데이터가 비어/오래됨 → 날짜 파라미터화 + 빈 데이터 안내 필수, 예외로 죽지 말 것.
