# 건너뜀(warn) 처리 및 1~5차 자동 재시도 반영

## Task

쿠팡이츠 배치에서 `skipped: true`(브랜드 불일치/임시 미수집) 건너뜀이 실제로는 자동 재시도 대상이어야 함에도
현재는 성공(`ordersOk = true`)으로 계산되어 `c.warn + c.fail = 0` 상태가 되고 있다.

`skipped` 건을 `warn` 처리하여 배지와 집계가 경고 상태로 올라가도록 하고,
기존 `MAX_AUTO_RETRY=5` 재시도 로직이 동작하도록 문서화 및 변경 포인트를 정리한다.

목표 동작: 1차 배치 + 5차 재시도(총 6라운드)에서 건너뜀이 계속 `warn`으로 누적된다.

## Scope and exclusions (반드시 지킬 것)

- 수정 대상은 `orders`, `CMG`, `orders 즉시재시도`, `메뉴` 수집의 `skipped` 처리 4군데만.
- `skip.orders / skip.cmg / skip.menu = true` 경로는 변경 금지.
  - 해당 경로는 재시도 라운드에서 이미 완료된 카테고리 건너뛰기 용도.
  - 여기서는 `ok = true`, 배지 `skip` 유지가 맞다.
- 재시도는 **미완료 카테고리만** 대상으로 한다.
  - 예: 마케팅(`CMG`)이 이미 수집 완료이면, 재시도에서 주문서(`orders`)와 상품관리(`menu`)만 수행한다.
- 변수명 변경 없음: `ordersOk`, `cmgOk`, `menuOk`.

## 핵심 확인 포인트

`recordCategoryResults(...)`는 `ordersOk/cmgOk/menuOk`을 통해 `fallbackOk`로 집계되므로, 위 4건 변경이면 자동 재시도 조건(`c.warn + c.fail > 0`)을 만족한다.

## Reference

### runner.js — 재시도 트리거

```js
if (c.warn + c.fail > 0 && !stopRequested) {
  // 5초 카운트다운 후 runRetry() 호출
}
```

### runner.js — 최대 자동 재시도

```js
const MAX_AUTO_RETRY = 5;  // 5회차 추가 재시도 (총 6라운드)

if (stillFailing && !stopRequested) {
  if (retryRound < MAX_AUTO_RETRY) {
    // 5초 카운트다운 → runRetry()
  } else {
    notifyFailures();
  }
}
```

## Test Cases

1. 배치 실행 후 건너뜀 발생 시, 해당 매장 배지가 `warn`으로 표시되는지 확인한다.
2. 배치 종료 시 `c.warn + c.fail > 0` 조건이 충족되어 5초 자동재시도 카운트다운이 시작되는지 확인한다.
3. 자동 재시도 진입 로그에서 `재시도 1/5회차 시작`, 대상 매장 수가 출력되는지 확인한다.
4. 재시도 후 해당 매장이 정상 수집되면 배지가 `ok`로 전환되는지 확인한다.
5. 5차 재시도 후에도 해결되지 않으면 `warn`/`fail` 상태가 유지되고 `notifyFailures()`가 호출되는지 확인한다.
