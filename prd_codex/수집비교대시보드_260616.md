# 수집 비교 대시보드 (Collection Compare Dashboard)

## Task
여러 수집 소스(toorder / unified / baemin_macro, **향후 더 추가됨**)가 같은 매출을 서로 다른 경로로 수집한다. 소스 간 금액이 어긋나면 수집 누락·중복·파싱 오류 신호다. **N개 소스를 날짜×매장×플랫폼 단위로 나란히 놓고 차이를 한눈에 보여** 문제 매장을 찾거나 재수집을 판단하는 PowerBI용 wide parquet **마트**를 만든다. 핵심은 **확장성** — 소스 추가가 어댑터 1개 추가로 끝나야 한다.

> 참고: 요청에 첨부된 DuckDB SQL은 "이렇게 집계해서 비교하겠다"는 **의도 예시**일 뿐. 실제 구현은 **Pandas**로 동일 집계를 재현한다 (DuckDB 미도입, 신규 의존성 없음).

## Implementation Steps

### 1. 소스 어댑터 레지스트리 (확장 핵심)
각 소스 = 정규화 long DataFrame `[date(str YYYY-MM-DD), store(str), platform(str), amt(int)]` 를 반환하는 loader 1개.
```python
from dataclasses import dataclass
from typing import Callable
import pandas as pd

@dataclass
class SourceSpec:
    key: str        # "toorder"
    label: str      # "토더"
    loader: Callable[[], pd.DataFrame]   # → [date, store, platform, amt]

SOURCES = [
    SourceSpec("toorder",      "토더",      load_toorder),
    SourceSpec("unified",      "통합",      load_unified),
    SourceSpec("baemin_macro", "배민매크로", load_baemin_macro),
    # 향후: SourceSpec("coupang_macro", "쿠팡매크로", load_coupang_macro),
]

COMPARE_PLATFORMS = ["배달의민족", "배민1", "배민 포장", "쿠팡이츠", "쿠팡 포장"]
GAP_THRESHOLD = 2.0   # gap_rate(%) 이 값 이상이면 "차이"
```

### 2. 공통 정규화 헬퍼
```python
def _normalize(df, date_col, store_col, platform_col, amt_col) -> pd.DataFrame:
    out = pd.DataFrame()
    out["date"] = pd.to_datetime(df[date_col], errors="coerce").dt.strftime("%Y-%m-%d")
    out["store"] = df[store_col].astype(str).str.strip()
    out["platform"] = df[platform_col].astype(str).str.strip()
    out["amt"] = pd.to_numeric(df[amt_col], errors="coerce").fillna(0)
    out = out.dropna(subset=["date"])
    out = (out.groupby(["date", "store", "platform"], as_index=False)["amt"].sum())
    out["amt"] = out["amt"].round().astype(int)
    return out
```

### 3. 어댑터별 loader
- **load_toorder**: `ANALYTICS_DB / "toorder_daily_store_platform" / "toorder_store_platform_daily.parquet"` 읽기 → 컬럼 `date, store, platform, price` → `_normalize(df, "date", "store", "platform", "price")`. (이미 일별 집계됨)
- **load_unified**: `MART_DB / "unified_sales_grp"` 에서 `glob("unified_sales_*.parquet")` 전부 읽어 concat → 컬럼 `sale_date, store, platform, total_price` → `_normalize(df, "sale_date", "store", "platform", "total_price")`. (read_parquet 시 `columns=` 로 4개만 로드)
- **load_baemin_macro**: `BAEMIN_ORDERS_DB` 에서 `glob("brand=*/store=*/ym=*/orders_*.parquet")` 전부 읽어 concat. 주문행이 메뉴별로 중복되므로 **주문번호 기준 dedup 필수**:
  ```python
  g = df.groupby("주문번호", as_index=False).agg(
      주문시각=("주문시각", "max"),
      store=("store", "max"),          # store 컬럼 없으면 "store_name" 사용
      총결제금액=("총결제금액", "max"),
  )
  g["총결제금액"] = pd.to_numeric(g["총결제금액"].astype(str).str.replace(",", ""), errors="coerce")
  g["platform"] = "배달의민족"           # v1: 전부 배달의민족
  ```
  date = `주문시각` 앞 12자 파싱 (`"2026. 06. 01."` → `"2026-06-01"`). `pd.to_datetime(g["주문시각"], errors="coerce").dt.strftime("%Y-%m-%d")` 로 처리 가능. 이후 `_normalize`(이미 dedup된 g 기준, amt_col="총결제금액").

### 4. build_collection_compare() — union → pivot wide → 지표 → 저장
```python
def build_collection_compare() -> str:
    frames = []
    for spec in SOURCES:
        try:
            f = spec.loader()
            f["source"] = spec.key
            frames.append(f)
        except Exception as exc:
            logger.warning("소스 로드 실패: %s | %s", spec.key, exc)
    long_df = pd.concat(frames, ignore_index=True)
    long_df = long_df[long_df["platform"].isin(COMPARE_PLATFORMS)]

    wide = long_df.pivot_table(
        index=["date", "store", "platform"], columns="source",
        values="amt", aggfunc="sum",
    ).reset_index()

    keys = [s.key for s in SOURCES]
    for k in keys:
        col = f"amt_{k}"
        wide[col] = wide[k].fillna(0).round().astype(int) if k in wide else 0
        wide[f"present_{k}"] = (wide.get(k).notna() if k in wide else False)
    amt_cols = [f"amt_{k}" for k in keys]
    wide["n_present"] = wide[[f"present_{k}" for k in keys]].sum(axis=1).astype(int)
    wide["amt_max"] = wide[amt_cols].max(axis=1)
    wide["amt_min"] = wide[amt_cols].min(axis=1)
    wide["gap"] = wide["amt_max"] - wide["amt_min"]
    wide["gap_rate"] = (wide["gap"] / wide["amt_max"].replace(0, pd.NA) * 100).fillna(0).round(2)

    def _status(r):
        if r["n_present"] < len(keys):
            return "누락"
        return "차이" if r["gap_rate"] >= GAP_THRESHOLD else "정상"
    wide["status"] = wide.apply(_status, axis=1)
    wide["checked_at"] = datetime.now(KST).strftime("%Y-%m-%d %H:%M:%S")

    # drop 원본 source 컬럼(k), amt_* 만 유지
    wide = wide.drop(columns=[k for k in keys if k in wide.columns])
    COLLECTION_COMPARE_PATH.parent.mkdir(parents=True, exist_ok=True)
    wide.to_parquet(COLLECTION_COMPARE_PATH, index=False, engine="pyarrow")
    return f"수집비교 {len(wide)}행 → {COLLECTION_COMPARE_PATH} (차이 {sum(wide['status']=='차이')}, 누락 {sum(wide['status']=='누락')})"
```
- `KST = ZoneInfo("Asia/Seoul")`, 매 실행 전체 재생성(단일 파일, `daily_summary.parquet` 패턴).

## Constraints
- DuckDB 도입 금지 — Pandas만 사용
- 소스 추가가 `SOURCES` 리스트 + loader 함수 1개로 끝나는 구조 유지 (빌더는 `SOURCES`/`keys` 기반으로 동적 동작, 소스 키 하드코딩 금지)
- `COMPARE_PLATFORMS`, `GAP_THRESHOLD` 는 모듈 상단 상수 (매직넘버 금지)
- baemin loader는 **주문번호 dedup 후** 집계 (메뉴별 중복행 → 금액 과대 방지)
- 단일 parquet 전체 재생성 (증분 아님), 소스 outer 합집합 (교집합 아님)
- 한 소스 로드 실패가 전체를 죽이지 않게 try/except + `logger.warning`
- store 컬럼명이 소스마다 다를 수 있음(`store` vs `store_name`) → baemin에서 존재하는 컬럼 사용
- **매장명 정합 caveat**: 소스 간 매장 표기가 다르면 한쪽만 값 있는 행으로 나타남(`present_*`/`status=누락`). v1은 strip만, 향후 `STORE_ALIAS` dict hook은 어댑터 레벨 확장 지점으로 docstring에 명시
