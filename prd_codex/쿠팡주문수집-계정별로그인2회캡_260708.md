### 2) 날짜 스탬프 로그인 카운터 헬퍼 추가
`getTodayStr`(runner.js:91)와 `markStoresBlocked`(runner.js:240) 사이 등 함수 스코프 내 적당한 위치에 추가.
`chrome.storage.local` 키 `ce_login_count = { date, counts:{accId:n} }`. 날짜 바뀌면 자동 리셋(다음날 백필 = 새 예산 2회). 기존 `ce_backfill_queue` 저장 패턴과 동일.
```js
  async function getLoginCount(accId) {
    const { ce_login_count } = await chrome.storage.local.get(['ce_login_count']);
    if (!ce_login_count || ce_login_count.date !== getTodayStr()) return 0;
    return (ce_login_count.counts && ce_login_count.counts[accId]) || 0;
  }
  async function bumpLoginCount(accId) {
    const today = getTodayStr();
    const { ce_login_count } = await chrome.storage.local.get(['ce_login_count']);
    const rec = (ce_login_count && ce_login_count.date === today) ? ce_login_count : { date: today, counts: {} };
    rec.counts[accId] = (rec.counts[accId] || 0) + 1;
    await chrome.storage.local.set({ ce_login_count: rec });
  }
```

## Test Cases
확장 로직이라 실제 쿠팡 재현이 어려우므로 **storage 카운터 추적 + 경로 검증** 중심.

1. [문법] chrome://extensions 에서 확장 재로드 → 에러 배지 없음.
2. [카운트 소비] 배치 1회 실행 후 devtools 콘솔 `chrome.storage.local.get('ce_login_count')` → 대상 계정 count=1. 재시도 1라운드 후 → count=2.
3. [3번째 스킵] 같은 계정이 3번째 로그인에 진입하면 로그에 **"로그인 2회 초과 — 백필 이월"** 출력 + 해당 계정이 `incompleteStores`에서 빠지고(당일 재시도 예약 안 뜸) `chrome.storage.local.get('ce_backfill_queue')`의 `failedStores`에 포함.
4. [세션재사용 제외] 이미 orders 도달(로그인 생략, `r.key==='orders'`)한 계정은 count 증가 안 하는지 대조.
5. [날짜 리셋] 콘솔에서 `chrome.storage.local.set({ce_login_count:{date:'20260707',counts:{someId:2}}})` 후 해당 계정 로그인 → count가 1로 리셋되는지(어제 날짜라 무시) 확인.
