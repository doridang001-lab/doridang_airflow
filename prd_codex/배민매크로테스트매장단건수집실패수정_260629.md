# 배민 매크로 — 테스트 매장 단건 수집 실패 수정

## Task
배민 매크로 DAG(`DB_Beamin_Macro_Dags.py`)에서 테스트 매장 한 곳만 지정하면 수집 0건인데 에러도 안 나는("실행이 안돼") 문제를 고친다. 원인은 수집 본체 `DB_Beamin_combined.py`의 매장명 필터링이 너무 엄격해서, 드롭다운 파싱 매장명과 요청 매장명이 안 맞으면 조용히 스킵되기 때문이다. 매칭을 견고하게 만들고, 매칭 실패가 조용히 성공처럼 보이지 않게 한다.

## Root Cause (코드 추적 결과)
1. `load_accounts(target_stores=["테스트점"], exact=True)` → `store_name="테스트점"` 계정 1건.
2. `collect_now_and_woori()`가 드롭다운 매장 목록을 파싱 후 `_filter_store_list_for_request(store_list, "테스트점")`로 거른다.
3. 필터가 엄격한 `==` 비교(`:761-771`) — 정규화/공백/접미사 처리 없음.
4. 드롭다운 파싱도 취약(`_parse_store_option`, `:734-748`): 매장명이 `점/지점/분점/직영점`으로 끝나야만 지점명 추출. 접미사 없으면 `store = text[:20]`(브랜드 포함 전체)로 폴백 → 절대 안 맞음.
5. 필터가 비면 `if not store_list: continue`(`:835-836`) — 조용히 건너뜀. 계정 1건 스킵 시 `success=0, fail=0, total=0` → 요약 `"성공 0/0 계정"`, `failed` 전부 비어 있음, 예외 없음.
6. `retry_failed`는 실패 목록이 비어 "재시도 없음" 종료 → 수집 0건이 알림상 정상처럼 보임.

(참고) Retry DAG가 되는 이유: `retry_once_failed`는 이미 파싱된 `store_info`를 그대로 순회 → 매장명 재필터를 안 해서 우회됨. 메인 경로를 이만큼 견고하게 만드는 게 목표.

## Implementation Steps

### 1. `_parse_store_option` 폴백 개선 (`:734-748`)
접미사(점/지점/분점/직영점) 매칭 실패 시 `text[:20]` 대신 **브랜드 접두어를 제거한 나머지**를 지점명으로 사용.
- 현재:
  ```python
  matches = re.findall(r"[가-힣]+(?:점|지점|분점|직영점)", text)
  store = matches[-1] if matches else text[:20]
  ```
- 변경:
  ```python
  matches = re.findall(r"[가-힣]+(?:점|지점|분점|직영점)", text)
  if matches:
      store = matches[-1]
  else:
      # 접미사 없는 매장(테스트 매장 등): 브랜드 접두어만 제거한 나머지를 지점명으로
      store = re.sub(rf"^{re.escape(brand)}\s*", "", text).strip() or text[:20]
  ```

### 2. `_filter_store_list_for_request` 견고한 매칭 (`:761-771`)
엄격 `==` → 정규화 기반 다단계 매칭으로 교체. `store_normalize.strip_brand` 재사용.
- 파일 상단 import에 추가:
  ```python
  from modules.transform.utility.store_normalize import strip_brand
  ```
- 매칭 헬퍼(접미사 제거 정규화) 추가 + 필터 교체:
  ```python
  _BRANCH_SUFFIX_RE = re.compile(r"(?:점|지점|분점|직영점)$")

  def _normalize_store_token(value: str) -> str:
      """비교용 정규화: 브랜드 접두어 제거 → 공백 제거 → 말단 지점 접미사 제거."""
      text = strip_brand(pd.Series([str(value or "")])).iloc[0]
      text = re.sub(r"\s+", "", str(text)).strip()
      return _BRANCH_SUFFIX_RE.sub("", text)
  ```
  ※ `store_normalize`가 pandas Series 기반이므로 `import pandas as pd` 필요(파일에 없으면 추가).
  - 단, pandas 의존을 피하고 싶으면 `_BRAND_PREFIX_RE = re.compile(r"^(도리당|나홀로)\s+")` 를 직접 두고 문자열로 처리해도 됨. KNOWN_BRANDS와 일관되게.
- 필터 본체:
  ```python
  def _filter_store_list_for_request(store_list, requested_store_name):
      _, requested_store = _split_requested_store_name(requested_store_name)
      if not requested_store:
          return list(store_list)

      req_norm = _normalize_store_token(requested_store)

      # 1) 접미사 비민감 정확 매칭
      exact = [s for s in store_list if _normalize_store_token(s.get("store")) == req_norm]
      if exact:
          return exact
      # 2) 부분 포함(양방향) 최후 폴백
      partial = [
          s for s in store_list
          if req_norm and (
              req_norm in _normalize_store_token(s.get("store"))
              or _normalize_store_token(s.get("store")) in req_norm
          )
      ]
      return partial
  ```
  순서 보장 필수: **접미사 정확 매칭 우선, 부분 포함은 최후 폴백** ("역삼점" vs "동역삼점" 오매칭 방지).

### 3. 조용한 스킵 제거 — 가시성 확보 (`:835-836`)
`requested_store_name`이 지정됐는데 필터 결과가 빈 경우, 가용 옵션을 error로 찍고 실패로 기록.
- 현재:
  ```python
  if not store_list:
      continue
  ```
- 변경:
  ```python
  if not store_list:
      if requested_store_name:
          available = [f"{p['brand']} {p['store']}" for p in (
              _parse_store_option(o) for o in raw_options
          ) if p]
          logger.error(
              "요청 매장 매칭 실패 [%s]: 요청=%r / 가용 옵션=%s",
              account_id, requested_store_name, available,
          )
          fail += 1
          failed_accounts.append(account)
          metrics["failed_accounts"].append(account_id)
      continue
  ```
  ※ `raw_options`는 매칭 성공 케이스에선 도달하지 않으므로(1·2로 해결됨) 이 분기는 진짜 오타/미존재 매장 방어용. `raw_options`가 try 블록 안 지역변수라 finally 이후 스코프에서 참조 가능한지 확인하고, 안 되면 `store_list` 계산 전에 `raw_options`를 함수 상위 스코프 변수로 빼둔다.

### modules/transform/pipelines/db/DB_Beamin_combined.py (현재 핵심부)
```python
import logging, random, re, time
from modules.extract.croling_beamin import get_store_options, wait_for_page, ...
from modules.transform.pipelines.db.beamin_stability import resolve_stability_profile
logger = logging.getLogger(__name__)
KNOWN_BRANDS = ["나홀로", "도리당"]

def _parse_store_option(opt: dict) -> dict | None:
    text = opt.get("text", "")
    brand = next((b for b in KNOWN_BRANDS if re.search(rf"(?<![가-힣]){re.escape(b)}", text)), None)
    if not brand:
        return None
    matches = re.findall(r"[가-힣]+(?:점|지점|분점|직영점)", text)
    store = matches[-1] if matches else text[:20]
    return {"store_id": str(opt["store_id"]), "brand": brand, "store": store}

def _split_requested_store_name(requested_store_name: str) -> tuple[str | None, str]:
    value = str(requested_store_name or "").strip()
    for brand in KNOWN_BRANDS:
        prefix = f"{brand} "
        if value.startswith(prefix):
            return brand, value[len(prefix):].strip()
    return None, value

def _filter_store_list_for_request(store_list, requested_store_name):
    _, requested_store = _split_requested_store_name(requested_store_name)
    if not requested_store:
        return list(store_list)
    return [s for s in store_list if requested_store == str(s.get("store") or "").strip()]

# collect_now_and_woori 내부 (요청 매장 필터 + 스킵 지점)
if requested_store_name:
    store_list = _filter_store_list_for_request(store_list, requested_store_name)

if not store_list:
    continue
```

### modules/transform/utility/store_normalize.py (재사용 유틸)
```python
_BRAND_PREFIX_RE = re.compile(r"^(도리당|나홀로)\s+")

def strip_brand(series: pd.Series) -> pd.Series:
    """매장명에서 브랜드 접두어('도리당 ', '나홀로 ') 제거 → 지점명만 반환."""
    return series.map(lambda s: _BRAND_PREFIX_RE.sub("", str(s)).strip())
# 예: "도리당 역삼점" → "역삼점"
```

## Test Cases
1. [파싱 폴백] `_parse_store_option({"store_id":"1","text":"도리당 테스트"})` → 기대: `store == "테스트"` (기존 "도리당 테스트" 아님)
2. [접미사 정상] `_parse_store_option({"store_id":"2","text":"도리당 역삼점"})` → 기대: `store == "역삼점"`
3. [필터 접미사 비민감] `_filter_store_list_for_request([{"store_id":"1","brand":"도리당","store":"테스트"}], "테스트점")` → 기대: 1건 반환 (빈 리스트 아님)
4. [필터 정확 우선] `_filter_store_list_for_request([{...,"store":"역삼점"},{...,"store":"동역삼점"}], "도리당 역삼점")` → 기대: "역삼점"만 반환 ("동역삼점" 미포함)
5. [import 무결성] `python -c "import modules.transform.pipelines.db.DB_Beamin_combined"` → 기대: ImportError/SyntaxError 없음
6. [미존재 매장 방어] 존재하지 않는 매장명으로 필터 → 빈 리스트 + (collect 경로에서) `logger.error`에 가용 옵션 출력 + fail 카운트(요약이 "성공 0/0" 아님). 실제 DAG 트리거 시 확인.

## Verification Loop
구현 완료 후 아래 루프를 **모든 Test Cases PASS까지** 반복한다.
```
LOOP until all PASS:
  1. Test Cases 1~5 를 작은 스크립트/`python -c`로 실행
  2. FAIL 항목 → 원인 분석
  3. 코드 수정
  4. 전체 Test Cases 재실행
종료 조건: 전체 PASS + Constraints 위반 없음
```
실수집 권장: 테스트 매장 1건으로 메인 DAG 트리거 `conf={"stores":["<테스트매장명>"]}` → `collect_all` 로그 `수집 매장 순서: [...]`에 테스트 매장이 찍히고 now/우가클/주문/광고 단계가 실제 실행되는지 확인.

## Constraints
- 수정 파일은 `DB_Beamin_combined.py` 하나. DAG 파일·다른 파이프라인 건드리지 말 것.
- 부분 포함(substring) 매칭은 **최후 폴백**으로만. 접미사 정확 매칭이 항상 먼저 시도되어야 함 (포함관계 오매칭 방지).
- `store_normalize`(매장명 정규화 SSOT) 재사용. 새 정규화 로직을 따로 만들지 말 것.
- print 금지, logger 사용. 기존 로그 메시지 톤(한국어) 유지.
- 다건(전체) 수집 경로도 같은 필터를 쓰므로, 정규화가 관대해져도 기존 매장들이 여전히 정확 매칭되는지 회귀 확인.
