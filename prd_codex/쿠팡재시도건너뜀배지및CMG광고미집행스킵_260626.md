## Implementation Steps

1. **건너뜀 배지 회색화** — `runner.js`의 `BADGES` 객체에서 `skip` 클래스를 `b-ok`(초록)에서 `b-wait`(회색)으로 변경.
   ```js
   // 변경 전
   fail  : ['b-fail', '실패'],  skip:['b-ok','건너뜀']
   // 변경 후
   fail  : ['b-fail', '실패'],  skip:['b-wait','건너뜀']
   ```
   `b-wait`는 `runner.html`에 `background:#2b313c; color:#9aa4b2;`(회색)로 이미 정의됨. 신규 CSS 불필요.

2. **CMG 광고미집행 → genuine skip 처리** — `processAccount`의 CMG 수집 루프(`MAX_CMG=3` for문) **종료 직후**, 아직 `cmgOk`가 false이고 마지막 페이로드가 `error:true`(= 매장목록 없음 / 광고 미집행)면, 실패가 아니라 "정상 건너뜀"으로 승격한다. 핵심: `cmgOk = true`로 만들어 `incompleteStores()`에서 빠지게 하고, 배지는 `skip`(회색), 비고는 `광고 미집행(건너뜀)`.

   대상 위치: `recordCategoryResults('cmg', acc, targetStores, lastCp, cmgOk, ...)` 호출 **직전**.
   ```js
   // CMG MAX_CMG 루프 종료 후, recordCategoryResults 호출 전에 삽입
   if (!cmgOk && !stopRequested && lastCp && lastCp.error && !lastCp.timeout) {
     // 계정내 3회 F5 재시도까지 했는데도 매장목록 없음 → 광고 미집행으로 확정 → 정상 건너뜀
     cmgOk = true;
     log(`[${acc.id}] CMG 광고 미집행 확정 — 건너뜀 처리(재시도 제외)`, 'info');
     setRow(i, {cmg:'skip', note:'광고 미집행'});
   }
   recordCategoryResults('cmg', acc, targetStores, lastCp, cmgOk, lastCp?.error ? 'CMG 광고 미집행(건너뜀)' : '');
   ```
   - 주의: `lastCp.timeout`(일시적 타임아웃)은 제외 — 진짜 광고미집행(`error:true && !timeout`)만 승격. 타임아웃은 기존대로 재시도 대상 유지.
   - `recordCategoryResults`는 `fallbackOk` 인자에 승격된 `cmgOk(true)`가 전달되므로 storeStatusRows의 cmgOk가 true로 기록 → `incompleteStores()`에서 제외됨.

3. **회귀 확인** — 위 변경이 정상 CMG 수집/타임아웃 재시도/orders·menu 흐름에 영향 없는지 확인. `error:true && !timeout` 조건이라 광고 집행 매장(정상 완료)·일시 타임아웃 매장은 그대로 동작.

## Test Cases
1. [배지 색] `chrome://extensions`에서 확장 재로드 → runner.html 실행 → CMG/orders 건너뜀 발생 매장의 배지가 **회색(b-wait)** 인지 육안 확인. 기대: 초록 아님.
2. [광고미집행 단독] CMG 광고 미집행 매장 1곳만 선택해 배치 실행 → 해당 매장 CMG가 3회 F5 후 `skip`(회색, 비고 "광고 미집행")으로 종료되고 **cmgOk=true** 인지 확인. 기대: 재시도 대상에서 빠짐.
3. [재시도 집합 축소] 광고미집행 매장 포함해 전체 배치 → 다운로드되는 `coupang_log_YYYYMMDD.md`에서 `재시도 N/5회차 시작 — X개 매장` 의 X가 회차마다 줄고, 광고미집행 매장이 재시도 로그에 더는 안 나오는지 확인. 기대: 헛도는 재시도 제거.
4. [회귀-정상CMG] 광고 집행 정상 매장 → CMG `완료`(b-ok) 그대로. 기대: 영향 없음.
5. [회귀-타임아웃] CMG 일시 타임아웃 매장 → 기존대로 `fail`/재시도 대상 유지(skip 승격 안 됨). 기대: `error:true && !timeout` 조건 덕분에 타임아웃은 제외.

## Constraints
- `content/03_coupangeats.js`(수집 엔진)는 수정 금지 — runner.js 오케스트레이션만 변경.
- skip 승격 조건은 반드시 `lastCp.error && !lastCp.timeout && !lastCp.stopped` — 일시 타임아웃/사용자 중단을 광고미집행으로 오판하지 말 것.
- `BADGES.wait`(대기)와 `BADGES.skip`(건너뜀)이 같은 `b-wait` 클래스를 공유하지만 텍스트가 달라 무방. CSS 신규 추가 금지.
- 기존 `not_target_brand` genuine skip 로직(브랜드 아님)과 충돌 없게 — 그 분기는 이미 위쪽 `if (lastCp.skipped)`에서 처리되므로 Step 2는 `else` 경로 종료 후에만 동작.
