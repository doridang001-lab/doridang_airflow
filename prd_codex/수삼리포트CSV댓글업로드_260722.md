# 수삼 리포트: XLSX 첨부 → CSV 저장 + 댓글 본문 표 텍스트 업로드

## Task
`Strategy_SusamReport_01_FlowUpload_Dags.py`가 전일 전체 상품 매출을 XLSX로 만들어 Flow 게시글 댓글에 **파일 첨부**로 올리던 것을, (1) **CSV 저장**, (2) `item_name like '%수삼%'` 항목만 필터, (3) 파일 첨부 대신 **댓글 본문에 표 텍스트**로 올리도록 변경한다. 로직은 전부 `scripts/analysis/susam_report.py`에 있고 DAG는 `main()`을 CLI로 호출하는 오케스트레이션만 담당한다.

댓글 본문 포맷(확정):
```
order_date | store | item_name | qty | 총매출액
2026-07-20 | 강동점 | [국물 맛집] 도리당 닭도리탕 | 17 | 345468
```

## Implementation Steps

### 1. 리포트 생성 (기존 `generate_xlsx` 대체)
`generate_xlsx(date_str) -> Path` 를 아래 3개 함수로 분리한다.

- `build_report_frame(date_str) -> pd.DataFrame`
  - 기존 `generate_xlsx`의 parquet 로드 → `REQUIRED_COLUMNS` 검증 → `order_date` 파생 → 날짜 필터 → store/item_name 정규화("미지정") → qty/total_price `_normalize_numeric` → groupby(order_date, store, item_name) 집계 → 정렬 로직을 **그대로 재사용**.
  - **추가 필터**(정규화 후 적용): `working = working.loc[working["item_name"].str.contains("수삼", na=False)].copy()`
  - 반환 컬럼 순서: `REPORT_COLUMNS = ("order_date", "store", "item_name", "qty", "총매출액")` 유지.
- `save_csv(report: pd.DataFrame, date_str: str) -> Path`
  - `OUTPUT_DIR.mkdir(parents=True, exist_ok=True)`
  - `output_path = OUTPUT_DIR / f"Susam_{date_str}.csv"`
  - `report.to_csv(output_path, index=False, encoding="utf-8-sig")` (Excel 한글 호환 BOM)
  - **openpyxl import/의존 제거**, `pd.ExcelWriter` 제거.
- `format_comment(report: pd.DataFrame, date_str: str) -> str`
  - 비어있지 않으면: 헤더 `" | ".join(REPORT_COLUMNS)` + 각 행 `" | ".join(str(v) for v in row)` 을 `"\n"` 으로 결합.
  - **빈 결과(해당일 수삼 판매 없음)**: `f"{date_str} 수삼 판매 내역 없음"` 반환. CSV는 헤더만 저장됨.

### 2. 업로드 (파일 첨부 → 댓글 텍스트)
- **삭제**: `_drop_file_to_comment`, `_attachment_is_staged` 및 관련 base64/MIME 로직.
- `_copy_to_browser_clipboard(driver, value)` **추가**: Airflow 컨테이너와 원격 Windows Chrome의 시스템 클립보드가 다를 수 있으므로 `pyperclip`은 사용하지 않는다. 현재 Flow 페이지의 `navigator.clipboard.writeText(value)`를 `execute_async_script`로 실행해 **브라우저 측 클립보드**에 댓글 본문을 기록하고, 거부/실패 시 즉시 예외를 발생시킨다.
- `_set_comment_text(driver, element, value)` **수정**: 멀티라인 댓글은 직접 `send_keys(value)` 또는 JS 값 할당으로 넣지 않는다. 입력란을 focus한 뒤 기존 내용을 `Ctrl+A`/Backspace로 비우고, `_copy_to_browser_clipboard` 호출 후 **실제 `Ctrl+V` 1회**로 붙여넣는다. Enter는 이 함수에서 보내지 않는다.
- `_comment_input_text(driver, element) -> str` **추가**: textarea/input은 `value`, contenteditable은 `innerText`(없으면 `textContent`)를 읽고, CRLF/CR을 LF로 정규화해 반환한다.
- `_wait_for_comment_text(driver, element, expected, timeout=30)` **추가**: 붙여넣기 직후 고정 `sleep`을 쓰지 않고, `_comment_input_text(...) == expected`가 될 때까지 최대 30초 기다린다. 대기 시작/완료/타임아웃에는 원문 대신 기대/실제 글자 수만 기록한다. 타임아웃이면 예외를 발생시켜 Enter와 업로드 완료 상태 저장을 막는다.
- `_matching_comment_count(driver, date_label)` — **filename 인자 제거**, `date_label`(=order_date, 새 댓글 모든 행에 존재하므로 마커로 적합) 하나만으로 매칭.
- `_upload_with_driver(driver, comment_text, date_label, post_url)` — `xlsx_path` → `comment_text`:
  - post_url 이동 → `_find_comment_input` → 로그인 세션 확인(`login`/`signin` in current_url)
  - `before = _matching_comment_count(driver, date_label)`
  - `_set_comment_text(driver, comment_input, comment_text)`
  - `_wait_for_comment_text(driver, comment_input, comment_text, timeout=30)`
  - `_submit_comment_with_enter(driver, comment_input)`
  - `wait.until(lambda d: _matching_comment_count(d, date_label) > before)`
- `upload_to_flow(comment_text, date_label, post_url, user_data_dir, profile, debugger_address=None)` / `upload_many_to_flow(reports, ...)` — 시그니처 `xlsx_path` → `comment_text`, **파일 존재 검증 제거**. Flow URL 검증(`_validate_flow_url`)·드라이버 생성·quit·재시도 로직은 유지. `reports` 항목은 `(comment_text, date_label)`.

### 3. `main()` 파이프라인
- 각 날짜 처리: `report = build_report_frame(date_str)` → `save_csv(report, date_str)` (CSV 항상 저장) → `comment_text = format_comment(report, date_str)`.
- `pending_reports` 항목을 `(comment_text, date_str)` 로 구성, 단건/다건 업로드에 `comment_text` 전달.
- skip/force/state(`flow_susam_uploaded.json`, `load_uploaded_dates`/`save_uploaded_dates`) 로직 그대로.
- `build_parser` 헬프 문구(`--no-upload`="CSV만 생성" 등) 갱신, description "XLSX"→"CSV/댓글".

### 4. DAG 문구
- `dags/strategy/Strategy_SusamReport_01_FlowUpload_Dags.py` 모듈 docstring 첫 줄·`DAG(description=...)`의 "XLSX" → "수삼 CSV/댓글 표 텍스트". 태스크/센서/인자 전달 변경 없음.

## Test Cases
1. **수삼 필터 + 집계** — `build_report_frame`이 `item_name` 에 "수삼" 포함 행만 집계, "일반 백숙" 등 제외. `python -m pytest tests/test_susam_report.py -q` → 기대: 신규/수정 테스트 PASS
2. **CSV 저장** — `save_csv` → `OUTPUT_DIR/Susam_{date}.csv` 생성, `pd.read_csv(path, encoding="utf-8-sig").columns == ["order_date","store","item_name","qty","총매출액"]`
3. **빈 결과** — 수삼 없는 날: `format_comment` → `"{date} 수삼 판매 내역 없음"`, CSV는 헤더만
4. **업로드 텍스트 전달 순서** — `_upload_with_driver`가 `comment_text`를 브라우저 클립보드에 기록하고 `Ctrl+V`한 뒤, `_wait_for_comment_text` 성공 후에만 `_submit_comment_with_enter`를 호출. `_drop_file_to_comment` 호출 없음 (해당 테스트에서 monkeypatch 제거)
5. **대용량 붙여넣기 지연** — 수천 자 멀티라인 본문이 여러 차례에 걸쳐 입력란에 반영되는 fake element/driver로 검사. 전체 문자열 일치 전에는 Enter 미호출, 일치 직후 Enter 1회 호출
6. **붙여넣기 타임아웃** — 30초 내 전체 문자열이 일치하지 않으면 예외, Enter 미호출, 해당 날짜 업로드 상태 미저장
7. **입력 요소별 판정** — textarea/input의 `value`와 contenteditable의 `innerText`/`textContent` 모두 개행 정규화 후 원문과 정확히 비교
8. **모듈 import** — `python -c "from scripts.analysis import susam_report; print('ok')"` → 기대: ImportError/openpyxl 참조 없음
9. **DAG import** — `python -c "from dags.strategy.Strategy_SusamReport_01_FlowUpload_Dags import dag"` → 기대: ImportError 없음
10. **오프라인 생성**(parquet 존재 시) — `python scripts/analysis/susam_report.py --date 2026-07-20 --no-upload` → `MART_DB/Flow/Susam_2026-07-20.csv` 에 수삼 항목만

## Verification Loop
```
LOOP until all PASS:
  1. Test Cases 1~7 순서대로 실행
  2. FAIL 항목 → 원인 분석
  3. 코드 수정
  4. 전체 Test Cases 재실행
종료 조건: 전체 PASS + Constraints 위반 없음
```
※ Flow 실제 댓글 등록(첨부 없이 개행 표 텍스트)은 라이브 Chrome 디버거 세션이 필요해 자동 테스트 불가 — 코드/단위테스트까지만 완료하고, 최초 1회 육안 검증에서 긴 본문이 완전히 나타난 뒤 Enter가 전송되는지 확인한다.

## Constraints
- `main()` CLI 인자 계약 변경 금지 (DAG가 `--date/--start/--end/--post-url/--debugger-address/--force` 전달)
- print 금지 → `logger` 사용
- 경로/스케줄 하드코딩 금지 → `paths.py`/`schedule.py` 상수 사용 (`OUTPUT_DIR = MART_DB / "Flow"` 유지)
- 파일 첨부(drag&drop) 관련 코드 완전 제거 — 댓글은 **본문 텍스트만**
- 댓글 입력은 브라우저 측 클립보드 기록 → 실제 `Ctrl+V` → 전체 문자열 일치 대기(최대 30초) → Enter 순서를 지킬 것
- 고정 `sleep`으로 붙여넣기 완료를 가정하지 말고, 타임아웃 시 Enter·업로드 완료 상태 저장을 수행하지 말 것
- 긴 댓글 원문은 로그에 남기지 않고 기대/실제 글자 수만 기록할 것
- CSV 인코딩 `utf-8-sig` (Excel 한글 깨짐 방지)
- state 파일(`flow_susam_uploaded.json`) skip/force 로직·정렬/집계 로직 보존
