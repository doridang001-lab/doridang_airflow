# 쿠팡 로그인 스로틀 캐스케이드 차단기

## Task
쿠팡 수집기(크롬 확장) `runner.js`는 쿠팡이 IP/세션 단위로 로그인을 막으면(`권한이 존재하지 않습니다` = throttle) 막힌 계정을 백필로 넘기고 10분만 기다린 뒤 다음 계정을 계속 로그인 시도한다. 스로틀은 IP 전체에 걸려 있어 모든 계정이 연쇄로 막히고, 2026-07-21 로그에서는 4시간 반 동안 24개 계정을 전량 실패시켰다. **연속 로그인 스로틀이 임계치(3회)에 도달하면 즉시 배치를 중단하고 남은 계정을 백필로 이관, 기존 2~3시간 후 자동재시도에 위임하는 회로차단기(circuit breaker)를 추가한다.**

## Project Conventions
- 대상은 **크롬 확장 JavaScript** (`runner.js`, IIFE 내부 모듈 패턴). Python/airflow 아님.
- 로깅: 기존 `log(msg, level)` 함수 사용. level은 `'info' | 'err' | 'ok'`. `console.log` 신규 추가 금지.
- 기존 스로틀 처리 패턴을 미러링한다: `hardThrottleStopped`(10057 하드 스로틀 차단 플래그)와 `throttleStreak`.
- **금지**: `hardThrottleStopped`를 로그인 스로틀에 재사용하지 말 것. 그 플래그는 finishBatch에서 자동재시도까지 비활성화한다(`!hardThrottleStopped` 게이트, `runner.js:1431`). 로그인 스로틀은 자동재시도를 **살려둬야** 하므로 별도 플래그를 쓴다.
- 변수/함수명: camelCase, 기존 파일과 동일 스타일.
- 주석: 최소화(WHY만).

### 3. 후속 단계 단락(short-circuit)
백필 단계 종료 후 정규 배치 진입 전에 `loginThrottleAborted`면 즉시 종료한다.
개발본 `runner.js:1290`:
```js
      if (hardThrottleStopped) {
        finishBatch();
        return;
      }
```
바로 위 또는 아래에 추가:
```js
      if (loginThrottleAborted) {
        finishBatch();
        return;
      }
```

### 4. 상태 리셋
`throttleStreak = 0;`을 리셋하는 모든 지점에 두 신규 플래그 리셋을 나란히 추가한다.
개발본 위치: `runBatch` line 1233, `runCustomDateOrders` line 1365, `runRetry` line 1459.
각 지점의 `throttleStreak = 0;` 다음 줄에:
```js
    loginThrottleStreak = 0;
    loginThrottleAborted = false;
```

### 5. finishBatch는 수정하지 않는다
`loginThrottleAborted`는 `hardThrottleStopped`와 별개이므로, finishBatch의 기존 경로
`if (incompleteStores().length > 0 && !stopRequested && !hardThrottleStopped) scheduleAutoRetry(...)` (line 1431)를 자연스럽게 타서 남은 계정이 백필 큐로 저장되고(`ce_backfill_queue`, line 1441) 2~3시간 후 자동재시도된다. 조기 `break`로 미처리된 계정의 매장은 배치 시작 시 `resetStoreStatus`로 이미 pending 상태라 `incompleteStores()`/`backfillStores()`에 그대로 포함된다.

### 6. 빌드본 동기화
개발본 편집 완료 후 `C:\airflow\coupang_extension_build\runner.js`에 동일 편집을 앵커 텍스트로 찾아 적용한다. 빌드본 참고 줄번호: 상수 line 35, `throttleStreak` line 49, 리셋 line 1213(runBatch)·1387(runRetry 등), 백필/정규/재시도 루프의 `로그인 쓰로틀 감지` 로그(1261·1320·1434). 텍스트 앵커가 동일하므로 줄번호가 아닌 **문자열 매칭**으로 찾을 것.

## Constraints
- `hardThrottleStopped`를 로그인 스로틀 차단에 **재사용 금지** (자동재시도가 꺼짐). 반드시 별도 `loginThrottleAborted` 사용.
- 신규 플래그는 `throttleStreak`와 동일 스코프에 선언 (IIFE 최상위 `let`).
- 세 리셋 지점(runBatch/runCustomDateOrders/runRetry) 모두에 신규 플래그 리셋 추가 — 하나라도 누락 시 다음 배치에서 이전 streak가 남아 오작동.
- `waitBeforeNextAccount`, `finishBatch`는 수정하지 않는다.
- 기존 `log()` 사용, 폴더 구조·파일명 변경 금지.
- 부차 이슈(CMG `조회 버튼 찾기 실패`, 조기 페이지 종료, WUJIE 미준비, 10057)는 이번 범위 밖 — 건드리지 말 것.
