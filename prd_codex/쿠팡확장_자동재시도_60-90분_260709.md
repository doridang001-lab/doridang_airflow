# 쿠팡 확장 자동 재시도: 60~90분 랜덤 × 최대 3회

## Task
쿠팡이츠 크롬 확장 배치의 **실패분 자동 재시도 대기 간격을 5~10분 → 60~90분 랜덤으로 늘리고, 재시도 횟수를 1회 → 3회로 확대**한다. 로그인 에러(`해당하는 요청을 처리할 권한이 존재하지 않습니다`)는 자격증명 문제가 아니라 연속 접속에 대한 쿠팡의 일시적 세션/권한 차단이며, 6분 뒤 재시도로는 안 풀리고 2~3시간 뒤 재접속 시 정상 성공한다(실측 확인됨). 이 수동 패턴을 자동화하는 것이 목적.

### runner.js — 자동 재예약 루프 (라인 507~524, 1366~1375)
```js
function scheduleAutoRetry(labelPrefix = '↩ 재시도') {
  if (autoRetryTimer) { clearInterval(autoRetryTimer); autoRetryTimer = null; }
  const delay = rand(AUTO_RETRY_DELAY_MIN_MS, AUTO_RETRY_DELAY_MAX_MS);
  const deadline = Date.now() + delay;
  el.retryBtn.style.display = '';
  el.retryBtn.textContent = `${labelPrefix} (${fmtRetryDelay(delay)} 후 자동 시작)`;
  log(`자동 재시도 예약 — ${fmtRetryDelay(delay)} 후 시작`, 'info');
  autoRetryTimer = setInterval(() => {
    const remaining = deadline - Date.now();
    if (remaining > 0) {
      el.retryBtn.textContent = `${labelPrefix} (${fmtRetryDelay(remaining)} 후 자동 시작)`;
    } else {
      clearInterval(autoRetryTimer); autoRetryTimer = null;
      el.retryBtn.textContent = '↩ 실패 재시도';
      runRetry();
    }
  }, 1000);
}
// finishRetry 내부:
const stillFailing = incompleteStores().length > 0;
if (stillFailing && !stopRequested && !hardThrottleStopped) {
  if (retryRound < MAX_AUTO_RETRY) {
    scheduleAutoRetry(`↩ 재시도 ${retryRound}/${MAX_AUTO_RETRY}`);
  } else {
    el.retryBtn.textContent = '↩ 실패 재시도';
    notifyFailures();
  }
}
```

## Test Cases
로컬에서 60~90분 대기 없이 검증하려면 상수를 임시 축소:

1. [임시 값] `AUTO_RETRY_DELAY_MIN_MS=30*1000`, `AUTO_RETRY_DELAY_MAX_MS=40*1000`, `MAX_AUTO_RETRY=2`로 두고 `chrome://extensions`에서 확장 새로고침 → 콘솔 에러 없이 로드.
2. [카운트다운] 일부 매장이 실패하도록 배치 실행 → 배치 완료 후 재시도 버튼에 "↩ 재시도 (NN초 후 자동 시작)" 표시 → 만료 시 `runRetry` 자동 발화 → 기대: 수동 클릭 없이 재시도 시작.
3. [실패분만] 재시도 로그가 `재시도 N/M회차 시작 — X개 매장`이며 X가 **실패 매장 수와 일치**(전체 아님) → 기대: 성공 매장 미포함.
4. [3회 상한] 계속 실패 시 `재시도 1/2`, `2/2`까지 자동 재예약되고, 회차 소진 후 백필 큐 저장 로그(`📋 백필 큐 저장 — 미완료 매장 N개`) 출력 → 기대: 무한 반복 없음.
5. [시간 표시] `fmtRetryDelay` 개선 확인 — 실제 값(60~90분)에서 라벨이 "1시간 X분 후 자동 시작"으로 나옴. 60분 미만은 "N분 NN초" 유지.
6. [최종 복원] 상수를 최종값(`60*60*1000` / `90*60*1000` / `3`)으로 되돌리고 다시 확장 새로고침 → 콘솔 에러 없음.

## Verification Loop
```
LOOP until all PASS:
  1. Test Cases 1~5를 임시 축소 상수로 순서대로 실행
  2. FAIL 항목 → 원인 분석 (확장 콘솔 로그 확인)
  3. 코드 수정
  4. 전체 재실행
종료 조건: 1~5 전체 PASS → Test Case 6(상수 최종값 복원) 수행
```

## Constraints
- **10057 하드스로틀 매장은 자동 재시도에서 계속 제외**해야 한다. `hardThrottleStopped` 가드(라인 1262·1268·1368)를 절대 제거/우회하지 말 것 — 당일 재접속은 차단 심화 위험.

- 최종 실패분의 백필 큐 저장(`ce_backfill_queue`) 동작 유지 — 다음날 배치가 자동 재수집.
