# 쿠팡이츠 주문서 수집 — 안정화 타임아웃 페이지 복귀 업그레이드

## Task
쿠팡이츠 주문 수집(크롬 확장) 중 SPA 로딩이 안 멈춰 `안정화 타임아웃`이 나면
현재는 그 페이지를 그대로 수집한 채 넘어가 주문이 누락된다. 타임아웃 시점의 active
페이지 번호를 기억했다가 조회(돋보기) 재클릭으로 목록을 다시 불러와 해당 페이지로
되돌아가 중복 없이 재수집하고, soft 복귀가 계속 실패하면 전체 새로고침(SHIFT+F5)
기반 재개로 폴백한다.

- 대상 사이트: `https://store.coupangeats.com/merchant/management/orders/{store_id}`
- 대상 폴더(작업 루트): `C:\Users\민준\OneDrive - 주식회사 도리당\Extention\doridang_collector_개발용`
- 프로젝트 타입: **Chrome Manifest V3 확장 (Vanilla JS)** — Python/Airflow 아님.

## Project Conventions (이 확장 기준)
- 전역 객체: 수집 로직은 `Sites['coupangeats']` (00_config.js의 `var Sites = {}` 레지스트리에 등록).

## Implementation Steps

### 1. 페이지 네비게이션 헬퍼 추가 (`_getCurrentPage`/`_clickNextPage` 근처, 현재 L1214 부근)
```js
// 현재 페이지네이션 창에 보이는 숫자 버튼 목록 (컨트롤 버튼 제외)
_getVisiblePageNumbers() {
  const btns = document.querySelectorAll('.merchant-pagination ul li button');
  const out = [];
  for (const b of btns) {
    if (b.classList.contains('pagination-btn')) continue; // first/prev/next/last 제외
    const n = parseInt((b.textContent || '').trim(), 10);
    if (!isNaN(n)) out.push({ num: n, btn: b });
  }
  return out;
},

// 보이는 창 안의 특정 번호 버튼 클릭
_clickPageNumber(n) {
  const hit = this._getVisiblePageNumbers().find(x => x.num === n);
  if (hit) { hit.btn.click(); return true; }
  return false;
},
```
> DOM 근거: 컨트롤 버튼은 모두 `class="pagination-btn first-page-btn|prev-btn|next-btn|last-page-btn"`,
> 숫자 버튼은 클래스 없음(active만 `class="active"`). active 읽기는 기존 `_getCurrentPage()`가 `button.active` 사용.

### 2. `_recoverToPage(targetPage)` 를 창 점프 방식으로 교체 (현재 L1724~1747)
한 칸씩 `next-btn` 누르던 기존 방식 → "끝번호 클릭 점프 + 창 이동" 방식.
```js
async _recoverToPage(targetPage) {
  Utils.updateProgressModal({ debug: `↩ ${targetPage}페이지로 복구 이동 (재조회)` });
  if (!await this._clickOrdersSearchButton()) return false;
  await this._waitForOrdersDataReady(8000);
  await this._waitForStableState(5000);

  let guard = 0;
  let cur = this._getCurrentPage();
  while (cur < targetPage && !this._stopFlag) {
    if (guard++ > 40) return false;
    const nums = this._getVisiblePageNumbers();
    if (nums.length === 0) return false;
    const prevId = this._getFirstOrderId();

    if (nums.some(x => x.num === targetPage)) {
      // 목표가 현재 창에 보임 → 직접 클릭 (정확 착지)
      this._clickPageNumber(targetPage);
    } else {
      const maxVisible = Math.max(...nums.map(x => x.num));
      if (cur < maxVisible) {
        // 끝번호로 점프 (사용자: "번호 없으면 끝번호 선택")
        this._clickPageNumber(maxVisible);
      } else {
        // 창의 끝 페이지 → next-btn으로 창 이동 (사용자: "next 옆으로 / 5번마다 옆페이지")
        if (!this._clickNextPage()) return false;
      }
    }

    let loaded = await this._waitForPageLoad(prevId, 15000);
    if (!loaded) {
      await this._randomDelay(1500, 2500);
      await this._closeErrorPopups();
      loaded = await this._waitForPageLoad(prevId, 10000);
      if (!loaded) return false;
    }
    cur = this._getCurrentPage();
    Utils.updateProgressModal({ message: `복구 이동 중... ${cur}/${targetPage}페이지` });
  }
  return this._getCurrentPage() === targetPage;
},
```

### 5. hard reload 폴백 + 재개 진입점 추가 (`_collectOrdersPages` 인접에 메서드 추가)
```js
async _reloadAndResume(resumePage, rows, idSet, shopInfo) {
  const today = new Date().toISOString().slice(0, 10).replace(/-/g, '');
  const storeIdMatch = location.href.match(/\/orders\/(\d+)/);
  await new Promise(r => chrome.storage.local.set({ ce_orders_resume: {
    date: today,
    storeId: storeIdMatch ? storeIdMatch[1] : '',
    resumePage,
    rows,
    collectedOrderIds: [...idSet],
    shopInfo
  }}, r));
  await new Promise(r => chrome.storage.local.set({ ce_orders_restart: { date: today } }, r));
  Utils.updateProgressModal({ debug: `🔄 SHIFT+F5 전체 새로고침 후 ${resumePage}페이지부터 재개` });
  location.reload();
},

// 05_main.js의 load 핸들러가 reload 후 호출
async _resumeOrdersFromReload(state) {
  const allRows = Array.isArray(state.rows) ? state.rows : [];
  const collectedOrderIds = new Set(state.collectedOrderIds || []);
  const shopInfo = state.shopInfo || this._getShopInfo();
  Utils.showProgressModal('쿠팡이츠 수집', `재개: ${state.resumePage}페이지로 복귀 중...`);

  // 조회 → 목표 페이지 복귀
  const recovered = await this._recoverToPage(state.resumePage);
  if (!recovered) {
    Utils.updateProgressModal({ debug: `❌ 재개 복귀 실패 (${state.resumePage}페이지)` });
  }
  // resumePage부터 기존 수집 루프를 누적 상태로 이어서 실행
  const result = await this._collectOrdersPages(shopInfo, { allRows, collectedOrderIds });
  await this._downloadOrdersResult(result.allRows, shopInfo, { isMultiStore: false });
},
```
> `_collectOrdersPages(shopInfo, seed = {})`로 시그니처를 소폭 일반화:
> 함수 상단에서 `const allRows = seed.allRows || []; const collectedOrderIds = seed.collectedOrderIds || new Set();`
> 로 초기화하도록 바꿔 재개 시 누적 rows/ID를 이어받게 한다(기존 호출부는 인자 없이 호출 → 빈 seed로 동작 동일).

### 6. `05_main.js` — reload 후 자동 재개 분기 추가 (현재 L40~63)
orders 페이지 load 핸들러에서 `ce_orders_resume`가 today면 처음부터가 아니라 재개로 분기:
```js
if (url.includes('store.coupangeats.com/merchant/management/orders/')) {
  chrome.storage.local.get(['ce_orders_restart', 'ce_orders_resume'], (result) => {
    const today = new Date().toISOString().slice(0, 10).replace(/-/g, '');

    // 페이지 복귀 재개 (안정화 타임아웃 폴백)
    if (result.ce_orders_resume && result.ce_orders_resume.date === today) {
      chrome.storage.local.remove(['ce_orders_restart', 'ce_orders_resume']);
      setTimeout(() => {
        console.log('[Collector] SHIFT+F5 재시작 - 페이지 복귀 재개');
        Sites['coupangeats']._resumeOrdersFromReload(result.ce_orders_resume);
      }, 3000);
      return;
    }

    // (기존 로직 유지) ce_orders_restart 배치 재시작
    const restart = result.ce_orders_restart;
    if (restart && restart.date === today) {
      chrome.storage.local.remove(['ce_orders_restart']);
      setTimeout(() => { runCollector({ source: 'batch' }); }, 3000);
      return;
    }
    if (sessionStorage.getItem('__collector_auto_run') === '1') {
      sessionStorage.removeItem('__collector_auto_run');
      setTimeout(() => { runCollector({ source: 'batch' }); }, 2500);
    }
  });
  return;
}
```

### content/03_coupangeats.js — 안정화 대기 (반환값 활용 대상)
```js
async _waitForStableState(maxWait = 5000) {
  const startTime = Date.now();
  let lastCount = 0;
  let stableCount = 0;
  while (Date.now() - startTime < maxWait) {
    await this._closeErrorPopups();
    const currentItems = document.querySelectorAll('.order-search-result-content > li.col-12');
    const currentCount = currentItems.length;
    if (currentCount === lastCount && currentCount > 0) {
      stableCount++;
      if (stableCount >= 3) {
        Utils.updateProgressModal({ debug: `✅ 페이지 안정화 (주문 ${currentCount}개)` });
        return true;
      }
    } else { stableCount = 0; lastCount = currentCount; }
    await this._randomDelay(300, 500);
  }
  Utils.updateProgressModal({ debug: `⚠️ 안정화 타임아웃 (주문 ${lastCount}개)` });
  return false; // ← 이 false를 _collectCurrentPageData가 전파해야 함
},
```

## Test Cases
확장은 자동 테스트가 없으므로 수동 검증. 각 단계는 `chrome://extensions`에서 확장
새로고침 후 orders 페이지에서 진행.

1. **문법/로드** — `chrome://extensions`에서 해당 확장 새로고침 → 기대: 에러 배지 없음, 콘솔에 `[Collector] 로드 완료.` 출력.
2. **헬퍼 동작** — orders 페이지 콘솔에서
   `Sites['coupangeats']._getVisiblePageNumbers().map(x=>x.num)` → 기대: `[16,17,18,19,20]` 형태 숫자 배열(컨트롤 버튼 제외).
   `Sites['coupangeats']._getCurrentPage()` → 기대: active 번호(예: `18`).
3. **정상 수집** — `/`(단축키) 또는 아이콘 클릭으로 수집 → 기대: 끝 페이지까지 진행, 다운로드 CSV 고유 주문 수 == 페이지 상단 "N건".
4. **타임아웃 복귀(soft)** — DevTools Network 스로틀(Slow 3G 등)로 안정화 타임아웃 유도 →
   기대 로그 순서: `⚠️ 안정화 타임아웃` → `🔁 N페이지 안정화 실패 → 재조회 복귀` → `복구 이동 중... cur/N페이지` → `✅ N페이지 복귀 - 이어서 재수집`.
   기대: 최종 CSV에 중복 주문 없음(고유 주문 수 == 상단 건수).
5. **창 점프 이동** — 깊은 페이지(예: 18+)에서 복귀 발생 시 → 기대: `복구 이동 중...`이 5칸 단위로 빠르게 증가(매 페이지 1클릭이 아니라 끝번호 점프), 정확히 N페이지에 착지.
6. **hard reload 폴백** — `_recoverToPage`가 항상 false 반환하도록 임시 강제 →
   기대: `🔄 SHIFT+F5 전체 새로고침...` 로그 후 페이지 reload → 3초 뒤 `[Collector] SHIFT+F5 재시작 - 페이지 복귀 재개` → resumePage부터 이어지고 누적 rows 보존(최종 CSV 건수 정상).

## Constraints
- 변경은 **쿠팡이츠 주문 수집 경로(03_coupangeats.js / 05_main.js)** 에만 국한. 배민/CMG/쿠폰 수집 로직 건드리지 말 것.
- 기존 `collectedOrderIds` dedup 메커니즘을 우회/제거하지 말 것 — 복귀 후 재수집의 중복 방지 핵심.
- `_stopFlag` 가드를 모든 신규 루프에 유지(ESC 중단 호환).
- `_collectOrdersPages` 시그니처에 `seed` 인자를 추가하더라도 **기존 무인자 호출이 그대로 동작**해야 함(기본값 처리).
- `location.reload()` 호출 이후 코드는 실행되지 않음 — reload 전에 반드시 `chrome.storage.local` 저장 await 완료.
- `chrome.storage.local` 저장 용량: 1매장/1일 누적 rows는 수백 KB 수준(기본 ~10MB 내). 초과 우려 시 rows 저장을 빼고 재개 시 처음부터 dedup 재수집하는 경량안으로 축소.
- `next-btn`이 "1페이지 이동"인지 "5칸 창 이동"인지 환경에 따라 다를 수 있으나, 새 `_recoverToPage` 루프는 active가 target에 도달할 때까지 점프하므로 어느 쪽이든 수렴 — 실제 페이지에서 Test Case 5로 확인.
