# 배민 광고 funnel 백필 업그레이드

## Task
배민 광고 funnel 수집(`stat/advertisement`)은 현재 "어제" 하루만 수집해, 인터넷 끊김 등으로 누락된 과거 날짜를 채울 수 없다. orders 수집에 이미 있는 날짜 직접선택 달력 로직(`_set_date_specific`)과 백필 메커니즘(`ORDERS_BACKFILL_DAYS`)을 광고 funnel에도 미러링해, 누락 날짜를 **하루씩 단일 선택**(DAY_MANUAL에서 해당 날짜 2번 클릭 = 1일 범위)으로 재수집할 수 있게 한다.

## 현재 구조 (확인 완료)
- 광고 페이지 필터 버튼 2개(`button.Filter-module__lRdH`): idx 0 = 노출수/클릭수, idx 1 = 주문수/주문금액.
- 각 필터 모달: `input[name="ad-stats-period-filter"][value="DAILY"]` 선택 → `input[name="period"][value="YESTERDAY"]` 선택 → 적용.
- 사용자 HTML: 같은 `period` 라디오 그룹에 `value="DAY_MANUAL"` 옵션 존재. 선택 시 달력(`DatePicker`)이 열린다. 달력은 **두 달(예: 5월·6월)을 동시 렌더**, 미래 날짜는 `disabled`/`aria-disabled="true"`. 각 달은 `<table>` 안에 `<caption>"YYYY년 M월"` + `button[aria-label="N일"]` 구조.
- `_save_ad_funnel_csv`는 이미 `target_date`로 upsert(같은 날짜 행 제거 후 append)하므로 백필에 그대로 사용 가능 — 저장부 변경 불필요.

### 2. `DB_Beamin_05_ad_funnel.py` — 날짜 직접선택 helper 추가
`_apply_specific_date_to_filter(driver, btn_idx, target_date, store_name) -> list[str] | None` 추가. `_apply_yesterday_to_filter`를 본떠 만들되 YESTERDAY 대신 DAY_MANUAL + 달력:
1. 필터 버튼 idx 클릭 (기존 L141-150 그대로).
2. DAILY 라디오 라벨 클릭 (기존 L161-174 그대로).
3. `input[name="period"][value="DAY_MANUAL"]` 라디오/라벨 클릭(YESTERDAY 자리 교체). 라벨 우선, 없으면 radio.click() fallback.
4. 달력에서 target_date 선택 — orders `_set_date_specific`의 달력 로직 이식:
   - 헤더/캡션에서 (year, month) 읽어 목표 월로 `이전달` 버튼 이동(최대 24회).
   - **두 달 동시 렌더 주의**: `button[aria-label="{day}일"]`이 양쪽 달에 존재 가능. 목표 월 `<caption>`("YYYY년 M월")로 해당 `<table>`을 특정한 뒤, 그 table 안에서 `aria-disabled="false"`인 day 버튼만 클릭. 미래/타월 오클릭 금지.
   - 해당 day 버튼 클릭 → 같은 버튼 한 번 더 클릭(시작=종료, 1일 범위). 사용자 표현 "2번클릭".
5. 적용 버튼 클릭 (기존 L197-208의 `String.fromCharCode(0xC801,0xC6A9)` "적용" 탐색 재사용).
6. 필터 텍스트가 target 날짜로 갱신될 때까지 대기(`WebDriverWait`, TimeoutException 시 None 반환) → `_extract_filter_vals` 호출해 반환.

### DB_Beamin_04_orders.py (`_set_date_specific` — 달력 로직 이식 원본)
```python
def _set_date_specific(driver, target_date):
    dt = pendulum.parse(target_date, tz=KST); day_label = f"{dt.day}일"
    # 1. DatePicker.Trigger 클릭
    driver.execute_script("""let btn=document.querySelector('[data-atelier-component="DatePicker.Trigger"]');
        if(!btn){const wrap=document.querySelector('[class*="DefaultDateFilter"]'); btn=wrap&&wrap.querySelector('button');}
        if(btn){btn.click();return true;} return false;""")
    WebDriverWait(driver,10).until(lambda d:d.execute_script(
        "return !!document.querySelector('[aria-label$=\"일\"]');"))
    # 2. 헤더 (year,month) 읽어 목표 월로 '이전달' 이동 (최대 24회)
    for _ in range(24):
        ym = _get_calendar_ym(driver)  # 헤더 textContent 정규식 (\d{4})[^\d]+(\d{1,2})
        if ym and (ym[0],ym[1])==(dt.year,dt.month): break
        if ym and (ym[0]*12+ym[1])>(dt.year*12+dt.month):
            driver.execute_script("""const prev=[...document.querySelectorAll('button')]
                .find(b=>b.getAttribute('aria-label')==='이전달'||b.textContent.trim()==='<'
                ||b.className.includes('prev')||b.className.includes('Prev')); if(prev)prev.click();""")
            time.sleep(0.3)
        else: break
    # 3+4. 시작일 클릭 → 종료일 클릭 (같은 날 1일 범위)
    driver.execute_script(f"""const btns=[...document.querySelectorAll('button[aria-label="{day_label}"]')];
        if(btns.length>0){{btns[0].click();return true;}} return false;""")
    time.sleep(0.2)
    driver.execute_script(f"""const btns=[...document.querySelectorAll('button[aria-label="{day_label}"]')];
        if(btns.length>0) btns[btns.length-1].click();""")
    # 5. 적용
    apply_btn = WebDriverWait(driver,10).until(EC.element_to_be_clickable(
        (By.XPATH,"//button[@data-atelier-component='Button'][.//span[text()='적용']]")))
    human_click(driver, apply_btn)
```
> 주의: 광고 달력은 두 달 동시 렌더 → 위 `aria-label` 전역 셀렉터를 그대로 쓰면 타월 오클릭 위험. 목표 월 `<caption>`로 `<table>` 특정 후 그 안에서 `aria-disabled="false"` 버튼만 클릭하도록 보강할 것.
