# 쿠팡이츠 수동수집 체크포인트 저장 및 속도 개선

## Task
쿠팡이츠 수동 수집(manual mode) 시 전체 완료 후 한 번만 저장하는 구조라 중간 실패 시 데이터가 전부 소실된다.
100건마다 partial CSV로 저장하고 최신 날짜부터 재조회하는 체크포인트 루프를 구현한다.
아울러 manual 전용 딜레이 값을 절반 수준으로 단축해 속도를 개선한다.

- 빌드 경로: `C:\airflow\coupang_extension_build\content\`

### 5. `03_coupangeats.js` — `_collectOrdersManualWithCheckpoints` 신규

`_downloadOrdersPartial` 바로 뒤에 삽입:

```javascript
async _collectOrdersManualWithCheckpoints(shopInfo) {
  const CHECKPOINT_EVERY = 100;
  const fmtDate = (d) => `${d.getFullYear()}.${String(d.getMonth()+1).padStart(2,'0')}.${String(d.getDate()).padStart(2,'0')}`;
  const originalEndDateMs = this._readOrdersDateRange().end;
  let seed = { checkpointEvery: CHECKPOINT_EVERY, checkpointFrom: 0 };
  let partialIndex = 0;

  Utils.showProgressModal('쿠팡이츠 수집', '수집 시작...');

  while (!this._stopFlag) {
    const { allRows, pageCount, hasError, reloaded, checkpointReached } =
      await this._collectOrdersPages(shopInfo, seed);

    if (reloaded) return false;

    if (!allRows.length) {
      if (partialIndex === 0) {
        Utils.showSuccessModal('데이터 없음', '추출할 주문이 없습니다.', { status: 'error' });
      } else {
        Utils.showSuccessModal('수집 완료', `총 ${partialIndex}개 파일 저장 완료`, { status: 'ok' });
      }
      return partialIndex > 0;
    }

    partialIndex++;
    await this._downloadOrdersPartial(allRows, shopInfo, `part${String(partialIndex).padStart(3, '0')}`);

    if (!checkpointReached || this._stopFlag) {
      Utils.showSuccessModal('수집 완료', `총 ${partialIndex}개 파일 저장 완료`, { status: 'ok' });
      return true;
    }

    const restartDate = this._getRestartDateFromRows(allRows);
    if (!restartDate) {
      Utils.showSuccessModal('수집 완료', `총 ${partialIndex}개 파일 저장 완료`, { status: 'ok' });
      return true;
    }

    const endDate = new Date(parseInt(originalEndDateMs, 10));
    endDate.setHours(0, 0, 0, 0);
    if (restartDate.getTime() >= endDate.getTime()) {
      Utils.showSuccessModal('수집 완료', `총 ${partialIndex}개 파일 저장 완료`, { status: 'ok' });
      return true;
    }

    const preRows = allRows.filter(r => this._isOrderRowBeforeDate(r, restartDate));
    const preUniqueCount = new Set(preRows.map(r => r.order_id).filter(Boolean)).size;
    Utils.updateProgressModal({
      debug: `💾 ${partialIndex}번째 저장 (${allRows.length}건) → ${fmtDate(restartDate)}~${fmtDate(endDate)} 재조회`
    });

    const ok = await this._setOrdersDateRange(restartDate, endDate);
    if (!ok || this._stopFlag) break;
    const prevCount = this._getExpectedOrderCount();
    if (!await this._clickOrdersSearchButton()) break;
    await this._waitForOrdersCountChange(prevCount, 8000);
    await this._waitForOrdersDataReady(8000);

    seed = {
      allRows: preRows,
      collectedOrderIds: new Set(preRows.map(r => r.order_id).filter(Boolean)),
      pageCount: 0,
      checkpointEvery: CHECKPOINT_EVERY,
      checkpointFrom: preUniqueCount,
      _originalEndDateMs: originalEndDateMs,
    };
  }

  if (partialIndex > 0) {
    Utils.showSuccessModal('수집 중단', `${partialIndex}개 파일 저장됨`, { status: 'warning' });
  }
  return partialIndex > 0;
},
```

## Reference Code

### `_restartFromErrorDate` (기존 — 날짜 재조회 패턴 참고)
```javascript
async _restartFromErrorDate(safeRows, shopInfo, originalEndDateMs, dateRestartCount = 0) {
  if (this._collectSource !== 'manual') return { restarted: false };
  if (dateRestartCount >= 1) return { restarted: false };  // 기존은 1회 한도 — 새 함수는 무제한
  const restartDateObj = this._getRestartDateFromRows(safeRows); // MAX 날짜 반환
  const preRows = (safeRows || []).filter(r => this._isOrderRowBeforeDate(r, restartDateObj));
  // re-query restartDateObj ~ endDate
  const seed = {
    allRows: preRows,
    collectedOrderIds: new Set(preRows.map(r => r.order_id).filter(Boolean)),
    pageCount: 0,
    _originalEndDateMs: originalEndDateMs,
    _dateRestartCount: dateRestartCount + 1
  };
  const { allRows: newRows, hasError } = await this._collectOrdersPages(shopInfo, seed);
  return { restarted: true, newRows, hasError };
}
```

### `_getRestartDateFromRows` (기존 — MAX 날짜 반환)
```javascript
_getRestartDateFromRows(rows) {
  let maxDate = null;
  for (const row of rows || []) {
    const d = this._parseOrderDateOnly(row?.order_date);
    if (!d) continue;
    if (!maxDate || d.getTime() > maxDate.getTime()) maxDate = d;
  }
  return maxDate; // 수집된 rows 중 가장 최신 날짜
}
```

## Constraints

- `_collectOrdersManualWithCheckpoints`는 **manual + !isMultiStore** 에서만 동작 — 배치 경로(`isMultiStore=true`) 건드리지 말 것
- `_restartFromErrorDate`는 수정하지 말 것 (에러 복구 전용, 체크포인트와 별개)
- `checkpointEvery` / `checkpointFrom` seed 키는 `_collectOrdersPages` 내부에서만 소비 — 외부 호출부에서 직접 읽지 말 것
- partial 파일명: `part001`, `part002`... (3자리 패딩) — 파이프라인 glob 매칭 편의
- 파이프라인(Airflow) 측 `*_part*.csv` 파일 병합 로직은 별도 구현 필요 (이 PRD 범위 외)
- OneDrive 수정이므로 수정 후 반드시 Chrome 확장 리로드 필수

## Do Not Ask — Decide Yourself

- 파일이 이미 존재하면: 덮어쓰기
- `_collectOrdersPages` 기존 return 경로에 `checkpointReached` 없으면: `undefined` (falsy) 처리로 충분 — 명시적 false 불필요
- `CHECKPOINT_EVERY` 값: 100 고정 (상수로 추출하지 말 것, 이미 함수 내 지역변수)
- 딜레이 값 소수점: 정수만 사용
- 주석 스타일: 기존 파일 한글 인라인 주석 패턴 따라가기
