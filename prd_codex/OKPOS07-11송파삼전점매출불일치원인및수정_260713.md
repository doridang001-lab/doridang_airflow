# OKPOS 정산 지연분 재수집 (settlement recheck window)

## Task
OKPOS 수집 파이프라인이 "이미 값이 있는(0원 아닌) 날짜"를 다시 수집하지 않아, 배달 매출이 1~2일 늦게 정산 반영되는 최근일이 초기 스냅샷 값에 영구 고정되는 버그가 있다(예: 송파삼전점 07-11 실매출 682,000 → 실제 795,200 미반영). 정산 지연 창(기본 3일) 이내의 날짜는 값이 있어도 자동 재수집·해당일 replace 하도록 재수집 필터를 확장한다.

1. **신규 헬퍼 추가** — `DB_OKPOS_Sales.py`, `_is_daily_zero_suspect`(`:1063`) 근처에 삽입.
   ```python
   # 배달 매출은 OKPOS에 1~2일 지연 정산됨 → 최근일은 값이 있어도 재수집해 갱신한다.
   _SETTLEMENT_RECHECK_DAYS = int(os.getenv("OKPOS_SETTLEMENT_RECHECK_DAYS", "3"))


   def _is_settlement_recheck_target(sale_date: str) -> bool:
       """sale_date가 정산 지연 창(기본 3일) 이내의 과거일이면 재수집 대상."""
       days = _SETTLEMENT_RECHECK_DAYS
       if days <= 0:
           return False
       try:
           sale_day = datetime.strptime(sale_date, "%Y-%m-%d").date()
       except Exception:
           return False
       delta = (_kst_now().date() - sale_day).days
       return 0 < delta <= days
   ```
   - `delta > 0`: 당일(오늘)은 이미 항상 재수집되므로 제외, 과거일만 대상.

def _is_daily_zero_suspect(daily_csv: Path, sale_date: str, store_name: str) -> bool:
    """daily가 0원인데 order에는 매출이 있으면 daily 재수집 대상으로 본다."""
    daily_sum, daily_reason = _sum_csv_amount_for_date(daily_csv, sale_date, "실매출액")
    if daily_reason is not None or daily_sum != 0:
        return False
    ym = sale_date[:7]
    store_short = store_name.replace("도리당 ", "", 1)
    order_csv = RAW_OKPOS_SALES / "brand=도리당" / f"store={store_short}" / f"ym={ym}" / "okpos_order.csv"
    order_sum, order_reason = _sum_csv_amount_for_date(order_csv, sale_date, "실매출액")
    if order_reason is not None or not order_sum:
        return False
    logger.warning("daily 0원 의심 → daily 재수집 대상: %s | %s | daily=%s order=%s",
                   sale_date, store_name, daily_sum, order_sum)
    return True

### DB_OKPOS_Sales.py — `_filter_missing_keys` (`:1153`, 수정 대상)
```python
_NO_DATA_RETRY_RECENT_DAYS = 2  # 파일 상단(:97) 기존 상수

def _filter_missing_keys(sale_dates: list, csv_name: str) -> list:
    """sale_dates × STORES 중 csv_name.csv에 해당 날짜 데이터가 없는 (date, store) 조합만 반환."""
    missing = []
    marker_cache: dict[Path, set[str]] = {}
    for sale_date in sale_dates:
        ym = sale_date[:7]
        for store in STORES:
            if not _should_collect(store["name"], sale_date):
                continue
            store_short = store["name"].replace("도리당 ", "", 1)
            csv_path = (RAW_OKPOS_SALES / "brand=도리당"
                        / f"store={store_short}" / f"ym={ym}" / f"{csv_name}.csv")
            marker_path = _no_data_marker_path(store_short, ym, csv_name)
            if marker_path not in marker_cache:
                marker_cache[marker_path] = _read_no_data_dates(marker_path)
            if sale_date in marker_cache[marker_path]:
                try:
                    sale_day = datetime.strptime(sale_date, "%Y-%m-%d").date()
                    is_recent = (_kst_now().date() - sale_day).days <= _NO_DATA_RETRY_RECENT_DAYS
                except Exception:
                    is_recent = False
                if not is_recent:
                    continue
            if not csv_path.exists():
                missing.append((sale_date, store))
                continue
            try:
                df = pd.read_csv(csv_path, dtype=str, usecols=["sale_date"])
                if df[df["sale_date"].str.strip() == sale_date].empty:
                    missing.append((sale_date, store))
                    continue
                if csv_name == "okpos_daily" and _is_daily_zero_suspect(csv_path, sale_date, store["name"]):
                    missing.append((sale_date, store))
            except Exception:
                missing.append((sale_date, store))
    return missing
```

## Constraints
- 기존 `_filter_missing_keys`의 NO_DATA/zero-suspect 분기 흐름을 **깨지 말 것** — window 체크는 "행 있음 → skip" 경로 맨 앞에 **추가**만 한다.
- window 상한은 env `OKPOS_SETTLEMENT_RECHECK_DAYS`로 조절(기본 3). `<=0`이면 기능 비활성(과거 동작과 동일).
- 재크롤 부담 제한: window는 최근 N일 × 4매장 수준. window를 과도히 키우지 말 것.
- 당일(오늘) 날짜는 window 대상에서 제외(`delta > 0`) — 당일은 기존 로직이 이미 매번 수집.
- print 금지, logging 사용. 절대 import 경로 유지. 파일명/폴더 구조 변경 금지.
- 즉시 복구(07-11 교정)는 코드 변경과 별개로 DAG를 `{"sale_dates":["2026-07-11"],"force_redownload":true}` conf로 트리거하면 됨.
