# 루트_잡파일정리_재생성방지

## Task
`C:\airflow` 저장소 루트에 쌓인 ppt 일회성 스크립트(`open_pptx.py`, `output_temp.pptx` 등)와
`tmp_*`/`check_*` 디버그 파일을 제거한다. 핵심 원인은 이 파일들이 **git에 커밋되어 있어**
로컬 삭제만으로는 `git checkout/stash/reset` 시 되살아나는 것이다. git 추적 해제 + `.gitignore`
보강 + AGENTS.md 규칙으로 재발을 막는다.

## Project Conventions
- 이 작업은 파이프라인/DAG가 아닌 **저장소 위생(housekeeping)** 작업
- 추적 파일 제거는 반드시 `git rm` (로컬 `rm` 만으로는 재추적/복구됨)
- 미추적 스크래치는 일반 `rm -rf`
- `.gitignore` 보강으로 재생성 시 자동 무시
- 운영 코드/설정/문서는 절대 건드리지 않음

## Implementation Steps
1. **추적 잡파일 제거**
   ```bash
    cd C:\airflow
   $root = "C:\airflow"
   $targets = @(
     "gen_slides.py","render_new_slides.py","repack_pptx.py","repack2.py",
     "check_pres.py","check_font.py","open_pptx.py","output_temp.pptx",
     "__read_excel_tmp.py","tmp_debug_coupang_calendar.py","tmp_emp_debug.txt",
     "tmp_trigger_dbu.py","all_logs_summary.txt","tmp_coupang_accounts.json"
   )
   $tracked = git -C $root ls-files
   $rmTargets = $targets | Where-Object { $tracked -contains $_ }
   if ($rmTargets.Count -gt 0) {
     git -C $root rm --ignore-unmatch -f -- $rmTargets
   } else {
     Write-Host "[skip] tracked cleanup target 없음"
   }
   # 미추적 tmp_coupang_accounts.json은 별도 정리
   if (Test-Path "$root\tmp_coupang_accounts.json") { Remove-Item -Force "$root\tmp_coupang_accounts.json" }
    ```

2. **미추적 스크래치 폴더 삭제**
   ```bash
   Get-ChildItem -Path $root -Directory -Force -Name | Where-Object {
     @("pptx_unpacked2","qa_slides","tmp","tmp_airflow","tmp_airflow_home","airflow_tmp",
       ".airflow_tmp_test",".docker_tmp",".tmp","__pycache__",".pytest_cache") -contains $_
   } | ForEach-Object { Remove-Item -Recurse -Force "$root\$_" }
   ```

3. **`.gitignore` 보강** — 파일 끝에 아래 블록 추가:
   ```
   # ppt/스크래치 산출물
   output_temp.pptx
   *_unpacked*/
   qa_slides/
   # 루트 임시/디버그 산출물
   tmp_*
   *_tmp.py
   *_debug.py
   *_debug.txt
   all_logs_summary.txt
   __read_excel_tmp.py
   tmp_debug_*.py
   tmp_debug_*.txt
   # airflow/도구 스크래치 디렉터리
   tmp/
   tmp_airflow/
   tmp_airflow_home/
   .tmp/
   airflow_tmp/
   .airflow_tmp_test/
   .docker_tmp/
   __pycache__/
   .pytest_cache/
    ```
   (`tmp_airflow/`, `.tmp/`, `__pycache__/`, `.pytest_cache/`는 기존 패턴과 중복되더라도 허용)

4. **`AGENTS.md` 규칙 추가** — 파일 끝에:
   ```
   # 작업 산출물 규칙
   - 임시/디버그/일회성 스크립트·파일은 저장소 루트에 만들지 말 것.
   - 루트 임시 산출물은 `.tmp/`(gitignore됨) 또는 운영 스크립트가 사용하는 경로에서만 생성하고, `git add` 하지 말 것.
   ```

5. **커밋** (현재 `chore/github-pr-infra-0610` 브랜치)
   ```bash
   git add .gitignore AGENTS.md
   git commit -m "chore: 루트 ppt/디버그 잡파일 제거 및 gitignore 보강"
   ```

```
LOOP until all PASS:
  1. Test Cases 1~5 순서대로 실행
  2. FAIL 항목 → 원인 분석 (특히 #2: git rm 누락, #3: gitignore 패턴 오타)

```
