# 상품 매핑: 수동분류 중복 텔레그램 알람 추가

## Task
`DB_FinProduct_Map` 파이프라인에서 같은 상품(`source`, `item_id`)에 서로 다른 `수동분류_edit`가 2개 이상 붙는 경우(=`중복_수동분류=Y`)를 텔레그램으로 알린다. 현재는 review CSV의 `중복_수동분류` 컬럼에 `Y`로만 표시될 뿐 알람이 없어 검수자가 놓친다. 기존 `표준_메뉴명` 충돌 알람(`write_join_map` → `send_telegram`)과 동일한 방식으로 알람만 추가한다.

## Implementation Steps

1. **포맷 헬퍼 추가** — `_format_join_conflicts`(630줄 근처) 구조를 그대로 템플릿 삼아 신규 함수 작성:
   ```python
   def _format_duplicate_labels(review_rows: pd.DataFrame, limit: int = 10) -> str:
       if review_rows.empty or DUP_LABEL_COLUMN not in review_rows.columns:
           return ""
       dup = review_rows[review_rows[DUP_LABEL_COLUMN].fillna("").astype(str).str.strip() == "Y"]
       if dup.empty:
           return ""
       lines = []
       grouped = dup.groupby(["source", "item_id"], dropna=False)
       for (source, item_id), rows in list(grouped)[:limit]:
           labels = ", ".join(sorted(rows["수동분류_edit"].dropna().astype(str).str.strip().replace("", pd.NA).dropna().unique().tolist()))
           std_names = ", ".join(sorted(rows["표준_메뉴명_edit"].dropna().astype(str).str.strip().replace("", pd.NA).dropna().unique().tolist()))
           samples = ", ".join(rows["item_name"].dropna().astype(str).unique().tolist()[:3])
           lines.append(
               f"- item_id={item_id}, source={source} | 수동분류={labels} | 표준명={std_names} | samples={samples}"
           )
       remaining = len(grouped) - len(lines)
       if remaining > 0:
           lines.append(f"- 외 {remaining}개 key")
       return "\n".join(lines)
   ```

2. **알람 헬퍼 추가**:
   ```python
   def _notify_duplicate_labels(review_rows: pd.DataFrame) -> int:
       if review_rows.empty or DUP_LABEL_COLUMN not in review_rows.columns:
           return 0
       dup = review_rows[review_rows[DUP_LABEL_COLUMN].fillna("").astype(str).str.strip() == "Y"]
       dup_key_count = int(dup.drop_duplicates(subset=["source", "item_id"]).shape[0])
       if dup_key_count:
           detail = _format_duplicate_labels(review_rows)
           send_telegram(
               "[상품 매핑] 수동분류 중복 감지\n"
               f"중복 상품: {dup_key_count}개 (같은 상품에 수동분류 2개 이상)\n"
               "fin_product_map_review_input.csv의 중복_수동분류=Y 행을 확인해주세요.\n"
               f"{detail}"
           )
       return dup_key_count
   ```

## Constraints
- **DAG 파일 수정 금지** — 알람 로직은 파이프라인 모듈(`DB_FinProduct_Map.py`)에만 둔다.
- 중복 감지 로직을 새로 만들지 말 것 — `build_review_rows`가 이미 `중복_수동분류` 컬럼을 계산해 반환한다. 그 결과에 알람만 얹는다.
- dry_run에서는 알람 미발송 (기존 join 충돌 알람과 동일 일관성).
- print 금지, logging만 사용. 기존 파일의 타입힌트/주석 스타일 유지.
