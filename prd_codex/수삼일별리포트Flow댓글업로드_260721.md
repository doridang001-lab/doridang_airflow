# 수삼(백숙) 일별 리포트 생성 → Flow 게시글 댓글 자동 업로드

## Task
`unified_sales_grp` 일별 parquet에서 `item_name/menu_name`에 "수삼" 포함 주문만 뽑아 **그날 하루치 xlsx**를 만들고, 이를 **Flow(flow.team) 고정 게시글의 댓글**에 날짜 라벨과 함께 자동 첨부한다. 수삼 이벤트 실적을 매일 Flow AI에 적재해 분석하는 것이 목적. 먼저 07-01 한 건만 테스트하고, 문제없으면 07-02~20 백필 후 매일 1건 증분.

## Files to Create / Modify
- **생성**: `scripts/analysis/susam_report.py` — 생성(A) + 업로드(B) + 오케스트레이션(C) 통합 CLI
- **생성(런타임)**: `LOCAL_DB/flow_susam_uploaded.json` — 업로드 완료 날짜 상태파일(중복방지)
- **출력물**: `MART_DB/Flow/Baeksuk_YYYY-MM-DD.xlsx` (Flow 폴더 자동 생성)

## 필수 입력값 (실행 전 사용자/환경에서 주입)
- `FLOW_POST_URL` — 댓글 달 고정 게시글 detail URL(`https://flow.team/main.act?detail=…` 완전형). env `FLOW_POST_URL` 또는 `--post-url` 인자.
- `CHROME_USER_DATA_DIR`, `CHROME_PROFILE` — Flow에 **로그인된** 크롬 프로필 경로/이름. env 또는 인자. 해당 프로필의 Chrome은 닫아둬야 프로필 락 충돌 없음.

## Implementation Steps

1. **CLI 파서** (`argparse`)
   - `--date YYYY-MM-DD` (단일일) 또는 `--start/--end` (기간 루프). 둘 중 하나 필수.
   - `--keyword`(기본 `"수삼"`), `--no-upload`(생성만), `--force`(중복 무시 재업로드).
   - `--post-url`(없으면 env `FLOW_POST_URL`).

2. **컴포넌트 A: `generate_xlsx(date_str, keyword) -> Path`**
   - 파일 경로: `UNIFIED_ROOT / f"unified_sales_{datetime.strptime(date_str,'%Y-%m-%d').strftime('%y%m%d')}.parquet"`. 없으면 `FileNotFoundError`로 명확히 로그 후 skip.
   - `pd.read_parquet` → `is_susam = item_name OR menu_name .str.contains(keyword, na=False)` 필터.
   - 금액컬럼(`total_price` 등)이 문자열/콤마 가능 → `pd.to_numeric(s.astype(str).str.replace(',',''), errors='coerce')`로 정규화.
   - 출력 폴더 `MART_DB/"Flow"` `mkdir(parents=True, exist_ok=True)`.
   - `pd.ExcelWriter(out, engine="openpyxl")`로 4시트 작성:
     1. `요약` — 주문(order_cnt합)·수량(qty합)·매출(total_price합), 판매매장수(store nunique), platform별 매출 요약
     2. `매장별` — groupby(store,region,담당자) 주문·매출, 매출 내림차순
     3. `채널별` — groupby(platform) 주문·매출·비중(%)
     4. `주문상세` — 라인 원본(store,order_time,menu_name,item_name,qty,unit_price,total_price,discount_amount,platform)
   - return out path.

3. **컴포넌트 B: `upload_to_flow(xlsx_path, date_label, post_url, user_data_dir, profile)`**
   - Chrome 옵션: `--user-data-dir={user_data_dir}`, `--profile-directory={profile}`, `--window-size=1920,1080`. **headless 금지**(세션/프로필 재사용). `webdriver_manager` → 실패 시 `webdriver.Chrome(options=)` 폴백(참조 매크로 패턴).
   - `driver.get(post_url)` → `WebDriverWait` 로 게시글 로딩 대기.
   - **파일 첨부(핵심)**: label(`label.js-remark-upload-button.comment-upload-button`)은 숨은 `input[type=file]` 트리거. **label 클릭(OS 파일창) 금지** → 연결된 `input[type=file]`을 찾아 `input.send_keys(str(xlsx_path.resolve()))` 로 직접 주입.
     - input 셀렉터가 label HTML만으론 불확실 → 런타임에 `driver.find_elements(By.CSS_SELECTOR, "input[type='file']")` 로 후보 나열 로그 후, 댓글영역에 속한 것 선택. **이 DOM 확인이 1순위 구현 스텝.**
   - **댓글 텍스트**: 댓글 `textarea`/입력창에 `date_label`("2026-07-01") 입력.
   - **등록**: 등록 버튼 `execute_script("arguments[0].click();", btn)` (참조 매크로처럼 JS click). 업로드/등록 완료 대기.
   - 실패 시 `traceback.print_exc()` 후 raise. `finally: driver.quit()`.

4. **컴포넌트 C: 오케스트레이션 `main()`**
   - 상태파일 `LOCAL_DB/"flow_susam_uploaded.json"` 로드(list of dates).
   - 대상 날짜 목록 산출(`--date` 또는 `--start~--end` 일자 iterate).
   - for date: `--force`아니고 상태파일에 있으면 skip 로그 → `generate_xlsx` → (`--no-upload`아니면) `upload_to_flow` → 성공 시 상태파일에 date append+저장 → 다음 날짜 전 `time.sleep(3~5)`.

5. **의존성**: `openpyxl` import 확인(없으면 `pip install openpyxl` 안내 로그).

### dags/strategy/Strategy_FdamCS_02_FlowMacro_Dags.py (Selenium 세션재사용·JS click 패턴)
```python
from selenium import webdriver
from selenium.webdriver.chrome.options import Options
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

chrome_options = Options()
chrome_options.add_argument('--no-sandbox')
chrome_options.add_argument('--disable-dev-shm-usage')
chrome_options.add_argument('--window-size=1920,1080')
try:
    from webdriver_manager.chrome import ChromeDriverManager
    from selenium.webdriver.chrome.service import Service
    driver = webdriver.Chrome(service=Service(ChromeDriverManager().install()), options=chrome_options)
except Exception:
    driver = webdriver.Chrome(options=chrome_options)   # 폴백
driver.set_window_size(1920, 1080)
driver.get("https://www.myflow.kr/")          # ← 명시 로그인 코드 없음: 프로필 세션 의존
wait = WebDriverWait(driver, 20)
# 등록 버튼: presence_of_element_located + scrollIntoView + JS click
submit_btn = wait.until(EC.presence_of_element_located((By.CSS_SELECTOR, "button.js-complete-btn")))
driver.execute_script("arguments[0].scrollIntoView({block:'center'});", submit_btn)
driver.execute_script("arguments[0].click();", submit_btn)
```

### modules/transform/pipelines/db/DB_UnifiedSales_common.py (경로 상수·파일 iterate)
```python
from modules.transform.utility.paths import LOCAL_DB, MART_DB, ONEDRIVE_DB

UNIFIED_ROOT = MART_DB / "unified_sales_grp"

def iter_unified_sales_files() -> list:
    """실제 unified_sales parquet만 반환(.bak_ 백업 제외)."""
    if not UNIFIED_ROOT.exists():
        return []
    return sorted(
        path for path in UNIFIED_ROOT.glob("unified_sales_*.parquet")
        if ".bak_" not in path.name
    )
```

## Test Cases (단계식 — 07-01 게이트)
1. [생성 검증] `python scripts/analysis/susam_report.py --date 2026-07-01 --no-upload` → 기대: `MART_DB/Flow/Baeksuk_2026-07-01.xlsx` 생성, 4시트 존재, `요약` 값이 해당일 parquet 수삼 합계와 일치.
2. [🚦07-01 단독 업로드 게이트] `FLOW_POST_URL=... python scripts/analysis/susam_report.py --date 2026-07-01` → 기대: 프로필 세션으로 게시글 열림 · `input[type=file]` send_keys로 첨부 · 댓글 "2026-07-01" 등록이 실제 Flow에 반영 · 상태파일에 `2026-07-01` 기록. **이 단계 PASS 전까지 다음 단계 진행 금지.**
3. [백필] `python scripts/analysis/susam_report.py --start 2026-07-02 --end 2026-07-20` → 기대: 19건 순차 업로드, 상태파일 20날짜, 재실행 시 전부 skip.
4. [증분] `python scripts/analysis/susam_report.py --date 2026-07-21` → 기대: 1건 업로드.
5. [모듈 import] `python -c "import scripts.analysis.susam_report"` (또는 파일 직접 실행 경로) → 기대: ImportError 없음.

## Verification Loop
구현 완료 후 아래 루프를 **Test Cases 1·5 PASS + 2번(07-01 업로드) 실제 반영 확인까지** 반복한다. 3·4는 2번 게이트 통과 후 사용자 승인하에 실행.
```
LOOP until PASS:
  1. Test Case 1(생성)·5(import) 실행
  2. FAIL → 원인 분석 → 코드 수정 → 재실행
  3. Test Case 2(07-01 업로드) 실행 → Flow 게시글에 댓글+첨부 반영 눈으로 확인
종료 조건: 1·5 PASS + 2 실제 반영 + Constraints 위반 없음
```

## Constraints
- **데이터 소스는 `unified_sales_grp` parquet 하나만** — 우가클/daily_summary 조인 금지(후속).
- 파일 내용은 **그날 하루치**만 — 월 누적/전환타겟 누적뷰 만들지 말 것(후속).
- Selenium 업로드는 **label 클릭으로 OS 파일창 열지 말고 `input[type=file].send_keys(절대경로)`** 사용.
- Chrome은 **비headless + 로그인된 프로필 재사용**. ID/PW 하드코딩 금지.
- 중복 업로드 방지: 상태파일 기준 skip(`--force`로만 재업로드).
- 폴더/파일명 규칙 유지: 출력은 `MART_DB/Flow/Baeksuk_YYYY-MM-DD.xlsx`.
- print 금지, logging 사용.
