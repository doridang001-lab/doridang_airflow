# 구로디지털단지점 쿠팡이츠 수집 불가 버그 수정

## Task
쿠팡이츠 배치 수집 시 "구로디지털단지점"이 orders 수집에서 누락된다.
원인은 `accounts.js`의 등록 매장명(`도리당 구로디지털점`)과 Coupang HTML 실제 매장명(`닭도리탕 전문 도리당 구로디지털단지점`) 불일치.
`accounts.js` 2줄 수정 + 빌드 JS 파일 2개를 OneDrive 확장 폴더에 복사하면 해결된다.

## Root Cause 분석

매칭 로직 (`03_coupangeats.js:322`):
```javascript
normalized.includes(target) || target.includes(normalized)
```

| 변수 | 값 |
|------|-----|
| `target` (accounts.js 등록명) | `도리당 구로디지털점` |
| `normalized` (Coupang HTML) | `닭도리탕 전문 도리당 구로디지털단지점` |

- `"닭도리탕 전문 도리당 구로디지털단지점".includes("도리당 구로디지털점")` → **false**
  - "디지털" 뒤에 HTML은 "단지점", target은 "점" → 문자열 불일치
- 결과: 브랜드 체크(도리당 포함) 통과 → 매장 필터에서 탈락 → orders 수집 0건

수정 후:
- `target` = `도리당 구로디지털단지점`
- `"닭도리탕 전문 도리당 구로디지털단지점".includes("도리당 구로디지털단지점")` → **true** ✓

### 03_coupangeats.js — 매칭 로직
```javascript
// _normalizeStoreName: 공백 정규화만 (소문자 변환 없음)
_normalizeStoreName(text) {
  return String(text || '').replace(/\s+/g, ' ').trim();
},

// _filterTargetStores: includes() 양방향 체크
_filterTargetStores(storeNames, opts = {}) {
  const brandStores = (storeNames || []).filter(storeName => this._isTargetBrandStoreName(storeName));
  const targetSet = this._targetStoreSet(opts);
  if (!targetSet) return brandStores;
  const targets = [...targetSet].filter(target => this._isTargetBrandStoreName(target));
  return brandStores.filter(storeName => {
    const normalized = this._normalizeStoreName(storeName);
    return targets.some(target => normalized === target || normalized.includes(target) || target.includes(normalized));
  });
},
```
