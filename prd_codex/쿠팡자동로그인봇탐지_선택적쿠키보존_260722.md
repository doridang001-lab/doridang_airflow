## Task
쿠팡 확장 배치 수집에서 **첫 계정만 로그인 성공하고 2번째 계정부터 전부 실패**한다(오류: `해당하는 요청을 처리할 권한이 존재하지 않습니다` = Akamai ACCESS_DENIED). 원인은 계정 전환마다 `clearSession()`이 Akamai 신뢰 쿠키(`_abck`/`bm_*`/`ak_bmsc`)까지 전량 삭제 → 합성 로그인이 봇으로 판정되기 때문. **계정 전환 시 Akamai 신뢰 쿠키는 보존하고 쿠팡 로그인 세션만 종료**하도록 바꾸고, `_abck` 검증 상태를 로그로 계측한다.

## Implementation Steps

1. **`manifest.json` 권한 추가**
   - line 6 `permissions` 배열에 `"cookies"` 추가. host_permissions에 store/advertising 도메인은 이미 존재하므로 추가 불필요.
   ```json
   "permissions": ["activeTab", "storage", "scripting", "downloads", "tabs", "alarms", "browsingData", "cookies"],
   ```

2. **`runner.js` — `abckState(url)` 헬퍼 추가** (clearSession 근처, 예: 794행 위)
   - `_abck` 쿠키를 읽어 **끝 세그먼트**를 반환. `~-1~`로 끝나면 Akamai 미검증/봇 판정, `~0~`/양수면 검증됨.
   ```js
   async function abckState(url) {
     try {
       const c = await new Promise(res => chrome.cookies.get({ url, name: '_abck' }, res));
       if (!c || !c.value) return 'none';
       const parts = c.value.split('~');
       return parts.length >= 2 ? '~' + parts.slice(-3).join('~') : 'set';
     } catch (e) { return 'err:' + e.message; }
   }
   ```

3. **`runner.js` — `clearSession()` 재작성** (현 794-804 전체 교체)
   - 전량 쿠키 삭제 → **Akamai 허용목록 제외 전부 삭제**(쿠팡 세션 쿠키 이름 몰라도 로그아웃 보장).
   - localStorage는 계속 초기화(쿠팡 토큰 잔류 방지, Akamai는 쿠키 기반이라 무관).
   - 삭제 개수/보존 쿠키/`_abck` tail을 `log`로 계측.
   ```js
   async function clearSession() {
     const DOMAINS = ['store.coupangeats.com', 'advertising.coupangeats.com'];
     const KEEP = ['_abck', 'ak_bmsc', 'bm_sz', 'bm_sv', 'bm_mi', 'bm_lso', 'bm_s'];
     const keep = (name) => KEEP.includes(name) || name.startsWith('bm_') || name === '_abck';
     let removed = 0; const kept = [];
     try {
       for (const domain of DOMAINS) {
         const cookies = await new Promise(res => chrome.cookies.getAll({ domain }, res)) || [];
         for (const ck of cookies) {
           if (keep(ck.name)) { kept.push(ck.name); continue; }
           const scheme = ck.secure ? 'https' : 'http';
           const url = `${scheme}://${ck.domain.replace(/^\./, '')}${ck.path}`;
           await new Promise(res => chrome.cookies.remove({ url, name: ck.name }, res));
           removed++;
         }
       }
       // localStorage만 초기화 (쿠키는 위에서 선택 삭제 완료)
       await new Promise(res => chrome.browsingData.remove(
         { origins: ['https://store.coupangeats.com', 'https://advertising.coupangeats.com'] },
         { localStorage: true }, res));
       const abck = await abckState('https://store.coupangeats.com/merchant/login');
       log(`세션 선택초기화 — 삭제 ${removed} / 보존 ${uniq(kept).join(',') || '없음'} | _abck ${abck}`);
       return true;
     } catch (e) { log(`세션 초기화 오류(무시): ${e.message}`); return false; }
   }
   ```
   - `uniq(...)`는 runner.js에 이미 존재하는 헬퍼 사용. 없으면 `[...new Set(kept)]`로 대체.

4. **`runner.js` — 로그인 제출 직전 `_abck` 상태 계측** (로그인 루프, `AUTO_LOGIN` sendMessage 직전 ~952행)
   - `chrome.tabs.sendMessage(... AUTO_LOGIN ...)` 바로 앞에 삽입:
   ```js
   log(`[${acc.id}] 로그인 제출 직전 _abck ${await abckState(LOGIN_URL)} (${attempt}/${MAX_LOGIN})`);
   ```
   - 목적: 매 attempt마다 "제출 시점 Akamai 검증 상태"를 남겨, 보존 전략이 신뢰를 유지하는지/봇 판정인지 확정.
