# 쿠팡확장중단버튼미작동수정

## Task
쿠팡 자동수집 크롬 확장에서 대시보드의 "중단" 버튼을 눌러도 실제로 진행 중인 수집(워크탭 내부의 페이지 스크래핑)이 멈추지 않는 버그를 고친다. 오케스트레이터(runner.js)의 stop 플래그와 워크탭에서 실행되는 content script의 stop 플래그가 서로 연결되어 있지 않아 발생하는 문제다.

## Project Conventions
- 이 저장소는 Airflow Python 프로젝트지만, 이번 작업은 크롬 확장(JS) 코드 수정이다. Python 컨벤션(logging, paths.py 등)은 이번 작업과 무관.
- 확장 코드 스타일: 기존 파일의 들여쓰기(2 spaces)·네이밍(camelCase 변수, UPPER_SNAKE 상수)을 그대로 따른다.
- 주석: 기존 코드에 이미 한글 주석이 많다. 새 주석도 필요한 곳(WHY)에만 한글로 최소 추가.
- **동일한 수정을 두 곳에 반드시 동일하게 적용**:
  1. `C:\airflow\coupang_extension_build` (저장소 추적 대상, git 커밋용)
  2. `C:\Users\민준\OneDrive - 주식회사 도리당\Extention\doridang_collector_개발용` (실사용 배포 복사본 — 실제 크롬에 로드되는 폴더)
  두 폴더의 `runner.js`, `content/03_coupangeats.js`, `content/05_main.js`는 구조가 거의 동일하다. 인코딩 차이(레포판은 일부 문자열이 mojibake로 깨져 있음, OneDrive판은 정상 한글)가 있을 수 있으니 각 파일 원문 인코딩을 유지한 채 수정할 것.

## Implementation Steps

### 1. `content/05_main.js` — STOP 메시지 핸들러 추가
`chrome.runtime.onMessage.addListener((msg, sender, sendResponse) => { ... })` 안, 기존 `COLLECT` 핸들러 블록 근처에 다음을 추가한다 (다른 `if (msg?.type === '...')` 블록들과 동일한 패턴):

```javascript
if (msg?.type === 'STOP') {
  try {
    if (Sites['coupangeats']) Sites['coupangeats']._stopFlag = true;
    if (Sites['baemin']) Sites['baemin']._stopFlag = true;
  } catch (e) {}
  sendResponse({ success: true });
  return true;
}
```

- `Sites`는 `content/00_config.js`에서 `var Sites = {};`로 선언되고, `content/02_baemin.js`(`Sites['baemin'] = {...}`), `content/03_coupangeats.js`(`Sites['coupangeats'] = {...}`)에서 채워진다. `_stopFlag`는 각 사이트 객체의 최상위 속성(`_stopFlag: false`)이며, `Escape` keydown 핸들러가 `this._stopFlag = true`로 설정하는 것과 동일한 값이다. 즉 STOP 메시지는 "가상 ESC"로 동작하면 된다.
- baemin도 동일 패턴(`_stopFlag`)을 쓰는지 `content/02_baemin.js`에서 `_stopFlag` grep으로 먼저 확인하고, 없으면 baemin 라인은 생략해도 무방 (있으면 포함해 일관성 유지).

### 2. `runner.js` — 현재 워크탭 ID를 모듈 변수로 추적
파일 상단 상태 변수 선언부(약 44-60번째 줄, `let queue = []; let running = false; let stopRequested= false; ...` 근처)에 추가:

```javascript
let currentWorkTabId = null;  // 현재 활성 워크탭 — STOP 메시지 라우팅용
```

`workTab = await chrome.tabs.create({url:'about:blank', active:true});` 가 등장하는 모든 위치(대략 1209번 줄 `bfWorkTab`, 1244번 줄, 1360번 줄 — `let workTab;` 선언 직후 tabs.create 호출부)에서, 탭 생성 직후 `currentWorkTabId = workTab.id;` (또는 `bfWorkTab.id`)를 설정한다. 해당 함수의 `finally` 블록(탭 정리/닫기 하는 부분)에서는 `currentWorkTabId = null;`로 리셋한다. 기존 tab 정리 로직(`chrome.tabs.remove` 등)을 찾아 그 근처에 추가.

### 3. `runner.js` — 중단 버튼 핸들러 수정
기존 핸들러:

```javascript
el.stopBtn.addEventListener('click', () => {
  if (autoRetryTimer) { clearInterval(autoRetryTimer); autoRetryTimer = null; }
  stopRequested = true; log('중단 요청됨 — 현재 수집 완료 후 멈춥니다.','err');
});
```

다음과 같이 수정:

```javascript
el.stopBtn.addEventListener('click', () => {
  if (autoRetryTimer) { clearInterval(autoRetryTimer); autoRetryTimer = null; }
  stopRequested = true;
  log('중단 요청됨 — 현재 수집을 즉시 중단합니다.', 'err');
  if (currentWorkTabId) {
    chrome.tabs.sendMessage(currentWorkTabId, { type: 'STOP' }, () => {
      if (chrome.runtime.lastError) {
        log(`[STOP] 워크탭 메시지 전달 실패: ${chrome.runtime.lastError.message}`, 'err');
      }
    });
  }
});
```

- `chrome.runtime.lastError`를 콜백에서 확인하는 이유: content script가 아직 주입 전이거나 탭이 이미 닫힌 경우 unchecked runtime.lastError 경고를 막기 위함.
- 로그 문구를 "현재 수집 완료 후 멈춥니다" → "즉시 중단합니다"로 바꾼 것은 실제 동작이 바뀌었기 때문 (UX 문구 일치).

2. **정적 검증**: `Get-Content 'C:\airflow\coupang_extension_build\runner.js' | Select-String "currentWorkTabId"` → 최소 4곳(선언 1 + 탭생성 시 대입 2~3 + 클릭핸들러 1) 매치.
3. **문법 검증**: `node --check "C:\airflow\coupang_extension_build\runner.js"` 및 `node --check "C:\airflow\coupang_extension_build\content\05_main.js"` → 에러 없음 (Node 미설치 시 생략 가능, 대신 브라우저 콘솔에서 SyntaxError 없는지 확인).
4. **동일성 검증**: OneDrive 배포본과 repo 빌드본에 동일한 수정이 적용됐는지 `diff`(또는 `Compare-Object`)로 두 `runner.js`, 두 `content/05_main.js`의 수정된 블록만 비교 — 로직 동일해야 함 (인코딩/공백 차이는 허용).
5. **수동 실기 테스트 (사용자가 직접 확인, Codex는 안내만)**: 확장 프로그램을 크롬에서 reload → 쿠팡이츠 1개 매장 수집 시작 → 몇 페이지 진행 후 대시보드 "중단" 클릭 → 워크탭 콘솔에 `ESC 눌림` 또는 동등한 중단 로그가 즉시 출력되고 수집이 몇 초 내에 멈추는지 확인.

## Verification Loop
```
LOOP until all PASS:
  1. Test Cases 1~4 순서대로 실행
  2. FAIL 항목 → 원인 분석 (예: Sites 참조 오류, 콜백 스코프 오류, currentWorkTabId 리셋 누락)
  3. 코드 수정
  4. 전체 Test Cases 재실행
종료 조건: Test Cases 1~4 전체 PASS. Test Case 5는 사용자가 실기로 확인 (Codex는 결과를 기다리지 않고 안내 문구만 출력하고 종료 가능).
```

## Constraints
- `content/03_coupangeats.js`는 수정하지 않는다 (`_stopFlag` 체크 로직은 이미 정상 작동, 문제는 신호가 도달하지 않는 것뿐).
- `stopRequested`(runner.js 오케스트레이터 플래그)의 기존 동작은 그대로 유지 — 제거하거나 대체하지 말고 STOP 메시지 전송을 **추가**한다.
- ESC 키 중단 경로(`content/03_coupangeats.js`의 keydown 리스너)는 절대 건드리지 않는다 — 기존 정상 동작 회귀 방지.
- 두 폴더(레포 빌드본 + OneDrive 배포본) 모두 반드시 수정. 하나만 고치면 실사용 환경에서 버그가 재현된다.
- 레포 빌드본 파일에 이미 존재하는 mojibake(깨진 한글 주석/로그 문자열)는 이번 작업 범위에서 손대지 않는다 — 별도 인코딩 이슈이며 이번 수정과 무관.
