### 3. orders 페이지네이션 견고화 (B2, 부산장림점)
`DB_Beamin_04_orders.py`.

- `_click_next_page(driver, current_page)`(라인 1230): 현재는 `current_page+1` **숫자 버튼만** 찾아 클릭 → 배민 페이지네이션이 번호를 윈도우 단위(예: 1~10)로만 노출하면 다음 번호가 창 밖일 때 `False` 반환 → 조기 종료로 이후 페이지 전량 유실. 보강:
  - 목표 번호(`current_page+1`) 버튼이 있으면 클릭(기존 동작).
  - 없으면 **다음 윈도우/다음 화살표 버튼**(배민 `»`/next, 보통 `aria-label`/`disabled` 속성 보유)을 찾아 클릭. 클릭 후 목표 번호 버튼이 나타나면 그 번호를 클릭.
  - 다음 화살표가 없거나 `disabled`일 때만 진짜 마지막 페이지로 간주해 `False` 반환.
  - 실제 배민 DOM 셀렉터(`ul.Pagination_b_r4ax_pb5p4v5 button.Pagination_b_r4ax_pb5p4vb` 및 화살표 버튼)는 구현 시 페이지에서 확인 후 반영. 화살표 셀렉터가 불명확하면 `ul.Pagination_...` 내 마지막 `button`(숫자가 아닌) 후보를 사용.
- 전환 타임아웃 처리(라인 1116–1131): 즉시 `break` 대신 `_page_signature(driver)`로 새 페이지 활성 여부 재확인 후에만 종료. 종료 시 `logger.warning("페이지 전환 종료 — page=%d 이후 유실 가능: %s", page_num, store)`로 진단성 확보. (기존 `_wait_for_page_transition` 재클릭 1회 로직은 유지)
- 행 단위 silent skip(라인 1090–1096, `"행 처리 실패 (Chrome OOM?)"`): 스킵된 `data-index`를 카운트해 `_collect_all_pages` 종료 시 `logger.warning("행 스킵 %d건: %s", skipped, store)` 요약. **수집 로직 자체는 변경하지 말고 가시성만 추가.**
- 목표: `_validate_collected`(라인 97)의 `actual_count`/`actual_amount`가 페이지 `TotalSummary` 기대치와 일치 → `_collect_with_retry_on_mismatch`(라인 249) `합계 불일치` 경고 소멸. 상수 `_MAX_PAGES=50`, `_PAGE_TRANSITION_TIMEOUT=25`는 유지.

def _quit_driver_safely(driver: Any) -> None:      # 라인 177
    try:

def _build_dashboard_session(account: dict, metrics: dict, profile: dict) -> Any | None:  # 라인 185
    account_id = account["account_id"]
    max_recovery = int(profile["max_session_recovery_per_account"])
    for attempt in range(max_recovery + 1):
        driver = _build_account_session(account, metrics, profile)
        if driver is None: return None
        try:
            if not is_on_main_dashboard(driver.current_url):
                driver.set_page_load_timeout(45); driver.get("https://self.baemin.com/")
            return driver
        except Exception as exc:
            if _is_recoverable_session_issue(exc):
                _close_driver_safely(driver, account_id)
                if attempt < max_recovery:
                    metrics["session_recovery_count"] += 1; continue
                return None
            return driver
    return None

## Constraints
- **동작 보존 원칙**: §1 중복 제거는 "실행되던 마지막 정의"를 반드시 남긴다. 앞쪽(죽은) 정의를 남기고 뒤쪽을 지우면 동작이 바뀌므로 금지.
- 병렬화·계정/매장 동시 실행·대기시간(`account_wait_range` 등) 축소 **금지** (배민 봇탐지 위험).
- `_MAX_PAGES`, `_PAGE_TRANSITION_TIMEOUT`, `_MAX_VALIDATION_RETRY`, `_ORDER_COLLECTION_ATTEMPTS` 상수 값 변경 금지.
- 수집 로직(행 expand/extract/collapse, upsert by 주문번호)은 변경하지 말 것 — §3은 페이지 이동/가시성만 보강.
- print 금지, logger 사용. 기존 로그 메시지 포맷/한글 톤 유지.
- Selenium DOM 셀렉터는 추측하지 말고 기존 코드의 셀렉터 패턴을 따르며, 새 셀렉터(페이지네이션 화살표)는 불확실하면 fallback(마지막 non-numeric 버튼)으로 안전하게 처리.
