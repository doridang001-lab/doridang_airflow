# 나홀로 1인 곱도리탕 역삼점 쿠팡 자동수집 누락 수정

## Task
쿠팡 `doriys` 로그인 하나에 `닭도리탕 전문 도리당 역삼점`과 `나홀로 1인 곱도리탕 역삼점` 두 매장이 묶여 있으나, `accounts.js`에 `도리당 역삼점`만 등록돼 있어 자동 배치 수집이 `나홀로` 매장을 드롭다운에서 선택·다운로드하지 않는다. `accounts.js`에 `나홀로` 매장을 같은 로그인으로 등록해 두 매장 모두 수집되게 한다.

## Project Conventions
- 대상 프로젝트: 크롬 확장 수집기 (도리당 collector). **Python 프로젝트 아님.**
- 작업 폴더: `C:\Users\민준\OneDrive - 주식회사 도리당\Extention\doridang_collector_개발용`
- 계정 데이터: `accounts.js`의 `DEFAULT_ACCOUNTS` 객체 (담당자별 배열, 매장 가나다순)
- 각 항목 형식: `{ store: '매장명', platform: 'baemin'|'coupang', id: '로그인ID', pw: '비번' }`
- runner 큐 빌드: `runner.js`가 `platform==='coupang'` 항목을 `id`로 묶어 `acc.stores`에 매장명을 모음
- 매장명 정규화: 공백/`·ㆍ.,` 제거, `^닭도리탕전문` 접두어 제거 (`runner.js:80`, `03_coupangeats.js:418`)

## Files to Create / Modify
- **수정**: `accounts.js` — `심성준 이사` 섹션, `도리당 역삼점` 쿠팡 항목(58행) 뒤에 `나홀로` 매장 1줄 추가
- (생성 파일 없음)

## Implementation Steps
1. `accounts.js`의 `심성준 이사` 배열에서 현재 58행:
   ```js
   { store: '도리당 역삼점', platform: 'coupang', id: 'doriys', pw: 'ever123@' }
   ```
   이 배열의 마지막 요소이므로, 콤마를 추가하고 다음 줄을 삽입한다:
   ```js
   { store: '도리당 역삼점', platform: 'coupang', id: 'doriys', pw: 'ever123@' },
   { store: '나홀로 1인 곱도리탕 역삼점', platform: 'coupang', id: 'doriys', pw: 'ever123@' }
   ```
2. 배민(`platform: 'baemin'`)에는 **추가하지 않는다** (사용자 요청: 쿠팡만).
3. `store` 값은 쿠팡 드롭다운 표시 텍스트와 정확히 일치시킨다: `나홀로 1인 곱도리탕 역삼점`.

## Why this approach
- 시스템 전체(`runner.js`)가 `accounts.js` 등록 매장 기준으로 타깃 선정·대시보드 상태행·manifest CSV·재시도·top50/bottom50 분할을 수행한다.
- 같은 `id`(`doriys`)로 등록하면 runner가 한 로그인 세션에서 두 매장을 순회 수집한다 (`03_coupangeats.js:2173` 도리당 먼저, 나홀로 나중).
- 매칭/선택 로직(`_isTargetBrandStoreName` `03_coupangeats.js:408`, `_selectStoreInOrdersDropdown` :2362)은 이미 `나홀로`를 정상 처리한다. 누락 원인은 타깃 목록(`_filterTargetStores` :433)에 미등록뿐이므로 데이터 등록만으로 해결된다.
- 필터/runner 로직 수정은 배치 의미·분할·상태행 생성에 영향을 줘 더 위험하므로 채택하지 않는다.
