# 배민 수집 DAG — "항상 부분성공/오류" 근본 원인 제거

## Task
`DB_Beamin_Macro_Dags`가 실제로는 수집에 성공하는데도 매일 결과가 "부분성공"으로 뜨고 이메일/텔레그램에 깨진 글자와 WARNING 트레이스백이 찍힌다. 원인은 새 버그가 아니라 매 실행마다 재현되는 3개 고정 원인(① 로깅 포맷 버그 ② 소스 파일 모지바케 ③ DAG 중복 정의)이다. 이를 제거해 수집이 깨끗할 때 상태가 "성공"으로 정상화되도록 한다.

## Project Conventions
- logging: `logger = logging.getLogger(__name__)` — print 절대 금지

## Implementation Steps

### A. (필수) 로깅 포맷 버그 수정 — `DB_Beamin_combined.py:534`
현재(깨진+버그):
```python
logger.info("?ㅼ쓬 怨꾩젙源뚯? %.0f珥??湲?, wait_sec=%s", wait_sec)
```
→ 수정 (형제 파일 `DB_Beamin_02_woori_shop_click.py:172`, `DB_Beamin_03_shop_change.py:133`과 동일 형태):
```python
logger.info("다음 계정까지 %.0f초 대기", wait_sec)
```
이유: 포맷 지정자가 2개(`%.0f`, `%s`)인데 인자는 1개 → `record.getMessage()`에서 `TypeError: not enough arguments for format string` → Airflow `FileTaskHandler.emit`이 트레이스백을 WARNING으로 출력 → 상태가 강제로 "부분성공". 잉여 `, wait_sec=%s` 제거 + 한글 복원으로 동시 해결.

## Constraints
- `KNOWN_BRANDS = ["도리당", "나홀로"]` 등 **이미 정상인 한글 리터럴은 절대 건드리지 않는다**.
- 파일 전체 일괄 재디코딩/일괄 치환 금지 — 정상/깨짐 문자가 섞여 있어 정상 문자가 깨진다. 깨진 문자열만 표적 교정.
- 모든 저장은 **UTF-8** (cp949/euc-kr 금지).
- DAG 하단 PythonOperator 배선(`t_dash >> t0 >> t1 >> t2 >> t3 >> [t4,t5,t6] >> t7`)과 task_id는 변경 금지.
- 함수 **로직**은 바꾸지 않는다(중복 정의 중 죽은 쪽 삭제 + 문자열 복원 + 방어선 패턴 추가만). 동작 변경 아님.
- ToOrder 실제 불일치(2개 매장)는 손대지 않는다.
